Deployment Workflow

Determine Next Lab Number

# Check current lab directories ls -la courses/firstline/labs/
Action: Count the existing lab directories to determine next number (e.g., if labs 01-10 exist, next is 11)

Create Lab Directory

# Replace [number] with actual lab number (e.g., 11) mkdir -p courses/firstline/labs/lab[number]/

Create Lab HTML Files

# Create main lab page cat > courses/firstline/labs/lab[number]/index.html << 'EOF' [PASTE YOUR FULL LAB HTML CONTENT HERE - NO TRUNCATION] EOF # Create any additional lab files (like simulation HTML) cat > courses/firstline/labs/lab[number]/[filename].html << 'EOF' [PASTE ADDITIONAL HTML FILES IF NEEDED] EOF

Verify Files Created

# Check files were created correctly ls -la courses/firstline/labs/lab[number]/ cat courses/firstline/labs/lab[number]/index.html | head -20

Git Add & Commit

# Add the new lab directory git add courses/firstline/labs/lab[number]/ # Commit with descriptive message git commit -m "Add Lab [number]: [Lab Title]" # Verify commit git log --oneline -3

Deploy to Azure

# Push to trigger Azure deployment git push origin master

Wait for Deployment

‚è±Ô∏è Wait 2-3 minutes for Azure Static Web Apps to deploy
Check deployment status in Azure portal if needed

Test Lab Access

  • Navigate to: https://lively-ocean-054bef103.2.azurestaticapps.net/courses/firstline/labs/lab[number]/
  • Verify lab loads correctly
  • Test all interactive elements

Update Main Portal Index

# Edit the main index.html to add new lab to listing # Add new lab card to appropriate section # (ServiceNow ITSM Labs or BIOS Configuration Labs)

Deploy Index Update

git add index.html git commit -m "Update portal: Add Lab [number] to main index" git push origin master

Final Verification

  • Wait 2-3 minutes for deployment
  • Visit main portal: https://lively-ocean-054bef103.2.azurestaticapps.net/
  • Verify new lab appears in the correct section
  • Test navigation to the new lab
‚úÖ Deployment Complete! Your lab is now live and accessible.
