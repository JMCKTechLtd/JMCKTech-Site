Welcome to the RHCSA System Log Files Lab. In this lab, you will learn how to analyze system logs using both traditional and modern tools.

1 List Log Files Current Step

First, let's list the log files in /var/log.

Type this command exactly:
ls -l /var/log

Understanding /var/log Directory

How it works: The /var/log directory contains traditional log files. Common files include messages (general system logs), secure (authentication logs), and boot.log (boot messages).

2 View Messages Log Pending

Now let's view the last 20 lines of the messages log.

Type this command exactly:
sudo tail -20 /var/log/messages

Understanding tail Command

How it works: The tail command displays the last lines of a file. The -20 flag shows the last 20 lines. This is useful for viewing recent log entries.

3 View Journal Pending

Now let's view the last 10 journal entries.

Type this command exactly:
journalctl -n 10

Understanding journalctl

How it works: journalctl queries the systemd journal, which stores logs from all system services. The -n flag limits output to the specified number of entries.

4 Filter by Time Pending

Now let's filter logs from the last hour.

Type this command exactly:
journalctl --since "1 hour ago"

Understanding Time Filtering

How it works: The --since option filters logs by time. You can use relative time ("1 hour ago") or absolute time ("2024-12-04 10:00:00").

5 Filter by Priority Pending

Now let's view only error messages.

Type this command exactly:
journalctl -p err

Understanding Priority Levels

How it works: The -p option filters by priority. Levels include: emerg, alert, crit, err, warning, notice, info, debug. This helps focus on important messages.

6 View Unit Logs Pending

Finally, let's view logs for the sshd service.

Type this command exactly:
journalctl -u sshd

Understanding Unit Filtering

How it works: The -u option filters logs for a specific systemd unit (service). This is essential for troubleshooting specific services.
