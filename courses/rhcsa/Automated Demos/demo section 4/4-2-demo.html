Welcome to the RHCSA ACL Lab. In this lab, you will learn how to manage Access Control Lists.

1 Check ACL Support Current Step

First, let's verify that the filesystem has ACL support enabled.

Type this command exactly:
mount | grep acl

Understanding ACL Support

How it works: Filesystems must be mounted with ACL support. Most modern Linux filesystems (ext4, XFS) enable this by default.

2 View Existing ACLs Pending

Now let's view ACLs on an existing file.

Type this command exactly:
getfacl /etc/passwd

Understanding getfacl

How it works: getfacl displays the file owner, group, and any additional ACL entries beyond standard permissions.

3 Set User ACL Pending

Now let's create a test file and set a user ACL.

Create test file:
touch testfile
Set user ACL:
setfacl -m u:student:rw testfile
View the ACL:
getfacl testfile

Understanding setfacl for Users

How it works: setfacl -m u:username:perms grants specific permissions to a user. Format: r=read, w=write, x=execute.

4 Set Group ACL Pending

Now let's add a group ACL.

Set group ACL:
setfacl -m g:developers:rx testfile
View the ACL:
getfacl testfile

Understanding setfacl for Groups

How it works: setfacl -m g:groupname:perms grants permissions to all members of a group.

5 Remove ACL Entry Pending

Now let's remove the group ACL.

Remove ACL:
setfacl -x g:developers testfile
Verify removal:
getfacl testfile

Understanding ACL Removal

How it works: setfacl -x removes specific ACL entries. Use -b to remove all ACLs.

6 Set Default ACL Pending

Finally, let's set a default ACL on a directory.

Create directory:
mkdir testdir
Set default ACL:
setfacl -m d:u:student:rwx testdir
View default ACL:
getfacl testdir

Understanding Default ACLs

How it works: Default ACLs (d:) apply to new files created in a directory. Only directories can have default ACLs.
