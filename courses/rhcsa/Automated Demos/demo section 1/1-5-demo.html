1 Creating Files with nano Current Step

First, create a file using the nano text editor.

Command to run in the real lab:
nano my_first_file.txt

Nano Editor Basics

After opening nano, follow these steps:

  1. Type some text, for example: Hello, this is my first file
  2. Press Ctrl+O to save (Write Out)
  3. Press Enter to confirm the filename
  4. Press Ctrl+X to exit nano

Understanding nano

What Simple terminal-based text editor
Why Easy to learn and ideal for quick edits
Where Commonly installed on most Linux systems
When When you need quick edits without advanced features

Keys to remember: Ctrl+O to write the file, Ctrl+X to exit.

2 Viewing and Listing Files Pending

Next, view the file you created and check its details with cat and ls.

Commands to run:
cat my_first_file.txt
ls -l my_first_file.txt

cat and ls -l

cat Displays file contents to the terminal
ls -l Shows detailed file information (permissions, owner, size, date)

Use these commands regularly to confirm that files were created and saved correctly.

3 Creating a File with vim Pending

Now create a second file using the vim editor.

Command to run:
vim my_second_file.txt

Vim Basics (Insert and Save)

  1. Press i to enter Insert mode.
  2. Type some text, for example: This is my second file created with vim
  3. Press Esc to return to Normal mode.
  4. Type :wq and press Enter to write the file and quit.

Understanding vim

What Powerful modal text editor
Why Efficient for advanced text editing
Where Installed by default on RHEL/Rocky/AlmaLinux
When When doing system admin or repeated edits

Key idea: vim has modes. Learn Insert, Normal, and how to save and quit.

4 Editing the File Again with vim Pending

Reopen the file in vim and perform a simple edit to simulate advanced operations.

Command to run:
vim my_second_file.txt

Example Edit Workflow

  1. Press i to enter Insert mode and add another line.
  2. Press Esc to return to Normal mode.
  3. Use the arrow keys to move around the file.
  4. Type :wq to save and quit again.

5 Comparing Files and Listing Text Files Pending

Finally, display the contents of both files and list all .txt files in the current directory.

Commands to run:
cat my_first_file.txt
cat my_second_file.txt
ls -l *.txt

Verifying Your Work

Reading files with cat and listing them with ls -l helps you verify that both editors created and saved files as expected.
