1 Basic grep Usage Current Step

Start with basic grep usage to search for simple text patterns in a log file.

Type these commands exactly in the lab:
grep "error" sample.log

Then:

grep -i "warning" sample.log

Understanding grep Basics

What Searches for patterns in text
Why To find specific information in files
Where Works on logs or any text file
When Whenever you need to filter or search text

How it works: grep scans each line of input for the given pattern. The -i option makes the search case-insensitive.

Option Description Example
-i Case-insensitive search grep -i "hello" file.txt
-v Invert match (show non-matching lines) grep -v "success" log.txt
-n Show line numbers grep -n "error" log.txt

Understanding Anchors

What ^ = start of line, $ = end of line
Why To find text in a precise position
Where Used in most regex engines
When When you only care about beginnings/ends of lines

3 Character Classes Pending

Use character classes to match sets of characters.

Type these commands in the lab:
grep "[0-9]%" sample.log

Then:

grep "[AEIOU]" sample.log

Understanding Character Classes

What Match any character from a set
Why To match variations of a pattern
Where Inside square brackets [ ]
When When you need flexible matching

How it works: [0-9] matches any digit, [AEIOU] matches any uppercase vowel. Use [^0-9] to match non-digits.

Pattern Matches Example
[abc] a, b, or c gr[ae]y (gray or grey)
[a-z] Any lowercase letter [a-z]@domain.com
[^0-9] Any non-digit [^0-9]%

4 Extended Regular Expressions Pending

Use extended regular expressions for more powerful pattern matching.

Type these commands in the lab:
grep -E "error|warning" sample.log

Then:

grep -E "[0-9]{3}-[0-9]{2}-[0-9]{4}" data.txt

Understanding Extended Regex

What More powerful pattern matching
Why For complex matching requirements
Where With grep -E or egrep
When When basic patterns are not enough

How it works: | means OR, and {n} means ‚Äúexactly n occurrences‚Äù. The SSN pattern [0-9]{3}-[0-9]{2}-[0-9]{4} matches values like 123-45-6789.

Pattern Description Example
error|warning error OR warning grep -E "error|warning"
[0-9]{3} Exactly three digits [0-9]{3}-[0-9]{2}-[0-9]{4}
