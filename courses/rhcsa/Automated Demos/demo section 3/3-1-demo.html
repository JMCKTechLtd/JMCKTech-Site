Welcome to the RHCSA Partition Management Lab. In this lab, you will learn how to work with disk partitions using multiple tools.

1 List Block Devices Current Step

First, let's list all block devices on the system.

Type this command exactly:
lsblk

Understanding lsblk

How it works: The lsblk command lists all block devices in a tree format, showing the relationships between disks, partitions, and mount points.

2 List Partitions with fdisk Pending

Now let's examine the partition table of a specific disk.

Type this command exactly:
sudo fdisk -l /dev/sda

Understanding fdisk -l

How it works: The fdisk -l command lists the partition table for the specified disk, showing partition types, sizes, and boundaries.

3 Interactive fdisk Pending

Now let's use fdisk interactively to manage partitions.

Type this command exactly:
sudo fdisk /dev/sdb

Understanding fdisk Interactive Mode

How it works: Interactive fdisk provides commands to create, delete, and modify partitions. Common commands: n (new), d (delete), p (print), w (write), q (quit).

4 Use gdisk for GPT Pending

Now let's use gdisk for GPT partition tables.

Type this command exactly:
sudo gdisk /dev/sdc

Understanding gdisk

How it works: gdisk is used for GPT (GUID Partition Table) disks, which support larger disks and more partitions than MBR. It has similar commands to fdisk.

5 Use parted Pending

Finally, let's use parted, a more modern partitioning tool.

Type this command exactly:
sudo parted /dev/sdd

Understanding parted

How it works: parted supports both MBR and GPT partition tables and can resize partitions. Use 'print' to view partitions and 'mkpart' to create new ones.
