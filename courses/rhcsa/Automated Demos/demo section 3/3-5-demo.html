Welcome to the RHCSA Swap Management Lab. In this lab, you will learn how to manage swap space.

1 Check Current Swap Current Step

First, let's check the current swap configuration.

Type this command exactly:
swapon --show

Understanding Swap Space

How it works: Swap space is disk space used when RAM is full. The swapon --show command displays all active swap areas.

2 Check Memory Usage Pending

Now let's check overall memory and swap usage.

Type this command exactly:
free -h

Understanding Memory Statistics

How it works: The free -h command displays memory statistics in human-readable format, showing total, used, and available memory plus swap.

3 Create Swap File Pending

Now let's create a 512MB swap file.

Type this command exactly:
sudo fallocate -l 512M /swapfile

Understanding fallocate

How it works: The fallocate command quickly creates a file of specified size. It's faster than dd for creating swap files.

4 Set Permissions and Format Pending

Now let's secure the file and format it as swap.

First, set secure permissions:
sudo chmod 600 /swapfile
Then format as swap:
sudo mkswap /swapfile

Understanding Swap Security

How it works: Swap files must have 600 permissions (root only) for security. The mkswap command formats the file as swap space.

5 Activate and Verify Swap Pending

Now let's activate the swap file and verify it.

Activate swap:
sudo swapon /swapfile
Verify activation:
swapon --show

Understanding swapon

How it works: The swapon command activates a swap area. Verify it appears in swapon --show output.

6 Deactivate Swap Pending

Now let's practice deactivating swap.

Type this command exactly:
sudo swapoff /swapfile

Understanding swapoff

How it works: The swapoff command deactivates a swap area. Use this before removing a swap file or partition.

7 Configure Persistent Swap Pending

Now let's make the swap persistent across reboots.

Add to fstab:
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
Activate all swap:
sudo swapon -a
Final verification:
swapon --show

Understanding Persistent Configuration

How it works: Adding swap to /etc/fstab ensures it's activated at boot. Use swapon -a to test without rebooting.
