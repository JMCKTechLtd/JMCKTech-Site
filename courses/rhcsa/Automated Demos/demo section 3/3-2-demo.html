0 Lab Introduction Completed

Welcome to the RHCSA LVM Management Lab. In this lab, you will learn how to work with LVM components including physical volumes, volume groups, and logical volumes.

1 Create Physical Volume Current Step

First, let's create a physical volume from the disk.

Type this command exactly:
sudo pvcreate /dev/sdb

Understanding Physical Volumes

How it works: A physical volume (PV) is a disk or partition that has been initialized for use with LVM. The pvcreate command prepares the storage device.

2 Display Physical Volume Pending

Now let's verify the physical volume was created.

Type this command exactly:
sudo pvdisplay

Understanding pvdisplay

How it works: The pvdisplay command shows detailed information about physical volumes including size, UUID, and which volume group they belong to.

3 Create Volume Group Pending

Now let's create a volume group from the physical volume.

Type this command exactly:
sudo vgcreate vg_data /dev/sdb

Understanding Volume Groups

How it works: A volume group (VG) combines one or more physical volumes into a storage pool. The vgcreate command creates a new volume group.

4 Display Volume Group Pending

Now let's verify the volume group.

Type this command exactly:
sudo vgdisplay vg_data

Understanding vgdisplay

How it works: The vgdisplay command shows volume group details including total size, free space, and the number of physical volumes.

5 Create Logical Volume Pending

Now let's create a logical volume within the volume group.

Type this command exactly:
sudo lvcreate -n lv_app -L 2G vg_data

Understanding Logical Volumes

How it works: A logical volume (LV) is created from free space in a volume group. The -n flag names it, and -L specifies the size.

6 Display and Format Logical Volume Pending

Now let's verify the logical volume and format it.

First, display the LV:
sudo lvdisplay /dev/vg_data/lv_app
Then format with XFS:
sudo mkfs.xfs /dev/vg_data/lv_app
Then check the UUID:
sudo blkid /dev/vg_data/lv_app

Understanding Filesystem Creation

How it works: After creating a logical volume, format it with a filesystem. XFS is the default filesystem for RHEL. Use blkid to verify the UUID.

7 Extend Logical Volume Pending

Now let's extend the logical volume and grow the filesystem.

First, extend the LV:
sudo lvextend -L +1G /dev/vg_data/lv_app
Then grow the filesystem:
sudo xfs_growfs /dev/vg_data/lv_app

Understanding LV Extension

How it works: Use lvextend to increase the logical volume size. Then use xfs_growfs (for XFS) or resize2fs (for ext4) to grow the filesystem.
