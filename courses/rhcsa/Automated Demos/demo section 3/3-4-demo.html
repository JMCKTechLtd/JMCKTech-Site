Welcome to the RHCSA Mount/Unmount Lab. In this lab, you will learn how to manage filesystem mounts.

1 List Current Mounts Current Step

First, let's list all currently mounted filesystems.

Type this command exactly:
mount

Understanding mount Command

How it works: The mount command without arguments displays all currently mounted filesystems, showing device, mount point, filesystem type, and mount options.

2 Mount Filesystem Temporarily Pending

Now let's mount a filesystem to /mnt temporarily.

Type this command exactly:
sudo mount /dev/sdb1 /mnt

Understanding Temporary Mounts

How it works: The mount command attaches a filesystem to a directory. Temporary mounts are lost after reboot unless added to /etc/fstab.

3 Verify Mount Pending

Now let's verify the filesystem is mounted.

Type this command exactly:
mount | grep /mnt

Understanding Verification

How it works: Using grep to filter the mount output helps verify specific mounts. You can also use df -h or findmnt.

4 Unmount Filesystem Pending

Now let's unmount the filesystem and verify.

First, unmount:
sudo umount /mnt
Then verify it's unmounted:
mount | grep /mnt

Understanding Unmounting

How it works: The umount command detaches a filesystem. Ensure no processes are using the filesystem before unmounting.

5 Configure Persistent Mount Pending

Now let's configure a persistent mount in /etc/fstab.

Type this command exactly:
sudo vim /etc/fstab

Understanding /etc/fstab

How it works: The /etc/fstab file defines filesystems to mount at boot. Format: device, mount point, type, options, dump, fsck.

6 Mount All Filesystems Pending

Finally, let's mount all filesystems defined in /etc/fstab.

Type this command exactly:
sudo mount -a

Understanding mount -a

How it works: The mount -a command mounts all filesystems listed in /etc/fstab. Use this to test fstab changes without rebooting.
