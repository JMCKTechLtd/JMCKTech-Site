Welcome to the RHCSA Filesystem Creation Lab. In this lab, you will learn how to create different filesystem types.

1 List Block Devices Current Step

First, let's list the available block devices and partitions.

Type this command exactly:
lsblk

Understanding Block Devices

How it works: The lsblk command lists all available block devices, showing disks and their partitions in a tree structure.

2 Create ext4 Filesystem Pending

Now let's create an ext4 filesystem on the first partition.

Type this command exactly:
sudo mkfs.ext4 /dev/sdb1

Understanding ext4

How it works: ext4 is a mature, reliable filesystem commonly used in Linux. It supports large files and volumes, journaling, and is well-tested.

3 Create XFS Filesystem Pending

Now let's create an XFS filesystem on the second partition.

Type this command exactly:
sudo mkfs.xfs /dev/sdb2

Understanding XFS

How it works: XFS is the default filesystem for RHEL. It's a high-performance filesystem designed for scalability and large files.

4 Create vfat Filesystem Pending

Now let's create a vfat filesystem on the third partition.

Type this command exactly:
sudo mkfs.vfat /dev/sdb3

Understanding vfat

How it works: vfat (FAT32) is useful for compatibility with Windows and other operating systems. It has file size limitations (4GB max).

5 Check Filesystem UUIDs Pending

Now let's check the UUIDs and filesystem types.

Type this command exactly:
sudo blkid /dev/sdb1 /dev/sdb2 /dev/sdb3

Understanding blkid

How it works: The blkid command displays block device attributes, including UUIDs and filesystem types. UUIDs are unique identifiers.

6 Verify Filesystems Pending

Finally, let's verify all filesystems are properly created.

Type this command exactly:
lsblk -f

Understanding lsblk -f

How it works: The lsblk -f command lists block devices with filesystem information, including type, label, UUID, and mount point.
