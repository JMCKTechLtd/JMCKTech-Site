Welcome to the RHCSA Service Management Lab. Learn to control system services with systemctl.

1 Check Service Status Current Step

First, let's check the status of the SSH service.

Type this command exactly:
systemctl status sshd

Understanding systemctl status

How it works: systemctl status shows if a service is active, enabled, and displays recent log entries.

2 Stop Service Pending

Now let's stop the SSH service.

Stop sshd:
sudo systemctl stop sshd
Verify:
systemctl status sshd

Understanding systemctl stop

How it works: systemctl stop stops a running service immediately. Changes are temporary (until reboot).

3 Start Service Pending

Now let's start the service again.

Start sshd:
sudo systemctl start sshd

Understanding systemctl start

How it works: systemctl start starts a stopped service. Use restart to stop and start in one command.

4 Enable/Disable Service Pending

Now let's control whether the service starts at boot.

Disable at boot:
sudo systemctl disable sshd
Check enabled status:
systemctl is-enabled sshd
Enable at boot:
sudo systemctl enable sshd

Understanding enable/disable

How it works: systemctl enable configures a service to start at boot. disable prevents auto-start.

5 Legacy Service Command Pending

Finally, let's use the legacy service command.

Restart with service:
sudo service sshd restart
List all services:
service --status-all

Understanding Legacy Commands

How it works: The service command is a legacy wrapper for systemctl. Use systemctl for new scripts.
