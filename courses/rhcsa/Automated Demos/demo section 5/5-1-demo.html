Welcome to the RHCSA Task Scheduling Lab. Learn to automate tasks using cron and at.

1 View Current Crontab Current Step

First, let's check if there are any existing cron jobs.

Type this command exactly:
crontab -l

Understanding crontab

How it works: crontab -l lists scheduled cron jobs for the current user. Each job runs at specified intervals.

2 Create Cron Job Pending

Now let's create a cron job that runs every minute.

Edit crontab:
crontab -e
Then verify:
crontab -l

Understanding Cron Format

How it works: Cron format: * * * * * command (minute hour day month weekday). Example: */5 * * * * runs every 5 minutes.

3 Verify Cron Job Output Pending

Check if the cron job created the log file.

Check log file:
ls ~/cron_test.log

Understanding Cron Output

How it works: Cron jobs can redirect output to files. Use >> to append or > to overwrite.

4 Schedule Task with at Pending

Now let's schedule a one-time task using the at command.

Schedule task:
echo "echo 'AT test' >> ~/at_test.log" | at now + 1 minute
View queue:
atq

Understanding at Command

How it works: at schedules one-time tasks. Use times like: now + 1 hour, 10:00 AM, tomorrow, etc.

5 Remove Scheduled Task Pending

Finally, let's remove the scheduled at job.

Remove job:
atrm 1
Verify removal:
atq

Understanding Job Management

How it works: atrm removes scheduled at jobs by job number. atq shows remaining jobs in the queue.
