Welcome to the RHCSA Compressed Backup Lab. In this lab, you will create a compressed backup of a directory using tar with bzip2 compression.

Follow the steps in order. You must complete each step before moving to the next one.

1 Verify Source Directory Current Step

First, let's verify that the source directory exists and check its contents.

Type this command exactly:
ls -la /usr/local/

This will show us the directory structure and confirm what we'll be backing up.

Understanding /usr/local

The /usr/local directory is used for locally installed software and data that's not part of the base operating system.

It typically contains:

  • Locally compiled programs in bin/ and sbin/
  • Local libraries in lib/
  • Configuration files in etc/
  • Documentation in share/

2 Navigate to Source Directory Pending

Now let's navigate to the source directory to create our backup from the correct location.

Type this command exactly:
cd /usr/local

This ensures we're in the right directory when we create the backup.

Why Navigate First?

Navigating to the source directory before creating the backup has several advantages:

  • Simplifies the tar command syntax
  • Ensures relative paths in the archive
  • Prevents including full pathnames in the archive
  • Makes restoration to different locations easier

3 Create Compressed Backup Pending

Now let's create the compressed backup using tar with bzip2 compression.

Type this command exactly:
tar -jcvf /root/backup.tar.bz2 .

This command creates a bzip2-compressed tar archive of the current directory.

Understanding tar Options

The tar command options used:

  • -j - Use bzip2 compression
  • -c - Create a new archive
  • -v - Verbose output (show files being archived)
  • -f - Specify the archive filename
  • . - Archive the current directory contents

Alternative compression options:

  • -z - gzip compression (.tar.gz)
  • -J - xz compression (.tar.xz)
  • No flag - No compression (.tar)

4 Verify Backup File Pending

Let's verify that the backup file was created successfully and check its properties.

Type these commands one by one:
ls -lh /root/backup.tar.bz2

Should show the backup file with its size

file /root/backup.tar.bz2

Should confirm it's a bzip2 compressed tar archive

tar -jtvf /root/backup.tar.bz2 | head -10

Should list the first 10 files in the archive to verify contents

Backup Verification

Always verify backups after creation:

  • ls -lh - Check file exists and size is reasonable
  • file - Confirm file type and compression
  • tar -jtvf - List archive contents without extracting
  • tar -jtf - Same but without verbose output

5 Test Backup Restoration Pending

Finally, let's test the backup by extracting it to a temporary location to ensure it works correctly.

Type these commands one by one:
mkdir -p /tmp/test-restore

Create a temporary directory for testing

cd /tmp/test-restore

Navigate to the test directory

tar -jxvf /root/backup.tar.bz2

Extract the backup to verify it works

ls -la

Verify the contents were extracted correctly

Testing Backups

Regularly testing backups is crucial for disaster recovery:

  • Test extraction to verify archive integrity
  • Compare file counts and sizes with original
  • Test on a different system if possible
  • Document the restoration procedure

Remember to clean up test directories after verification.
