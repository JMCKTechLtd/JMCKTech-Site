File Operations Concepts

find command Search for files with specific criteria:
find [path] [options] [actions]
Common options: -user, -name, -type
exec action Execute command on found files:
-exec command {} \;
{} is replaced by each found file
\; terminates the command
cp command Copy files with options:
-r: recursive
-p: preserve attributes
-f: force overwrite
Directory creation mkdir creates directories:
mkdir /path/to/dir
Use -p to create parent directories if needed

Step 1: Create Target Directory

First, create the directory where we'll copy the files:

Create the directory:

Type: mkdir -p /opt/dir

The -p option ensures parent directories are created if needed.

Understanding Directory Creation

The /opt directory is a standard location for optional software packages. We're creating a subdirectory here to store our copied files.

Step 2: Find Files Owned by harry

Use the find command to locate all files owned by user 'harry':

Basic find command:

Type: find / -user harry

This searches the entire filesystem for files owned by harry.

Understanding the find Command

The find command is powerful for locating files based on various criteria:

  • /: Start searching from root directory
  • -user harry: Only show files owned by user 'harry'
  • You can add -type f to only find regular files (not directories)

Step 3: Copy Files to Target Directory

Combine the find command with copy operation to move all files:

Complete copy command:

Type: find / -user harry -exec cp -rfp {} /opt/dir/ \;

This command does the following:

  • find / -user harry: Finds all files owned by harry
  • -exec ... \;: Executes the following command for each found file
  • cp -rfp: Copies files recursively, forcing overwrites, preserving attributes
  • {}: Placeholder for each found file
  • /opt/dir/: Destination directory

Understanding the Copy Operation

The -exec option allows you to run commands on each found file. Important flags for cp:

  • -r: Recursive copy (for directories)
  • -f: Force overwrite if files exist
  • -p: Preserve file attributes (owner, permissions, timestamps)

Step 4: Verify the Results

Check that the files were copied correctly:

List copied files:

Type: ls -l /opt/dir

This should show all files owned by harry that were copied.

Check file attributes:

Type: ls -l /opt/dir | head -n 3

Verify the owner and permissions were preserved.

Verification Tips

When verifying:

  • Check that all expected files are present
  • Verify ownership and permissions match originals
  • For large numbers of files, use wc -l to count them
  • Compare with original locations if needed
