0 Lab Introduction Completed

Welcome to the RHCSA Swap Partition Lab. In this lab, you will add additional swap space to your system.

Follow the steps in order. You must complete each step before moving to the next one.

1 Check Current Disk Layout Current Step

First, let's check the current disk layout to identify available space.

Type this command exactly:
fdisk -l

This will show all disks and partitions on your system.

Understanding Disk Layout

The fdisk -l command displays partition tables for all available disks.

Look for available space or unused partitions where you can create the swap partition.

2 Create Swap Partition Pending

Now let's create a new partition for swap. We'll use /dev/vda as our target disk.

Type this command exactly:
fdisk -cu /dev/vda

Once in fdisk, follow these steps:

  1. Press n to create a new partition
  2. Press e for extended partition or p for primary
  3. Accept default first sector by pressing Enter
  4. For size, type +754M and press Enter
  5. Press t to change partition type
  6. Select partition number (likely 1-5)
  7. Type 82 for Linux swap
  8. Press p to verify the partition table
  9. Press w to write changes and exit

Understanding Partition Types

Partition type 82 indicates a Linux swap partition.

The +754M specifies the partition size as 754 megabytes.

3 Format Partition as Swap Pending

Now let's format the new partition as swap space.

Type this command exactly:
mkswap /dev/vda5

Note: The partition number might be different. Use the partition you created in the previous step.

Understanding Swap Formatting

The mkswap command sets up a Linux swap area on a device or in a file.

This prepares the partition to be used as virtual memory by the system.

4 Enable Swap Partition Pending

Now let's enable the swap partition so the system can use it.

Type this command exactly:
swapon /dev/vda5

Then verify the swap is active:

swapon -s

Understanding Swap Activation

The swapon command enables swap space for paging and swapping.

swapon -s displays a summary of swap usage and configuration.

5 Configure Automatic Mounting Pending

To make the swap partition persistent across reboots, we need to add it to /etc/fstab.

Type this command exactly:
vim /etc/fstab

Add this line to the file:

/dev/vda5 swap swap defaults 0 0

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Understanding fstab Configuration

The /etc/fstab file contains information about filesystems and swap partitions.

The format for swap entries is: device swap swap options dump pass

For swap partitions, both dump and pass should be set to 0.

6 Verify Configuration Pending

Finally, let's verify that our swap configuration is working correctly.

Type these commands one by one:
free -h

Should show the additional swap space

swapon --show

Should display all active swap partitions

cat /proc/swaps

Should show swap partitions information

Verification Commands

After configuring swap, always verify with these commands:

  • free -h - Shows memory usage including swap
  • swapon --show - Displays active swap partitions
  • cat /proc/swaps - Shows kernel swap information
