Network Configuration Methods

Global Network Settings /etc/sysconfig/network contains:
NETWORKING=yes
HOSTNAME=station?.example.com
GATEWAY=X.X.X.X
Interface Settings ifcfg-eth0 contains:
DEVICE=eth0
BOOTPROTO=static
IPADDR=X.X.X.X
NETMASK=X.X.X.X
GATEWAY=X.X.X.X
Service Management systemctl restart network
or
ifdown eth0 && ifup eth0
Verification ip addr show eth0
ip route show
ping 192.168.1.1

Step 1: Configure Global Network Settings

Edit the global network configuration file:

Edit /etc/sysconfig/network:

Type: vim /etc/sysconfig/network

Ensure the file contains these settings:

NETWORKING=yes HOSTNAME=station50.example.com GATEWAY=192.168.0.254

Understanding Global Network Settings

The /etc/sysconfig/network file contains system-wide network configuration:

  • NETWORKING=yes: Enables networking
  • HOSTNAME: Sets the system hostname
  • GATEWAY: Default gateway (can also be set in interface config)

Step 2: Configure Network Interface

Edit the network interface configuration file:

Navigate to network scripts directory:

Type: cd /etc/sysconfig/network-scripts/

Edit the interface configuration (ifcfg-eth0):

Type: vim ifcfg-eth0

Ensure the file contains these settings:

DEVICE=eth0 ONBOOT=yes BOOTPROTO=static IPADDR=192.168.0.40 NETMASK=255.255.255.0 GATEWAY=192.168.0.254

Understanding Interface Configuration

The interface configuration file controls how your network interface behaves. Key parameters:

  • BOOTPROTO=static: Uses static IP configuration
  • IPADDR: Sets the static IP address
  • NETMASK: Defines the network mask
  • GATEWAY: Default gateway (optional if set in global config)

Step 3: Apply Network Changes

Restart network service:

Type: systemctl restart network

or

Type: ifdown eth0 && ifup eth0

Verify configuration:

Type: ip addr show eth0 (should show 192.168.0.40)

Type: ip route show (should show gateway 192.168.0.254)

Network Verification

After making changes, always verify with these commands:

  • ip addr: Shows IP address configuration
  • ip route: Displays routing table
  • ping: Tests network connectivity

Step 4: Test Network Connectivity

Ping host on 192.168.1.0 network:

Type: ping -c 3 192.168.1.254

You should see successful ping responses

Understanding Multi-Network Configuration

This lab simulates a server with interfaces on two networks (192.168.0.0 and 192.168.1.0). The server should be able to:

  • Communicate on its primary network (192.168.0.0)
  • Route traffic to the secondary network (192.168.1.0)
  • Respond to pings from both networks
