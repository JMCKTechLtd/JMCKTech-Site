Welcome to the RHCSA String Search and File Processing Lab. In this lab, you will search for specific strings in a file and save the results to a new file.

Follow the steps in order. You must complete each step before moving to the next one.

1 Understand the grep Command Current Step

The grep command is used to search for patterns in files. We'll use it to find lines containing the string "ich".

Understanding grep

The grep command searches for patterns in files and prints matching lines. Key options:

  • grep pattern file - Basic search
  • grep -i pattern file - Case-insensitive search
  • grep -v pattern file - Invert match (show non-matching lines)
  • grep -n pattern file - Show line numbers

We'll use output redirection (>) to save the results to a file.

2 Search for String and Save Output Pending

Now let's search for lines containing "ich" and save them to the output file.

Type this command exactly:
grep ich /usr/share/mime/packages/freedesktop.org.xml > /root/lines

This command:

  • Searches for "ich" in the specified file
  • Uses > to redirect output to /root/lines
  • Preserves the original line order automatically
  • Does not include blank lines in the output

Output Redirection

The > operator redirects standard output to a file, overwriting the file if it exists. To append instead of overwrite, use >>.

3 Verify the Output File Pending

Let's verify that the output file was created correctly with the matching lines.

Type this command exactly:
cat /root/lines

This should display all lines that contain the string "ich" from the original file.

You can also check the line count:
wc -l /root/lines

This will show how many lines were saved to the output file.

4 Verify No Blank Lines Pending

Let's verify that the output file contains no blank lines.

Type this command exactly:
grep -c '^$' /root/lines

This command counts blank lines (lines with only start and end, no content). The result should be 0.

Regular Expressions in grep

The pattern '^$' is a regular expression that matches blank lines:

  • ^ - Beginning of line
  • $ - End of line
  • No characters between them means an empty line

5 Compare with Original File Pending

Finally, let's verify that the lines in the output file are exact copies of the original.

Type these commands to compare:
head -5 /root/lines

Shows the first 5 lines of the output file.

grep ich /usr/share/mime/packages/freedesktop.org.xml | head -5

Shows the first 5 matching lines from the original file for comparison.

File Comparison Tools

Other useful commands for file comparison:

  • diff file1 file2 - Shows differences between files
  • cmp file1 file2 - Compares two files byte by byte
  • md5sum file - Generates checksum for file verification
