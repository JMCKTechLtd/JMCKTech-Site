0 Lab Introduction Completed

Welcome to the RHCSA Logical Volume Management Lab. In this lab, you will create and configure LVM components including physical volumes, volume groups, and logical volumes.

Follow the steps in order. You must complete each step before moving to the next one.

1 Create Physical Volumes Current Step

First, let's create physical volumes from the available disk partitions.

Type these commands exactly:
pvcreate /dev/sda7 /dev/sda8

This will initialize the partitions as physical volumes for LVM.

Understanding Physical Volumes

Physical volumes (PVs) are the base building blocks of LVM. They are typically disk partitions or entire disks that have been initialized for use with LVM.

2 Create Volume Group Pending

Now let's create a volume group with 16MB extents.

Type this command exactly:
vgcreate -s 16M vg1 /dev/sda7 /dev/sda8

This creates a volume group named "vg1" with 16MB physical extents from the specified physical volumes.

Understanding Volume Groups

Volume groups (VGs) pool together physical volumes to create a storage pool. The extent size determines the smallest allocatable unit of space.

3 Create Logical Volume Pending

Now let's create a logical volume with 50 extents.

Type this command exactly:
lvcreate -l 50 -n lvm02 vg1

This creates a logical volume named "lvm02" with 50 extents (800MB total) from volume group "vg1".

Understanding Logical Volumes

Logical volumes (LVs) are the usable "partitions" created from volume groups. They can be resized, moved, and have snapshots taken.

4 Create Filesystem Pending

Now let's format the logical volume with the ext4 filesystem.

Type this command exactly:
mkfs.ext4 /dev/vg1/lvm02

This creates an ext4 filesystem on the logical volume, making it ready for use.

5 Create Mount Point Pending

Now let's create the mount point directory.

Type this command exactly:
mkdir -p /mnt/data

This creates the directory where we will mount our logical volume.

6 Configure Auto-Mount Pending

Now let's configure the system to automatically mount the logical volume at boot.

Type this command exactly:
vim /etc/fstab

Add this line to the file:

/dev/vg1/lvm02 /mnt/data ext4 defaults 0 0

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

7 Mount and Verify Pending

Finally, let's mount the filesystem and verify our configuration.

Type these commands one by one:
mount -a

Mounts all filesystems defined in /etc/fstab

mount | grep /mnt/data

Should show the mounted logical volume

df -h /mnt/data

Should show the filesystem usage

Verification Commands

After configuring LVM, always verify with these commands:

  • pvs - Shows physical volumes
  • vgs - Shows volume groups
  • lvs - Shows logical volumes
  • mount - Shows mounted filesystems
