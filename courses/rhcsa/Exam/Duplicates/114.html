Welcome to the RHCSA User and Group Management Lab. In this lab, you will create users and groups with specific membership requirements.

Follow the steps in order. You must complete each step before moving to the next one.

1 Create the Group Current Step

First, let's create the group that will be used as a secondary group for some users.

Type this command exactly:
groupadd sharegrp

This creates a new group named "sharegrp".

Understanding Linux Groups

Groups in Linux are used to organize users and manage permissions. Each user has a primary group and can belong to multiple secondary groups.

2 Create User Harry Pending

Now let's create the user Harry and add them to the sharegrp group.

Type this command exactly:
useradd harry

This creates the user Harry with default settings.

3 Add Harry to sharegrp Group Pending

Now we need to add Harry to the sharegrp group as a secondary group.

Type this command exactly:
usermod -aG sharegrp harry

The -aG option adds the user to the specified group without removing them from other groups.

4 Create User Natasha Pending

Now let's create the user Natasha.

Type this command exactly:
useradd natasha

5 Add Natasha to sharegrp Group Pending

Add Natasha to the sharegrp group as a secondary group.

Type this command exactly:
usermod -aG sharegrp natasha

6 Create User Copper with Non-Interactive Shell Pending

Now create user Copper with a non-interactive shell. Copper should not be a member of sharegrp.

Type this command exactly:
useradd -s /sbin/nologin copper

The -s /sbin/nologin option sets the login shell to prevent interactive login.

Non-Interactive Shells

Setting a user's shell to /sbin/nologin or /bin/false prevents them from logging in interactively while still allowing other system functions.

7 Set Passwords for All Users Pending

Now set the passwords for all three users to "redhat".

Type these commands one by one:
echo "redhat" | passwd --stdin harry
echo "redhat" | passwd --stdin natasha
echo "redhat" | passwd --stdin copper

This sets the password for each user to "redhat" without interactive prompts.

8 Verify Configuration Pending

Finally, let's verify that all configurations are correct.

Type these commands one by one:
id harry

Should show harry is in sharegrp group

id natasha

Should show natasha is in sharegrp group

id copper

Should show copper is NOT in sharegrp group

grep copper /etc/passwd

Should show copper has /sbin/nologin shell

Verification Commands

After creating users and groups, always verify with these commands:

  • id <username> - Shows user's groups and IDs
  • groups <username> - Shows user's group memberships
  • grep <username> /etc/passwd - Shows user configuration
  • getent group <groupname> - Shows group members
