Step 1: Check Current Configuration

First, check the current size of your Logical Volume and filesystem:

Check mounted filesystems:

Type: df -hT /home

Check Logical Volume details:

Type: lvdisplay /dev/vg0/vo

Understanding the Output

The df -hT command shows mounted filesystems with their types and sizes. The lvdisplay command shows detailed information about the Logical Volume including its current size and path.

Step 2: Resize the Logical Volume

Depending on whether you need to extend or reduce the LV, use the appropriate commands:

For extending the LV:

Type: lvextend -L 290M /dev/vg0/vo

Or to add space: lvextend -L +100M /dev/vg0/vo

For reducing the LV (if needed):

First unmount: umount /home

Check filesystem: e2fsck -f /dev/vg0/vo

Resize filesystem: resize2fs /dev/vg0/vo 290M

Reduce LV: lvreduce -L 290M /dev/vg0/vo

Remount: mount /dev/vg0/vo /home

Resizing Considerations

When extending, you can specify either the exact size (-L 290M) or add to current size (-L +100M). When reducing, you must first resize the filesystem to be smaller than the new LV size.

Step 3: Resize the Filesystem

After resizing the LV, you must resize the filesystem to match:

For ext2/ext3/ext4 filesystems:

Type: resize2fs /dev/vg0/vo

For XFS filesystems:

Type: xfs_growfs /home

Filesystem Differences

ext4 filesystems can be grown or reduced (with unmounting). XFS filesystems can only be grown, not reduced. Always check your filesystem type with df -T before resizing.

Step 4: Verify the Changes

Confirm the new size is within the acceptable range (270M-320M):

Check the new size:

Type: df -h /home

Type: lvdisplay /dev/vg0/vo

Verification

The df -h output should show the filesystem size in the required range. The lvdisplay output should show the exact LV size. Remember that filesystem overhead means these numbers won't match exactly.
