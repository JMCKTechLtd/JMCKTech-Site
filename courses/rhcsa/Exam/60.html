LVM Concepts

Physical Volume (PV) Standard partition added to LVM. Can be a primary, logical partition, or RAID array.
Volume Group (VG) Composed of Logical Volumes. Organizational group for LVM.
Logical Volume (LV) Composed of Logical Extents (LEs). Filesystems are created on LVs.
Extents PE (Physical Extent) - Chunk of disk space on PV
LE (Logical Extent) - Chunk of disk space mapped to PE

Step 1: Verify Initial LV Size

Check current LV size:

Type: lvdisplay /dev/vg0/lv1

Should show current size of 100MB

Understanding LV Display

The lvdisplay command shows detailed information about a logical volume including:

  • LV Path (e.g., /dev/vg0/lv1)
  • LV Name
  • VG Name
  • LV Size
  • Current LE (Logical Extents) count

Step 2: Extend the Logical Volume

Extend LV to 500MB:

Type: lvextend -L+400M /dev/vg0/lv1

Or alternatively: lvextend -L500M /dev/vg0/lv1

Understanding LV Extension

The lvextend command increases the size of a logical volume:

  • -L+400M - Adds 400MB to current size
  • -L500M - Sets absolute size to 500MB
  • Requires free space in the volume group
  • Does not affect filesystem size (need separate command)

Step 3: Resize Filesystem Online

Resize filesystem without unmounting:

Type: ext2online -d /dev/vg0/lv1

Online Filesystem Resizing

The ext2online command resizes ext2/ext3/ext4 filesystems while mounted:

  • -d - Debug output
  • Works on mounted filesystems
  • Alternative: resize2fs for unmounted filesystems

Step 4: Verify Final Configuration

Check final LV size:

Type: lvdisplay /dev/vg0/lv1

Should now show size of 500MB

Check filesystem size:

Type: df -h /dev/vg0/lv1

Should reflect the new size

Verification Commands

After resizing, verify with:

  • lvdisplay - Shows LV information
  • df -h - Shows filesystem usage
  • lsblk - Shows block device information
