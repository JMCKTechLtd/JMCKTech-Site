0 Lab Introduction Completed

In this lab you will create a compressed backup of /usr/local and verify SELinux is running in enforcing mode, as required in the RHCSA exam objective: Archiving and Transferring Files & SELinux.

Follow the steps in order. The terminal on the right simulates the commands needed to complete the task.

1 Create /root/backup.tar.bz2 from /usr/local Current Step

Use tar with bzip2 compression to create the backup file /root/backup.tar.bz2 containing the content of /usr/local.

Type this command exactly:
tar -cvjf /root/backup.tar.bz2 /usr/local

tar + bzip2

The option -j enables bzip2 compression. The full option set is: -c (create), -v (verbose), -j (bzip2), -f (archive file name).

2 Verify the archive file exists Pending

Confirm that /root/backup.tar.bz2 has been created.

ls /root/backup.tar.bz2

3 Check the archive file type Pending

Use the file command to confirm that the backup is recognised as a bzip2-compressed tar archive.

file /root/backup.tar.bz2

4 Check current SELinux mode Pending

Check the current SELinux mode. If the output is anything other than Enforcing, you must change it.

getenforce

SELinux runtime mode

The getenforce command returns one of: Enforcing, Permissive, or Disabled.

5 Set SELinux to enforcing mode Pending

If SELinux is not already enforcing, change it to enforcing mode at runtime:

setenforce 1

In a real system you would also update /etc/selinux/config to make this persistent across reboots, but this lab focuses on the runtime change required by the exam question.
