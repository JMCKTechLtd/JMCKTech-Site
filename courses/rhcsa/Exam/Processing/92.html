0 Lab Introduction Completed

Welcome to the RHCSA Web Server Configuration Lab. In this lab, you will install and configure an Apache web server with a custom web page.

Follow the steps in order. You must complete each step before moving to the next one.

1 Install Apache Web Server Current Step

First, let's install the Apache HTTP server package.

Type this command exactly:
yum install -y httpd

This will download and install the httpd package along with any dependencies.

Understanding Apache Web Server

Apache HTTP Server (httpd) is the most widely used web server software. It serves web content over HTTP and HTTPS protocols.

2 Navigate to Web Directory Pending

Now let's navigate to the web server's document root directory.

Type this command exactly:
cd /var/www/html

This is the default directory where Apache serves web content from.

3 Download Web Page Pending

Now let's download the web page from the instructor's server.

Type this command exactly:
wget http://instructor.example.com/pub/serverX.html

This will download the serverX.html file to the current directory.

About wget

wget is a command-line utility for downloading files from the web. It supports HTTP, HTTPS, and FTP protocols.

4 Configure as Index Page Pending

Now let's rename the downloaded file to index.html so it serves as the default page.

Type this command exactly:
mv serverX.html index.html

This renames the file so Apache will serve it as the default page when someone visits your web server.

5 Start and Enable Web Service Pending

Now let's start the Apache web service and enable it to start automatically at boot.

Type these commands one by one:
systemctl start httpd
systemctl enable httpd

This starts the service immediately and ensures it will start automatically after system reboot.

6 Verify Configuration Pending

Finally, let's verify that our web server is running and configured correctly.

Type these commands one by one:
systemctl status httpd

Should show: active (running)

systemctl is-enabled httpd

Should show: enabled

ls -l /var/www/html/

Should show: index.html

Verification Commands

After configuring services, always verify with these commands:

  • systemctl status - Shows service status
  • systemctl is-enabled - Checks if service starts at boot
  • ls -l - Verifies files are in place
