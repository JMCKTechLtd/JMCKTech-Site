0 Lab Introduction Completed

Welcome to the RHCSA Volume Group Configuration Lab. In this lab, you will create a volume group with a specific physical extent size, create a logical volume, format it, and mount it persistently.

Follow the steps in order. You must complete each step before moving to the next one.

1 Create a New Partition Current Step

First, we need to create a new partition for our LVM setup. We'll use fdisk to create the partition.

Type this command exactly:
fdisk /dev/vda

Once in fdisk, follow these steps:

  1. Type n to create a new partition
  2. Select partition type (primary or extended)
  3. Accept default first sector
  4. Specify size (e.g., +4G for 4GB)
  5. Type t to change partition type
  6. Select the partition number
  7. Type 8e for Linux LVM
  8. Type w to write changes and exit

Understanding LVM Partitions

LVM (Logical Volume Manager) allows flexible disk management. For LVM, partitions must have the type "Linux LVM" (8e).

2 Update Partition Table Pending

After creating the partition, we need to update the kernel's partition table.

Type this command exactly:
partprobe

This command informs the operating system about partition table changes without requiring a reboot.

3 Create Physical Volume Pending

Now we'll create a physical volume from our new partition.

Type this command exactly:
pvcreate /dev/vda6

Replace /dev/vda6 with the actual partition you created in step 1.

About Physical Volumes

A physical volume (PV) is a storage device or partition that has been initialized for use by LVM. The pvcreate command prepares the partition for LVM.

4 Create Volume Group with 8MB PE Size Pending

Now we'll create a volume group with a physical extent size of 8MB.

Type this command exactly:
vgcreate -s 8M vg1 /dev/vda6

This creates a volume group named vg1 with 8MB physical extents using the physical volume we created.

Understanding Physical Extents

Physical extents (PE) are the smallest allocatable units in a volume group. The -s 8M option sets the PE size to 8MB. Smaller PE sizes allow for more precise allocation of space.

5 Create Logical Volume Pending

Now we'll create a logical volume with 50 extents named lvshare.

Type this command exactly:
lvcreate -n lvshare -l 50 vg1

This creates a logical volume named lvshare with 50 extents (400MB total, since each extent is 8MB) in volume group vg1.

About Logical Volumes

Logical volumes (LV) are the usable "partitions" created from volume groups. They can be resized, moved, and have snapshots taken of them.

6 Format the Logical Volume Pending

Now we'll format the logical volume with the ext4 filesystem.

Type this command exactly:
mkfs.ext4 /dev/vg1/lvshare

This creates an ext4 filesystem on our logical volume, making it ready for use.

7 Create Mount Point Pending

We need to create a directory to mount our filesystem.

Type this command exactly:
mkdir -p /mnt/data

This creates the directory /mnt/data where we'll mount our logical volume.

8 Configure Persistent Mount Pending

Now we'll configure the system to automatically mount our logical volume at boot.

Type this command exactly:
vim /etc/fstab

Add this line to the file:

/dev/vg1/lvshare /mnt/data ext4 defaults 0 0

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

9 Mount the Filesystem Pending

Now let's mount the filesystem to test our configuration.

Type this command exactly:
mount -a

This command mounts all filesystems defined in /etc/fstab.

10 Verify Configuration Pending

Finally, let's verify that all our configurations are working correctly.

Type these commands one by one:
pvs

Should show your physical volume

vgs

Should show volume group vg1 with 8M PE size

lvs

Should show logical volume lvshare with 50 extents

df -h

Should show /dev/mapper/vg1-lvshare mounted at /mnt/data

Verification Commands

After LVM configuration, always verify with these commands:

  • pvs - Shows physical volumes
  • vgs - Shows volume groups
  • lvs - Shows logical volumes
  • df -h - Shows mounted filesystems
