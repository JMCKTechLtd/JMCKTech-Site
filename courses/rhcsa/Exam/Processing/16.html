Welcome to the RHCSA Kernel Upgrade Lab. In this lab, you will upgrade the kernel while ensuring the old kernel remains available as a boot option.

Follow the steps in order. You must complete each step before moving to the next one.

1 Check Current Kernel Version Current Step

First, let's check the current kernel version to establish a baseline.

Type this command exactly:
uname -r

Understanding Kernel Versions

The kernel is the core component of the Linux operating system. It manages system resources and facilitates communication between hardware and software.

Kernel version format: major.minor.patch-build

2 Download New Kernel Package Pending

Now let's download the new kernel package from the FTP server.

Type this command exactly:
cd /boot
wget ftp://server.domain11.example.com/pub/updates/kernel-2.6.36.7.1.rpm

Note: The exact URL may vary in your exam environment. Follow the instructions provided.

3 Install New Kernel Package Pending

Now let's install the new kernel package using RPM.

Type this command exactly:
rpm -ivh kernel-2.6.36.7.1.rpm

RPM Package Management

RPM (Red Hat Package Manager) is used to install, update, and remove software packages.

Common RPM options:

  • -i: Install package
  • -v: Verbose output
  • -h: Show progress with hash marks

4 Configure GRUB Bootloader Pending

Now we need to configure GRUB to use the new kernel as the default.

Type this command exactly:
vim /etc/grub.conf

Once in the editor, ensure the default parameter is set to 0:

default=0 timeout=5 splashimage=(hd0,0)/grub/splash.xpm.gz hiddenmenu title Red Hat Enterprise Linux Server (2.6.36.7.1) root (hd0,0) kernel /vmlinuz-2.6.36.7.1 ro root=/dev/mapper/vol0-root rd_NO_LUKS rd_NO_MD initrd /initramfs-2.6.36.7.1.img title Red Hat Enterprise Linux Server (Previous Kernel) root (hd0,0) kernel /vmlinuz-2.6.32-71.el6.x86_64 ro root=/dev/mapper/vol0-root rd_NO_LUKS rd_NO_MD initrd /initramfs-2.6.32-71.el6.x86_64.img

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Understanding GRUB Configuration

GRUB (GRand Unified Bootloader) is responsible for loading the operating system.

Key parameters in grub.conf:

  • default: Index of the default boot entry (0-based)
  • timeout: Time in seconds to wait for user input
  • title: Label for each boot entry

5 Verify Configuration Pending

Finally, let's verify that our configuration is correct.

Type these commands one by one:
ls /boot/vmlinuz*

Should show both old and new kernel files

cat /etc/grub.conf | grep default

Should show: default=0

cat /etc/grub.conf | grep title

Should show both kernel entries

Verification Commands

After kernel upgrade, always verify with these commands:

  • ls /boot/vmlinuz* - Lists all available kernels
  • cat /etc/grub.conf - Shows GRUB configuration
  • rpm -q kernel - Shows installed kernel packages
