0 Lab Introduction Completed

Welcome to the RHCSA File Permissions with ACLs Lab. In this lab, you will configure advanced file permissions using Access Control Lists (ACLs).

Follow the steps in order. You must complete each step before moving to the next one.

1 Copy fstab File Current Step

First, let's copy the /etc/fstab file to /var/tmp/fstab.

Type this command exactly:
cp /etc/fstab /var/tmp/

This command copies the fstab file to the target location.

Understanding File Copy

  • cp: Copies files and directories
  • -p: Preserves permissions and timestamps
  • -a: Archive mode (preserves all attributes)
  • Without options, the copy gets new permissions and timestamps

2 Set Owner and Group Pending

Now let's ensure the file is owned by root user and root group.

Type this command exactly:
chown root:root /var/tmp/fstab

This command sets both the owner and group to root.

File Ownership Commands

  • chown user:group file: Changes owner and group
  • chown user file: Changes only owner
  • chgrp group file: Changes only group
  • Only root can change file ownership

3 Remove Execute Permissions Pending

Now let's remove execute permissions for all users.

Type this command exactly:
chmod a-x /var/tmp/fstab

This command removes execute permissions for user, group, and others.

Alternative method: You can also use chmod 664 /var/tmp/fstab

4 Set Natasha's ACL Permissions Pending

Now let's set Access Control List permissions for user natasha.

Type this command exactly:
setfacl -m u:natasha:rw- /var/tmp/fstab

This gives natasha read and write permissions via ACL.

Understanding setfacl Syntax

  • -m: Modify ACL entries
  • u:natasha:rw-: User natasha gets read+write
  • g:groupname:perms: For group permissions
  • o:perms: For other users
  • m:perms: For mask (maximum effective permissions)

5 Set Harry's ACL Permissions Pending

Now let's set Access Control List permissions for user harry.

Type this command exactly:
setfacl -m u:harry:--- /var/tmp/fstab

This explicitly denies harry any access to the file.

ACL Permission Denial

When using ACLs:

  • ACL entries are evaluated in order
  • Explicit deny takes precedence
  • If no ACL entry matches, traditional permissions apply
  • Use --- to explicitly deny all permissions

6 Verify ACL Configuration Pending

Finally, let's verify that our ACL configuration is correct.

Type these commands one by one:
ls -l /var/tmp/fstab

Should show -rw-rw-r--+ indicating ACLs are present

getfacl /var/tmp/fstab

Should show detailed ACL entries for natasha and harry

ACL Verification Commands

After setting ACLs, always verify with these commands:

  • ls -l - The + sign indicates ACLs
  • getfacl file - Shows all ACL entries
  • setfacl -x entry file - Removes specific ACL entry
  • setfacl -b file - Removes all ACL entries
