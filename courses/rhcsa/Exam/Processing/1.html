Welcome to the RHCSA Network Configuration Lab. In this lab, you will configure basic network settings on a Red Hat Enterprise Linux system.

Follow the steps in order. You must complete each step before moving to the next one.

1 Configure Persistent Hostname Current Step

First, let's set the persistent hostname by editing the network configuration file.

Type this command exactly:
vim /etc/sysconfig/network

Once in the editor, add this line to the file:

HOSTNAME=station.domain40.example.com

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Understanding Hostname Configuration

The hostname identifies your system on the network. For the changes to persist after reboot, you must edit the configuration file.

2 Set Temporary Hostname Pending

Now let's set the hostname temporarily using the hostname command.

Type this command exactly:
hostname station.domain40.example.com

Note: This change will not persist after reboot, but we need it for the current session.

3 Navigate to Network Scripts Directory Pending

Now we need to configure the network interface. First, navigate to the network scripts directory.

Type this command exactly:
cd /etc/sysconfig/network-scripts/

4 Edit Network Interface Configuration Pending

Now let's edit the network interface configuration file.

Type this command exactly:
vim /etc/sysconfig/network-scripts/ifcfg-eth0

Modify the file to contain these exact settings:

DEVICE=eth0 BOOTPROTO=none ONBOOT=yes IPADDR=172.24.40.40 NETMASK=255.255.255.0 GATEWAY=172.24.40.1 DNS1=172.24.40.1

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Understanding Network Interface Configuration

The interface configuration file controls how your network interface behaves. Key parameters:

  • IPADDR: Sets the static IP address
  • GATEWAY: Defines the default gateway
  • DNS1: Primary DNS server
  • ONBOOT=yes: Brings interface up at boot

5 Restart Network Service Pending

Now let's apply the changes by restarting the network service.

Type this command exactly:
systemctl restart network

6 Verify Configuration Pending

Finally, let's verify that all our configurations are working correctly.

Type these commands one by one:
hostname

Should show: station.domain40.example.com

ip addr show eth0

Should show IP address: 172.24.40.40

ip route show

Should show gateway: 172.24.40.1

cat /etc/resolv.conf

Should show DNS: 172.24.40.1

Verification Commands

After making changes, always verify with these commands:

  • hostname - Shows current hostname
  • ip addr - Displays IP addresses
  • ip route - Shows routing table including gateway
  • cat /etc/resolv.conf - Shows DNS configuration
