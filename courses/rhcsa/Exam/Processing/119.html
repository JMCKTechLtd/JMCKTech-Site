Welcome to the RHCSA User Account Creation Lab. In this lab, you will create a user account with a specific UID and set its password.

Follow the steps in order. You must complete each step before moving to the next one.

1 Create User with Specific UID Current Step

First, let's create the user account with the specified User ID.

Type this command exactly:
useradd -u 3945 feed

This command:

  • Creates a user named feed
  • Sets the User ID to 3945 using the -u option
  • Creates a home directory at /home/feed

Understanding User IDs (UIDs)

User IDs are numerical identifiers for users in Linux:

  • 0: Root user (superuser)
  • 1-999: System users and services
  • 1000+: Regular users

Using specific UIDs can be important for:

  • Consistency across multiple systems
  • File ownership and permissions
  • Application requirements

2 Set User Password Pending

Now let's set the password for the new user using the echo command with password redirection.

Type this command exactly:
echo "iamredhatman" | passwd --stdin feed

This command:

  • Uses echo to output the password
  • Pipes (|) the password to the passwd command
  • Uses --stdin to read the password from standard input
  • Sets the password for user feed

Password Management

There are several ways to set user passwords in Linux:

  • passwd username - Interactive password setting
  • echo "password" | passwd --stdin username - Non-interactive
  • chpasswd - Batch password changes

Security Note: The --stdin method may leave the password visible in shell history. Use with caution in production environments.

3 Verify User Creation Pending

Let's verify that the user was created correctly by checking the system files.

Type this command exactly:
grep feed /etc/passwd

This should display the user entry with the correct UID.

You can also check the user's information:
id feed

This will show the user's UID, GID, and group memberships.

4 Verify UID and Home Directory Pending

Let's verify that the UID is correctly set and the home directory exists.

Type these commands to verify:
getent passwd feed

Shows detailed user information including UID.

ls -ld /home/feed

Verifies the home directory exists and shows its permissions.

su - feed -c "whoami"

Tests if you can switch to the new user (optional verification).

User Verification Commands

Useful commands for verifying user accounts:

  • id username - Shows UID, GID, and groups
  • getent passwd username - Shows user database entry
  • finger username - Shows user information (if installed)
  • ls -ld /home/username - Checks home directory
