In the previous task, you configured a rootless container logserver managed by user systemd. This lab extends that configuration so that the container writes logs to a persistent directory under /home/shangrila, populated with host journald logs.

Follow the steps in order. The terminal on the right simulates the commands needed to complete the task.

1 Enable persistent journald storage Current Step

Ensure that the host journal is stored persistently across reboots by creating /var/log/journal and restarting systemd-journald.

Run these commands in sequence:
mkdir -p /var/log/journal
systemctl restart systemd-journald

Persistent journald

When /var/log/journal exists and storage is not forced to volatile, journald writes logs there, keeping them across reboot.

2 Create the container log directory Pending

Create the directory that will hold journald files and be mounted into the container as /var/log.

mkdir -p /home/shangrila/container-logserver

3 Copy host journal files into the log directory Pending

Copy all current journal files into the new directory.

Type this command exactly:
cp -a /var/log/journal/* /home/shangrila/container-logserver/

In a real system you might need sudo to read /var/log/journal. This lab simulates the copy.

4 Start logserver with the persistent log mount Pending

Start (or re-create) the logserver container so that it mounts the host directory as /var/log.

podman run -d --name logserver -v /home/shangrila/container-logserver:/var/log registry.domain15.example.com:5000/rhel8/rsyslog

Bind mounts for container logs

The -v option binds a host directory into the container, making container logs persistent on the host filesystem.

5 Verify the mount inside logserver Pending

Confirm that the container has the host directory mounted as /var/log by inspecting the container.

podman inspect logserver --format "{{ .Mounts }}"

RHCSA-style verification

Inspecting the container ensures that the required volume mount is present and correctly mapped.
