Welcome to the RHCSA LVM Creation Lab. In this lab, you will create and configure LVM components including physical volumes, volume groups, and logical volumes.

Follow the steps in order. You must complete each step before moving to the next one.

1 Create Physical Volumes Current Step

Initialize disk partitions as physical volumes for LVM:

Required Command:
pvcreate /dev/sda7 /dev/sda8

About Physical Volumes

Physical volumes are the building blocks of LVM. They can be:

Whole disks /dev/sdb
Disk partitions /dev/sda7
RAID devices RAID arrays

Verify with pvdisplay or pvs.

2 Create Volume Group Pending

Create a volume group with 16M extends:

Required Command:
vgcreate -s 16M vg1 /dev/sda7 /dev/sda8

Volume Group Extends

Extends are the smallest allocatable units in a VG. Key points:

  • Default size is 4MB (can be 1K-16G)
  • Larger extends reduce management overhead
  • All LVs in a VG share the same extend size

Verify with vgdisplay vg1.

3 Create Logical Volume Pending

Create a logical volume with 50 extends (800M total):

Required Command:
lvcreate -l 50 -n lvm02 vg1

Logical Volume Creation

Key options for lvcreate:

-l Size in extends (50 extends √ó 16M = 800M)
-L Absolute size (e.g., -L 800M)
-n Name for the LV

Verify with lvdisplay /dev/vg1/lvm02.

4 Create Filesystem Pending

Format the logical volume as ext4:

Required Command:
mkfs.ext4 /dev/vg1/lvm02

Filesystem Creation

The mkfs command creates a filesystem:

mkfs.ext4 For ext4 filesystem
-L option Add filesystem label
blkid Verify with blkid /dev/vg1/lvm02

5 Configure Mounting Pending

Set up automatic mounting at /mnt/data:

Required Commands:
mkdir -p /mnt/data

Get the UUID of the filesystem:

blkid /dev/vg1/lvm02

Copy the UUID for use in fstab.

Edit fstab file:
vim /etc/fstab

Add this line (replace UUID with actual value):

UUID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx /mnt/data ext4 defaults 0 0

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Finally, mount the filesystem:

mount -a

Persistent Mounting

Key points about /etc/fstab:

  • UUID is preferred over device names (more reliable)
  • Test with mount -a before rebooting
  • Verify with mount | grep /mnt/data
