Welcome to the RHCSA LVM Expansion Lab. In this lab, you will extend a logical volume and resize its filesystem.

Follow the steps in order. You must complete each step before moving to the next one.

1 Check Available Space Current Step

Before extending the LV, verify you have enough space in the volume group:

Type this command exactly:
vgdisplay

Look for "Free PE / Size" to ensure enough space exists.

Understanding vgdisplay Output

Key fields to check:

  • VG Name: Name of your volume group
  • VG Size: Total size of the volume group
  • Free PE / Size: Available space for extension

If space is insufficient, you'll need to add physical volumes with pvcreate and extend the VG with vgextend.

2 Extend the Logical Volume Pending

Increase the size of logical volume vo from 190M to 300M:

Type this command exactly:
lvextend -L 300M /dev/vg0/vo

Or alternatively: lvextend -L +110M /dev/vg0/vo

LVM Extension Options

The lvextend command has two syntax options:

  • -L 300M: Sets absolute size to 300M
  • -L +110M: Adds 110M to current size

The filesystem remains unchanged after LV extension - you must resize it separately.

3 Resize the Filesystem Pending

After extending the LV, resize the filesystem to use the new space:

Type this command exactly:
resize2fs /dev/vg0/vo

Filesystem Resizing Notes

resize2fs automatically detects the new LV size and expands to fill it.

For unmounted filesystems, use resize2fs -f to force the operation.

Always verify with df -h after resizing.

4 Verify the Changes Pending

Confirm the LV is now between 280M-320M and mounted correctly:

Type these commands one by one:
lvdisplay /dev/vg0/vo

Look for "LV Size" in the output.

df -h /mnt/vo

Verify the filesystem shows the new size.

Verification Commands

Essential verification commands:

  • lvdisplay - Shows LV details including size
  • df -h - Shows mounted filesystem sizes
  • mount - Verifies filesystem is mounted

LVM Expansion Concepts

Volume Groups (VG) Storage pool created from physical volumes (PVs). Check free space with vgdisplay.
Logical Volumes (LV) Can be resized dynamically. Use lvextend to grow and lvreduce to shrink.
Filesystem Resizing After LV resize, filesystem must be adjusted with resize2fs for ext4.
Verification Always verify changes with lvdisplay and df -h.
