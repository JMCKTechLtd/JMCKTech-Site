Swap Partition Concepts

Purpose Swap space acts as virtual memory when physical RAM is full. It's essential for system stability.
Partitioning Use fdisk to create partitions. Swap partitions should be type 82 (Linux swap).
Activation mkswap formats, swapon activates. Use swapon -s to verify.
Persistence Add to /etc/fstab using UUID for automatic activation at boot.

Step 1: Create Swap Partition

Launch fdisk on your disk (typically sda):

Type: fdisk /dev/sda

Follow these steps in fdisk:

n (create new partition) p (primary partition) 8 (partition number) +2G (size) t (change type) 82 (Linux swap) w (write changes)

View current partition table:

Type: fdisk -l /dev/sda

Should show your new partition (sda8) with type 'Linux swap':

Device Boot Start End Sectors Size Id Type /dev/sda8 2099200 6293503 4194304 2G 82 Linux swap

Understanding fdisk

fdisk is an interactive partition table manipulator. Key commands:

  • n - Create new partition
  • p - Print partition table
  • t - Change partition type
  • w - Write changes to disk
  • q - Quit without saving

Step 2: Update Kernel Partition Table

Inform kernel about new partition:

Type: partx -a /dev/sda

Then: partprobe

Partition Table Updates

After modifying partitions, the kernel needs to be informed:

  • partx - Tells the kernel about partition table changes
  • partprobe - Another way to inform the kernel about partition changes
  • Rebooting also works but isn't necessary with these commands

Step 3: Format and Activate Swap

Format partition as swap:

Type: mkswap /dev/sda8

Activate swap partition:

Type: swapon /dev/sda8

Verify swap is active:

Type: swapon -s or free -h

Should show your new swap partition in the list:

Filename Type Size Used Priority /dev/sda8 partition 2G 0B -2

Step 4: Make Swap Persistent

Get UUID of swap partition:

Type: blkid /dev/sda8

Copy the UUID value (will look like XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX)

Edit /etc/fstab:

Type: vim /etc/fstab

Add this line (replace XXXXX with your UUID):

UUID=XXXXX swap swap defaults 0 0

Verify persistence:

Type: swapoff -a then swapon -a

Check with swapon -s to ensure swap reactivated

Understanding fstab

The file /etc/fstab controls what filesystems are mounted at boot. Key fields:

  1. Device (can use UUID, which is preferred)
  2. Mount point (swap for swap partitions)
  3. Filesystem type (swap)
  4. Mount options (defaults)
  5. Dump flag (0)
  6. Filesystem check order (0 for swap)
