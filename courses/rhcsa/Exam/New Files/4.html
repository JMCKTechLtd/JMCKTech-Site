Welcome to the RHCSA SSH Configuration Lab. In this lab, you will configure SSH server settings on a Red Hat Enterprise Linux system.

Follow the steps in order. You must complete each step before moving to the next one.

1 Backup SSH Configuration Current Step

Before making any changes, create a backup of the SSH configuration file:

Type this command exactly:
cp /etc/ssh/sshd_config /etc/ssh/sshd_config.backup

This creates a backup file that you can restore if needed.

Why Backup Configuration Files?

Always backup configuration files before making changes. This allows you to quickly restore the original configuration if something goes wrong.

2 Edit SSH Configuration Pending

Edit the SSH configuration file to implement the required changes:

Type this command exactly:
vim /etc/ssh/sshd_config

This will open the SSH configuration in vim.

Make the following changes:

  1. Find the line with #Port 22 and change it to Port 2222
  2. Find the line with #PermitRootLogin yes and change it to PermitRootLogin no

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

SSH Configuration Directives

Port

Specifies the port number that sshd listens on

PermitRootLogin

Specifies whether root can log in using SSH

3 Restart SSH Service Pending

Restart the SSH service to apply the configuration changes:

Type this command exactly:
systemctl restart sshd

Then verify the service is running:

systemctl status sshd

SSH Service Management

Common SSH service commands:

  • systemctl start sshd - Start SSH service
  • systemctl stop sshd - Stop SSH service
  • systemctl restart sshd - Restart SSH service
  • systemctl status sshd - Check SSH service status
