Autofs Configuration Concepts

Automatic Mounting Autofs automatically mounts filesystems when accessed and unmounts them after a period of inactivity.
Configuration Files /etc/auto.master - Main configuration file
/etc/auto.misc - Map file for specific mounts
Mount Options rw - Read-write access
soft - Soft mount (allows timeout)
intr - Allows interruption
Service Management systemctl restart autofs - Restarts service
systemctl enable autofs - Enables at boot

Step 1: Configure auto.master

Edit the master configuration file for autofs:

Edit /etc/auto.master:

Type: vim /etc/auto.master

Add or modify the following line:

/mnt /etc/auto.misc --timeout=50

Understanding auto.master

The auto.master file defines the base mount point and associated map file. The --timeout=50 option specifies that unmount should occur after 50 seconds of inactivity.

Step 2: Configure auto.misc

Edit the map file to define the specific mount:

Edit /etc/auto.misc:

Type: vim /etc/auto.misc

Add the following line:

data -rw,soft,intr server1.example.com:/data

Understanding auto.misc

The map file defines the specific mount points under the base directory. In this case:

  • data - Will create /mnt/data mount point
  • -rw,soft,intr - Mount options (read-write, soft mount, interruptible)
  • server1.example.com:/data - NFS share to mount

Step 3: Restart and Enable Service

Restart the autofs service:

Type: systemctl restart autofs

Enable autofs to start at boot:

Type: systemctl enable autofs

Service Management

Changes to autofs configuration require a service restart. Enabling the service ensures mounts will be available after reboot.

Step 4: Test the Configuration

Access the mount point:

Type: ls /mnt/data

Verify mount status:

Type: mount | grep /mnt/data

Testing Autofs

The directory should automatically mount when accessed and unmount after 50 seconds of inactivity. Verify with:

  • ls /mnt/data - Triggers mount
  • mount | grep /mnt/data - Shows mount status
  • Wait 50+ seconds and check again - Should auto-unmount
