NIS Client Concepts

NIS (Network Information Service) A client-server directory service protocol for distributing system configuration data.
authconfig Command-line tool to configure system authentication (NIS, LDAP, Kerberos, etc.).
Automount Automatically mounts directories when accessed, unmounts when idle.
auto.master Main configuration file for automount that maps mount points to map files.

Step 1: Configure NIS Client

Join the RHCE domain as a client using authconfig:

Run authconfig:

Type: authconfig --nisserver=192.168.0.254 --nisdomain=RHCE --update

Or use the graphical tool: system-config-authentication

Step 2: Create Mount Directory

Create the local directory where NIS home directories will be mounted:

Create directory (replace x with your station number):

Type: mkdir /rhome/stationx

Step 3: Configure Automount Master

Edit the automount master configuration file:

Edit /etc/auto.master:

Type: vim /etc/auto.master

Add this line at the end (replace x with your station number):

/rhome/stationx /etc/auto.home --timeout=60

Step 4: Configure Automount Map

Create the automount map file for home directories:

Edit /etc/auto.home:

Type: vim /etc/auto.home

Add this content (replace x with your station number):

* -rw,soft,intr 192.168.0.254:/rhome/stationx/&

Step 5: Restart Autofs Service

Apply the automount configuration:

Restart service:

Type: systemctl restart autofs

Step 6: Verify Configuration

Test the NIS client configuration:

Check NIS users:

Type: getent passwd nisuser2001

Should show user details from NIS server

Test automount:

Type: ls /rhome/stationx/nisuser2001

Should show contents of user's home directory

Understanding Automount

The automounter will automatically mount home directories when accessed. The configuration:

  • /etc/auto.master specifies the mount point and map file
  • /etc/auto.home contains the mount details for home directories
  • The asterisk (*) matches any username
  • The ampersand (&) substitutes the matched username
