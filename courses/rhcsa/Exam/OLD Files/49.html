Swap Partition Concepts

Swap Space Extension of physical memory that uses disk space when RAM is full
Partition Types 82: Linux Swap
83: Linux Filesystem
8e: Linux LVM
Commands mkswap: Creates swap filesystem
swapon: Activates swap space
swapoff: Deactivates swap space
Verification free -m: Shows memory and swap usage
swapon --show: Lists active swap spaces

Step 1: Create Swap Partition

Use fdisk to create a 100MB swap partition on /dev/hda:

Start fdisk:

Type: fdisk /dev/hda

Then follow these steps interactively:

Command (m for help): n Partition type: l (for logical) First cylinder: (press Enter for default) Last cylinder or +size: +100M Command (m for help): t Partition number: (select the new partition) Hex code: 82 Command (m for help): w

Understanding fdisk

fdisk is an interactive partition table manipulator. Key commands:

  • n: Create new partition
  • t: Change partition type
  • p: Print partition table
  • w: Write changes and exit
  • q: Quit without saving

Step 2: Prepare Swap Space

After creating the partition, prepare it for use as swap:

Create swap filesystem:

Type: mkswap /dev/hdaX (replace X with your partition number)

Activate swap space:

Type: swapon /dev/hdaX

Verify swap is active:

Type: free -m or swapon --show

Swap Initialization

The mkswap command initializes the partition for use as swap space. swapon makes it immediately available to the system. These changes are temporary unless added to /etc/fstab.

Step 3: Make Swap Persistent

To ensure the swap space is available after reboot:

Edit /etc/fstab:

Type: vim /etc/fstab

Add this line (replace X with your partition number):

/dev/hdaX swap swap defaults 0 0

Verify after reboot:

Type: reboot

After logging back in, verify with free -m

Understanding /etc/fstab

The fstab file defines how disk partitions and other storage devices should be initialized and mounted. Swap entries have these fields:

  1. Device/partition
  2. Mount point (swap)
  3. Filesystem type (swap)
  4. Mount options (defaults)
  5. Dump flag (0)
  6. Filesystem check order (0)
