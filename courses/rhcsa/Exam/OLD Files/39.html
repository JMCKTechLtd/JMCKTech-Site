Backup Creation Methods

tar Command tar [options] [archive-file] [files-to-archive]
Common options:
-c: Create archive
-x: Extract archive
-v: Verbose output
-f: Specify filename
-j: Use bzip2 compression
-z: Use gzip compression
Compression Types bzip2: Higher compression (slower), uses .bz2 extension
gzip: Faster compression (less efficient), uses .gz extension
xz: Very high compression (slowest), uses .xz extension
Verification Always verify backups by:
1. Checking archive integrity (tar -tf)
2. Extracting to test location
3. Comparing with original files
Common Directories /usr/local: Locally installed software
/etc: System configuration files
/home: User home directories
/var: Variable data like logs

Step 1: Create the Backup

Navigate to /usr/local:

Type: cd /usr/local

Create compressed backup:

Type: tar -jcvf /root/backup.tar.bz2 .

This command will:

  • -j: Use bzip2 compression
  • -c: Create new archive
  • -v: Verbose output
  • -f: Specify filename

Understanding tar Options

The order of options matters in tar commands. The -f option must come immediately before the filename. The dot (.) at the end specifies to archive the current directory contents.

Step 2: Verify the Backup

Create test directory:

Type: mkdir /test

Extract backup for verification:

Type: tar -jxvf /root/backup.tar.bz2 -C /test

This command will:

  • -j: Use bzip2 decompression
  • -x: Extract files
  • -v: Verbose output
  • -f: Specify filename
  • -C: Change to directory before extraction

Backup Verification

After extraction, you should compare the contents of /test with the original /usr/local directory to ensure all files were properly backed up. You can use commands like diff -r /usr/local /test or ls -lR /usr/local > orig.txt; ls -lR /test > test.txt; diff orig.txt test.txt
