YUM Repositories Local repositories can be used when internet access is unavailable or for faster package installation.

Step 1: Configure Local YUM Repository

Create a local repository file to install packages from the existing file server:

Navigate to YUM repositories directory:

Type: cd /etc/yum.repos.d/

Create or edit local repository file:

Type: vim local.repo

Ensure the file contains these settings:

[local] name=local.repo baseurl=file:///mnt enabled=1 gpgcheck=0

Understanding YUM Repositories

YUM repositories provide a way to install and manage packages. Key parameters:

  • baseurl: Location of the repository packages
  • enabled: Whether the repository is active
  • gpgcheck: Whether to verify package signatures

Step 2: Create YUM Cache

After configuring the repository, update the package cache:

Create YUM cache:

Type: yum makecache

About YUM Cache

The YUM cache stores repository metadata to speed up package operations. Creating the cache after repository changes ensures YUM has current package information.

Step 3: Install vsftpd Package

Install the FTP server package using YUM:

Install vsftpd:

Type: yum install -y vsftpd

About vsftpd

vsftpd (Very Secure FTP Daemon) is the default FTP server on RHEL. It's known for its security features and performance. After installation, it needs to be configured and started.

Step 4: Configure FTP Server

Edit the vsftpd configuration file to enable anonymous access:

Edit vsftpd configuration:

Type: vim /etc/vsftpd/vsftpd.conf

Ensure the file contains this setting:

anonymous_enable=YES

FTP Server Configuration

The main configuration file for vsftpd controls server behavior. Key settings include:

  • anonymous_enable: Allows anonymous logins
  • local_enable: Permits local users to log in
  • write_enable: Allows file uploads
  • anon_root: Sets the root directory for anonymous users

Step 5: Start and Enable FTP Service

Start the FTP service and configure it to start at boot:

Start vsftpd service:

Type: systemctl start vsftpd

Enable vsftpd at boot:

Type: systemctl enable vsftpd

Verify service status:

Type: systemctl status vsftpd

Service Management

System services are managed with systemctl. Key commands:

  • systemctl start: Starts a service
  • systemctl stop: Stops a service
  • systemctl restart: Restarts a service
  • systemctl enable: Enables at boot
  • systemctl disable: Disables at boot
  • systemctl status: Shows service status
