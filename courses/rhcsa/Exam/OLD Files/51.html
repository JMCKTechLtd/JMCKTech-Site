Logical Volume Reduction Concepts

Order of Operations Always reduce filesystem before reducing LV. Doing it in reverse can cause data loss.
Filesystem Check fsck -f ensures filesystem is clean before resizing. Mandatory for reduction.
Size Flexibility Final size between 200MB-210MB is acceptable. Exact size isn't critical for this task.
Verification Always verify with both lvdisplay (shows LV size) and df -h (shows mounted size).

Step 1: Check Initial Size

Display current Logical Volume information:

Type: lvdisplay /dev/vo/myvol

This will show the current size (should be 400MB initially).

Understanding lvdisplay Output

Key fields to look for:

  • LV Size: Current size of the Logical Volume
  • VG Name: Volume Group containing this LV
  • LV Path: Full device path (/dev/vo/myvol)

Step 2: Prepare Filesystem for Reduction

Check filesystem consistency:

Type: fsck -f /dev/vo/myvol

This forces a check even if the filesystem appears clean.

Why fsck is Required

Filesystem check ensures:

  • No corruption exists that could cause data loss during resize
  • All metadata is consistent before structural changes
  • The resize operation has a clean starting point

Step 3: Reduce Filesystem

Shrink filesystem to 200MB:

Type: resize2fs /dev/vo/myvol 200M

This prepares the filesystem for the smaller LV size.

Filesystem Reduction

Important notes:

  • Must be done before reducing LV
  • 200M means 200 Megabytes (can also use 200MB)
  • Ensure you have enough free space in filesystem

Step 4: Reduce Logical Volume

Shrink Logical Volume to 200MB:

Type: lvreduce -L 200M /dev/vo/myvol

This actually reduces the LV size after filesystem was prepared.

LV Reduction Safety

Safety checks:

  • Never reduce LV smaller than filesystem
  • Confirm operation when prompted (unless using -f)
  • 200-210MB is acceptable range for this task

Step 5: Verify Final Size

Check Logical Volume size:

Type: lvdisplay /dev/vo/myvol

Should show LV Size ‚âà 200.00 MiB

Check mounted size:

Type: df -h

Should show size ‚âà 200M for /dev/mapper/vo-myvol

Verification Methods

Why both commands?

  • lvdisplay shows actual LV size
  • df -h shows mounted filesystem size
  • Both should match after successful operation
