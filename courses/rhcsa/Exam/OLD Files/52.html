Logical Volume Extension Concepts

Online Extension Unlike reduction, extending can be done while mounted. No fsck needed.
Size Specification -L+200M adds 200MB. -L 300M would set total to 300MB.
Alternative Tools lvresize can also resize LVs. resize2fs works for ext2/3/4.
Online Resizing ext2online brings new space online immediately without unmounting.

Step 1: Check Initial Size

Display current Logical Volume information:

Type: lvdisplay /dev/test0/testvolume1

This will show the current size (should be 100MB initially).

Understanding lvdisplay Output

Key fields to look for:

  • LV Size: Current size of the Logical Volume
  • VG Name: Volume Group containing this LV
  • LV Path: Full device path (/dev/test0/testvolume1)
  • VG Size: Total size of Volume Group

Step 2: Extend Logical Volume

Increase LV size by 200MB:

Type: lvextend -L+200M /dev/test0/testvolume1

This adds 200MB to the existing size (100MB ‚Üí 300MB).

Size Specification Options

Different ways to specify size:

  • -L+200M: Add 200MB to current size
  • -L 300M: Set total size to 300MB
  • -l +100%FREE: Use all remaining space in VG

Step 3: Bring New Size Online

Resize filesystem online:

Type: ext2online -d /dev/test0/testvolume1

This makes the additional space available without unmounting.

Online Resizing Methods

Alternative approaches:

  • ext2online: Works for ext2/3/4 filesystems
  • resize2fs: Traditional method (may require unmount)
  • XFS uses xfs_growfs

Step 4: Verify Final Size

Check Logical Volume size:

Type: lvdisplay /dev/test0/testvolume1

Should show LV Size ‚âà 300.00 MiB

Check mounted size:

Type: df -h

Should show size ‚âà 300M for /dev/mapper/test0-testvolume1

Verification Importance

Always verify:

  • LV size matches expectation (300MB)
  • Filesystem shows correct available space
  • Data remains accessible after resize
