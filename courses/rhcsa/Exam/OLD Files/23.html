File Permission Concepts

Basic Permissions chmod - Change file mode bits
chown - Change file owner and group
chgrp - Change group ownership
Access Control Lists getfacl - View ACLs
setfacl - Modify ACLs
Syntax: setfacl -m u:username:perms file
Permission Notation rwx rwx rwx
Owner | Group | Others
r=read, w=write, x=execute
Special Permissions SUID (4) - Run as owner
SGID (2) - Inherit group
Sticky (1) - Restricted deletion

Step 1: Copy File

Copy fstab to /var/tmp:

Type: cp /etc/fstab /var/tmp/

Understanding File Copy

The copied file will inherit the permissions of the target directory. By default, it will be owned by the user who created it (root) with default permissions (rw-r--r--).

Step 2: Change Group Ownership

Change group to admin:

Type: chgrp admin /var/tmp/fstab

Understanding Group Ownership

Changing group ownership allows members of that group to access the file according to the group permissions. The admin group must exist for this command to work.

Step 3: Set ACL Permissions

Grant full access to user1:

Type: setfacl -m u:user1:rwx /var/tmp/fstab

Remove all access for user2:

Type: setfacl -m u:user2:--- /var/tmp/fstab

Understanding ACLs

Access Control Lists provide finer-grained permissions than standard Unix permissions. They allow you to set permissions for specific users regardless of file ownership or group membership.

Step 4: Verify Configuration

Check file permissions:

Type: ls -l /var/tmp/fstab

Should show: -rw-rw-r--+ 1 root admin

Check ACL settings:

Type: getfacl /var/tmp/fstab

Verification Commands

After setting permissions, always verify with these commands:

  • ls -l - Shows basic permissions and ownership
  • getfacl - Shows detailed ACL permissions
  • stat - Shows complete file attributes
