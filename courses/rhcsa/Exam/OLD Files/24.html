Cron Job Configuration Methods

Method 1: As Administrator Use crontab -u username -e to edit another user's crontab. Requires root privileges.
Method 2: As User First switch to the user with su - username, then use crontab -e to edit their own crontab.
Cron Syntax minute hour day month day-of-week command
Example: 23 14 * * * /bin/echo "file" runs at 14:23 daily
Cron Locations User crontabs are stored in /var/spool/cron/username
System crontabs are in /etc/crontab and /etc/cron.d/

Step 1: Configure Cron Job as Administrator

Edit natasha's crontab as root:

Type: crontab -u natasha -e

Add the following line:

23 14 * * * /bin/echo "file"

Understanding Administrator Method

When configuring cron jobs for other users as an administrator:

  • You must have root privileges
  • The -u flag specifies which user's crontab to edit
  • The changes take effect immediately after saving
  • You can verify with crontab -u natasha -l

Step 2: Configure Cron Job as Natasha

Switch to natasha user:

Type: su - natasha

Edit natasha's own crontab:

Type: crontab -e

Add the following line:

23 14 * * * /bin/echo "file"

Understanding User Method

When configuring your own cron jobs:

  • You must be the target user (use su to switch if needed)
  • No need for -u flag as you're editing your own crontab
  • The editor specified in your EDITOR or VISUAL environment variable will be used
  • Verify with crontab -l

Step 3: Verify Configuration

View natasha's cron jobs as root:

Type: crontab -u natasha -l

View cron jobs as natasha:

Type: su - natasha

Then: crontab -l

Cron Verification

To confirm your cron job is properly configured:

  • Use crontab -l to list your own jobs
  • As root, use crontab -u username -l to list another user's jobs
  • Check system logs with journalctl -u crond for errors
  • Remember cron jobs run with the user's environment and permissions
