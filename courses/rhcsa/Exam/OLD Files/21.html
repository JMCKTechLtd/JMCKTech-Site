User Management Concepts

useradd Command Creates a new user with specified UID and shell:
useradd -u UID -s /sbin/nologin username
passwd Command Sets or changes user password:
passwd username
Non-Interactive Shell /sbin/nologin prevents login
Used for system/service accounts
Verification Check user with:
id username
grep username /etc/passwd

Step 1: Add User

Create user1 with UID 601 and non-interactive shell:

Command to create user:

Type: useradd -u 601 -s /sbin/nologin user1

Understanding User Creation

The useradd command creates a new user entry in /etc/passwd. The -u option specifies the user ID (UID), and -s sets the login shell. Using /sbin/nologin prevents interactive login.

Step 2: Set Password

Set the password for user1 to "redhat":

Command to set password:

Type: passwd user1

Then enter "redhat" when prompted

Password Management

The passwd command updates the user's password in /etc/shadow. For scripting, you can use echo "redhat" | passwd --stdin user1, though this is less secure.

Step 3: Verify Configuration

Check that the user was created correctly:

Command to verify user:

Type: id user1

Expected output: uid=601(user1) gid=601(user1) groups=601(user1)

Type: grep user1 /etc/passwd

Expected output: user1:x:601:601::/home/user1:/sbin/nologin

User Verification

After creating a user, always verify:

  • The user exists with correct UID
  • The shell is set correctly
  • The home directory was created (if specified)
