User Management Concepts

User Creation useradd username - Basic user creation
useradd -G groupname username - Add to secondary group
useradd -s /sbin/nologin username - No shell access
Group Management groupadd groupname - Create new group
usermod -aG groupname username - Add user to group
groups username - Show user's groups
Password Management passwd username - Interactive password set
echo "redhat" | passwd --stdin username - Non-interactive
chage -l username - Check password aging
Verification id username - Show user info
getent passwd username - Check user exists
getent group groupname - Check group exists

Step 1: Create the Admin Group

First, create the adminuser group that will be used for natasha and harry:

Create the group:

Type: groupadd adminuser

Verify the group exists:

Type: getent group adminuser

About Linux Groups

Groups are used to organize users and manage permissions. Each user has a primary group (usually same as username) and can belong to multiple secondary groups. The adminuser group will be a secondary group for natasha and harry.

Step 2: Create User Accounts

Create the three required user accounts with their specific attributes:

Create natasha with adminuser group:

Type: useradd -G adminuser natasha

Create harry with adminuser group:

Type: useradd -G adminuser harry

Create sarah with no shell access:

Type: useradd -s /sbin/nologin sarah

User Creation Options

The -G option adds the user to secondary groups. The -s option sets the login shell. /sbin/nologin prevents interactive login while still allowing other system access like FTP or email.

Step 3: Set Passwords

Set the password for all users to "redhat":

For natasha:

Type: echo "redhat" | passwd --stdin natasha

For harry:

Type: echo "redhat" | passwd --stdin harry

For sarah:

Type: echo "redhat" | passwd --stdin sarah

Password Security

While this lab uses a simple password for practice, in production you should use complex passwords. The --stdin option allows setting passwords non-interactively, useful for scripts.

Step 4: Verify Configuration

Check that all users and groups are configured correctly:

Check user information:

Type: id natasha

Type: id harry

Type: id sarah

Check shell assignments:

Type: getent passwd natasha

Type: getent passwd harry

Type: getent passwd sarah
