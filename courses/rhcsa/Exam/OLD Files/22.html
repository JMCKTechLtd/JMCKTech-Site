User Management Concepts

useradd Creates new user accounts. Common options:
-G - Add to supplementary groups
-m - Create home directory
-s - Set login shell
passwd Changes user passwords. Can set password directly:
passwd username password
Or interactively if no password is provided.
Group Management Users can belong to multiple groups:
Primary group (in /etc/passwd)
Supplementary groups (in /etc/group)
Verification Check user details with:
id username
groups username
getent passwd username

Step 1: Create Users with Admin Group

Create user2 with admin group:

Type: useradd -G admin user2

Create user3 with admin group:

Type: useradd -G admin user3

Understanding User Creation

The -G admin option adds the users to the supplementary group 'admin'. If the admin group doesn't exist, it will be created automatically.

Step 2: Set User Passwords

Set password for user2:

Type: passwd user2 redhat

Set password for user3:

Type: passwd user3 redhat

Understanding Password Management

While setting passwords directly in the command is convenient for labs, in production environments you should use interactive mode (passwd username) for better security.

Step 3: Verify Configuration

Check user details:

Type: id user2 (should show admin group)

Type: id user3 (should show admin group)

Check group membership:

Type: groups user2

Type: groups user3

Verification Commands

After creating users, always verify with these commands:

  • id username - Shows UID, GID, and groups
  • groups username - Shows group membership
  • getent passwd username - Shows user account details
  • getent group groupname - Shows group members
