LDAP Authentication Concepts

LDAP Authentication Allows users to authenticate against a central directory service rather than local /etc/passwd
Required Packages sssd - System Security Services Daemon
authconfig-gtk - GUI configuration tool
pam-krb5 - PAM modules for Kerberos
krb5-workstation - Kerberos utilities
Configuration Files /etc/openldap/ldap.conf - LDAP client config
/etc/sssd/sssd.conf - SSSD configuration
/etc/nsswitch.conf - Name service switch
Verification getent passwd ldapuser40 - Check user resolution
id ldapuser40 - Verify user information
systemctl status sssd - Check SSSD service

Step 1: Install Required Packages

Install authentication packages:

Type: yum install -y sssd authconfig-gtk pam-krb5 krb5-workstation

Package Purpose

These packages provide the necessary components for LDAP authentication:

  • sssd: Caches credentials and provides identity/authentication services
  • authconfig-gtk: Graphical tool for authentication configuration
  • pam-krb5: PAM modules for Kerberos authentication
  • krb5-workstation: Kerberos client utilities

Step 2: Download and Install CA Certificate

Download the certificate:

Type: wget http://instructor.example.com/pub/ldap.crt -O /etc/openldap/certs/ldap.crt

Update CA trust store:

Type: update-ca-trust

Certificate Importance

The CA certificate ensures secure, encrypted communication with the LDAP server. Without it, authentication would fail or be insecure.

Step 3: Configure LDAP Authentication

Method 1: Graphical Configuration

Type: system-config-authentication

Authentication Configuration
‚òë User Account Database: LDAP
‚òë LDAP Search Base DN: dc=example,dc=com
‚òë LDAP Server: ldap://instructor.example.com
‚òë Download CA Certificate
‚òë Authentication Method: LDAP Password

Method 2: Command Line Configuration

Type: authconfig --enableldap --enableldapauth --ldapserver=ldap://instructor.example.com --ldapbasedn="dc=example,dc=com" --enablemkhomedir --update

Configuration Details

Key configuration points:

  • Server must be in domain form (ldap://instructor.example.com), not IP
  • Search Base DN must match your LDAP directory structure
  • Certificate must be properly installed for secure connection
  • SSSD service will be automatically configured

Step 4: Verify Configuration

Check LDAP user resolution:

Type: getent passwd ldapuser40

Expected output: ldapuser40:*:10000:10000:LDAP User:/home/ldapuser40:/bin/bash

Check SSSD service status:

Type: systemctl status sssd

Troubleshooting

If authentication fails:

  • Check /var/log/secure for authentication errors
  • Verify network connectivity to LDAP server
  • Ensure time is synchronized (LDAP often requires accurate time)
  • Check /etc/nsswitch.conf includes sss for passwd and group
