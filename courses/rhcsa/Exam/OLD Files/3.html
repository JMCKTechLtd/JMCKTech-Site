Linux Permissions Concepts

Directory Permissions Read (r): List directory contents
Write (w): Create/delete files
Execute (x): Access files in directory
Special Permissions SUID (4): Execute as owner
SGID (2): New files inherit group
Sticky (1): Only owner can delete
Ownership Commands chown: Change owner/group
chgrp: Change group only
chmod: Change permissions
Numeric Permissions 7: rwx (read/write/execute)
6: rw- (read/write)
5: r-x (read/execute)
4: r-- (read only)

Step 1: Create the Directory

First, create the directory under /home:

Navigate to /home directory:

Type: cd /home/

Create the admins directory:

Type: mkdir admins

Understanding Directory Creation

The mkdir command creates new directories. When created, directories typically inherit the group of their parent directory unless you specify otherwise.

Step 2: Set Group Ownership

Change the group ownership of the directory to 'admin':

Using chown command:

Type: chown .admin admins/

or alternatively:

Type: chgrp admin admins/

Understanding Group Ownership

Group ownership determines which users can access the directory based on group permissions. The chown command can change both owner and group, while chgrp only changes the group.

Step 3: Set Permissions

Configure the permissions to allow group members full access while restricting others:

Set permissions to 770:

Type: chmod 770 admins/

Set the SGID bit:

Type: chmod g+s admins/

or using numeric mode:

Type: chmod 2770 admins/

Understanding Permissions and SGID

The permission mode 770 means:

  • Owner: read, write, execute (7)
  • Group: read, write, execute (7)
  • Others: no permissions (0)

The SGID bit (setgid) ensures that files created in this directory will automatically belong to the admin group, maintaining consistent access control.

Step 4: Verify Configuration

Check directory permissions:

Type: ls -ld /home/admins

Expected output: drwxrws--- 2 root admin 4096 [date] /home/admins

Test file creation:

Type: touch /home/admins/testfile

Type: ls -l /home/admins/testfile

Verify group is 'admin' and permissions are correct

Verification Commands

Key commands to verify your configuration:

  • ls -ld - Shows directory permissions and ownership
  • stat - Displays detailed file/directory information
  • getfacl - Shows Access Control Lists (if set)
