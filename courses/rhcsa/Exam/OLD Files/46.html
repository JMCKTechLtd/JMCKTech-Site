IPTables Concepts

Chains INPUT - Incoming traffic
OUTPUT - Outgoing traffic
FORWARD - Routed traffic
Actions -A - Append rule
-D - Delete rule
-F - Flush rules
Targets ACCEPT - Allow traffic
DROP - Silently block
REJECT - Block with error
Management service iptables save
service iptables restart
Rules lost on reboot if not saved

Step 1: Flush Existing Rules

Clear all existing iptables rules:

Type: iptables -F

Understanding Rule Flushing

Flushing rules (-F) removes all existing firewall rules, giving you a clean slate to work with. This is often the first step when reconfiguring firewall settings.

Step 2: Add Rejection Rule

Block traffic from 172.25.0.0 network:

Type: iptables -A INPUT -s 172.25.0.0/16 -j REJECT

Understanding Network Blocks

The -s 172.25.0.0/16 specifies the entire 172.25.0.0 network (all addresses from 172.25.0.1 to 172.25.255.254). The /16 is the network mask in CIDR notation.

Step 3: Save and Restart

Save the iptables configuration:

Type: service iptables save

Expected output: iptables: Saving firewall rules to /etc/sysconfig/iptables: [ OK ]

Restart the iptables service:

Type: service iptables restart

Expected output: iptables: Setting chains to policy ACCEPT: filter [ OK ]

Understanding Service Management

Saving rules writes them to /etc/sysconfig/iptables so they persist after reboot. Restarting the service applies any changes immediately.
