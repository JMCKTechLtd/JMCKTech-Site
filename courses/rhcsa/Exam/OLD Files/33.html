Step 1: Install chrony

Install the chrony package:

Type: yum -y install chrony

Understanding chrony

chrony is a versatile implementation of the Network Time Protocol (NTP). It:

  • Synchronizes the system clock with NTP servers
  • Works well in changing network conditions
  • Can synchronize clocks faster than traditional NTP implementations
  • Includes tools for monitoring and verification

Step 2: Configure NTP Server

Edit the chrony configuration file:

Type: vim /etc/chrony.conf

Add or modify the server line to:

server classroom.example.com iburst

The iburst option allows for faster initial synchronization.

NTP Server Configuration

Key points about NTP server configuration:

  • Multiple server lines can be added for redundancy
  • iburst sends a burst of packets when the server is unreachable
  • Other options like prefer can mark a preferred server
  • Local servers (like classroom.example.com) are often more accurate than public ones

Step 3: Enable and Start Service

Enable the chronyd service to start at boot:

Type: systemctl enable chronyd

Restart the service to apply changes:

Type: systemctl restart chronyd

Service Management

chronyd runs as a systemd service. Key commands:

  • systemctl start chronyd - Start the service
  • systemctl stop chronyd - Stop the service
  • systemctl restart chronyd - Restart the service
  • systemctl enable chronyd - Enable at boot
  • systemctl status chronyd - Check service status

Step 4: Verify Configuration

Check time synchronization status:

Type: timedatectl status

Look for "System clock synchronized: yes"

Check NTP sources:

Type: chronyc sources

Should show classroom.example.com with a ^* (reachable and selected)

Verification Commands

After configuration, verify with these commands:

  • timedatectl status - Shows if system clock is synchronized
  • chronyc tracking - Shows detailed synchronization info
  • chronyc sources - Lists configured NTP sources
  • chronyc sourcestats - Shows statistics about sources
