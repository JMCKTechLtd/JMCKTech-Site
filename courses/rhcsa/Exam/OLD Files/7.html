Understanding Swap Space

What is Swap? Swap space is used when the system's physical RAM is full. It acts as an overflow area, moving less-used memory pages to disk.
Swap Partition vs Swap File Swap partitions are dedicated disk partitions for swap, while swap files are regular files configured as swap space.
Swap Management swapon - Activates swap space
swapoff - Deactivates swap space
swapon -s - Shows current swap usage
Persistence To make swap persistent across reboots, entries must be added to /etc/fstab using either device path or UUID.

Step 1: Create a New Partition

Use fdisk to create a new 2G partition on /dev/sda:

Start fdisk:

Type: fdisk /dev/sda

This will enter the fdisk interactive mode. Follow these steps:

1. Press p to print the partition table (note available space) 2. Press n to create a new partition 3. Select partition type (primary, extended, or logical) 4. Specify partition size as +2G 5. Press t to change partition type 6. Enter 82 for Linux swap type 7. Press w to write changes and exit

Current Partition Table (Example)

Device Size Type Mount Point
/dev/sda1 1G Linux /boot
/dev/sda2 20G LVM (LVM PV)
/dev/sda3 2G Linux swap [SWAP]
/dev/sda4 10G Extended -
/dev/sda5 5G Linux /home

Note: Your system may have different partitions. The key is to create a new 2G partition without affecting existing ones.

Step 2: Format as Swap Space

After creating the partition, format it as swap space:

Update kernel partition table:

Type: partprobe /dev/sda or partx -a /dev/sda

Format as swap:

Type: mkswap /dev/sdaX (replace X with your partition number)

Understanding mkswap

The mkswap command formats a partition as swap space. It creates a swap signature and prepares the partition for use as virtual memory.

After running mkswap, you'll see output showing the UUID of the new swap space. This UUID can be used in /etc/fstab for persistent mounting.

Step 3: Activate the Swap Space

Enable the new swap partition without affecting existing swap:

Activate swap:

Type: swapon /dev/sdaX (replace X with your partition number)

Verify activation:

Type: swapon -s or free -h

You should see both the original swap and your new 2G swap listed.

Temporary vs Persistent Activation

The swapon command only activates swap temporarily. To make it persistent across reboots, you must add an entry to /etc/fstab.

Using UUIDs in /etc/fstab is recommended as partition numbers may change if disks are reordered.

Step 4: Configure Persistent Mounting

Edit /etc/fstab to enable automatic swap activation at boot:

Get the UUID of your swap partition:

Type: blkid /dev/sdaX (replace X with your partition number)

Look for the UUID in the output (or use the UUID from mkswap output).

Edit /etc/fstab:

Type: vim /etc/fstab

Add a line like this (using your actual UUID):

UUID=xxxx-xxxx-xxxx-xxxx none swap sw 0 0

Save the file and exit.

Understanding /etc/fstab

The /etc/fstab file controls what filesystems are mounted at boot. Each line has six fields:

  1. Device: Can be device path (/dev/sdaX) or UUID (recommended)
  2. Mount point: none for swap
  3. Filesystem type: swap
  4. Options: sw for swap
  5. Dump: 0 (not used for swap)
  6. Fsck order: 0 (no filesystem check for swap)
