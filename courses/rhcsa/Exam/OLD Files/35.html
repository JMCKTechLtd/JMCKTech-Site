User Account Management Concepts

useradd Creates new user accounts with various options:
-u: Specify UID
-g: Primary group
-G: Supplementary groups
passwd Sets or changes user passwords. Root can change any password without knowing the current one.
id Displays user and group information for specified user.
/etc/passwd Contains user account information including UID, GID, home directory, and default shell.

Step 1: Create the User Account

Create user with specific UID:

Type: useradd -u 3400 iar

This creates the user iar with UID 3400.

Understanding User Creation

The useradd command creates a new user account. When specifying a UID, ensure it's not already in use (UIDs below 1000 are typically reserved for system accounts).

Step 2: Set the User Password

Set password for the new user:

Type: passwd iar

When prompted, enter the password redhat twice.

Password Security

While this lab uses a simple password for demonstration, in production environments always use strong passwords following your organization's password policy.

Step 3: Verify the User Account

Check user information:

Type: id iar

Should show: uid=3400(iar) gid=3400(iar) groups=3400(iar)

Check password entry:

Type: grep iar /etc/shadow

Should show an entry for user iar with an encrypted password.

Verification Commands

After creating a user, always verify with these commands:

  • id username - Shows UID, GID, and groups
  • grep username /etc/passwd - Shows account details
  • grep username /etc/shadow - Shows password details
