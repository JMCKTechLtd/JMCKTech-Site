Shared Directory Concepts

Directory Permissions drwxrwxr-x - Owner and group have full access
drwxr-xr-x - Standard directory permissions
Special Bits SETGID (2) - New files inherit directory's group
Sticky (1) - Only owner can delete their files
Key Commands mkdir - Create directories
chmod - Change permissions
chgrp - Change group ownership
Verification ls -ld - View directory permissions
stat - Detailed file information
getfacl - View ACLs if set

Step 1: Create the Directory

Create the shared directory:

Type: mkdir /home/admins

Understanding Directory Creation

The mkdir command creates new directories. When creating system directories, you typically need root privileges (use sudo or be logged in as root).

Step 2: Set Group Ownership

Change the group owner to adminuser:

Type: chgrp adminuser /home/admins

Understanding Group Ownership

The chgrp command changes group ownership of files/directories. The group must exist in /etc/group. You must have permissions to change the group (typically requires root).

Step 3: Configure Permissions

Add group write permissions:

Type: chmod g+w /home/admins

Set the SETGID bit:

Type: chmod g+s /home/admins

Understanding Special Permissions

The SETGID bit (g+s) ensures that any files created in the directory will inherit the directory's group (adminuser) rather than the creator's primary group. This is crucial for shared directories.

Final permissions should look like: drwxrwsr-x

Step 4: Verify Configuration

Check directory permissions:

Type: ls -ld /home/admins

Should display: drwxrwsr-x

Verify group ownership:

Type: stat -c "%G" /home/admins

Should return: adminuser

Verification Commands

After configuration, verify with these commands:

  • ls -ld /home/admins - Shows permissions and ownership
  • stat /home/admins - Detailed information including special bits
  • getfacl /home/admins - Shows ACLs if configured
