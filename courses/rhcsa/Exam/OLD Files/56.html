Understanding SGID Bit

What is SGID? The Set Group ID (SGID) bit is a special permission that causes files created in a directory to inherit the directory's group ownership.
Why Use SGID? Ensures all files created in a shared directory belong to the same group, facilitating collaboration among group members.
Setting SGID Symbolic: chmod g+s directory
Numeric: chmod 2770 directory (2 sets SGID)
Verification Look for s in group permissions:
drwxrws--- (lowercase s if execute is set)
drwxrwS--- (uppercase S if execute is not set)

Step 1: Set SGID Bit

Configure the /data directory so that files created in it inherit the directory's group ownership:

Set SGID bit:

Type: chmod g+s /data

How SGID Works

When SGID is set on a directory:

  • New files created in the directory will have the same group as the directory
  • New subdirectories will inherit the SGID bit
  • This is useful for shared directories where multiple users need to collaborate

Step 2: Verify SGID Bit

Check that the SGID bit was set correctly:

View directory permissions:

Type: ls -ld /data

Expected output:

drwxrws--- 2 root sysadmin 4096 Mar 16 18:08 /data

Understanding the Output

The ls -ld output shows:

  • drwxrws---: The s in group permissions indicates SGID is set
  • If execute was not set for group, you would see S (uppercase)
  • This ensures files created in /data will have sysadmin as their group
