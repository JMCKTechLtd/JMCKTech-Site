Understanding Directory Permissions

Permission Basics Linux uses three permission types: read (r), write (w), execute (x)
These are set for three entities: owner (u), group (g), others (o)
Numeric Permissions Read = 4, Write = 2, Execute = 1
770 means:
Owner: 7 (rwx), Group: 7 (rwx), Others: 0 (---)
chmod Command Syntax: chmod [options] mode file
Examples:
chmod 770 /data (numeric)
chmod u=rwx,g=rwx,o= /data (symbolic)
Verification Use ls -ld to view directory permissions
Expected output:
drwxrwx--- 2 root sysadmin 4096 Mar 16 18:08 /data

Step 1: Set Directory Permissions

Configure the /data directory so that only the owner (root) and group members (sysadmin) can access it:

Set permissions to 770:

Type: chmod 770 /data

Why 770?

The permission 770 means:

  • 7 (first digit): Owner has read (4) + write (2) + execute (1) = full access
  • 7 (second digit): Group has read + write + execute = full access
  • 0 (third digit): Others have no permissions

Step 2: Verify Permissions

Check that the permissions were set correctly:

View directory permissions:

Type: ls -ld /data

Expected output:

drwxrwx--- 2 root sysadmin 4096 Mar 16 18:08 /data

Understanding the Output

The ls -ld output shows:

  • drwxrwx---: Directory with rwx for owner and group, no permissions for others
  • 2: Link count
  • root: Owner
  • sysadmin: Group
  • 4096: Size
  • Mar 16 18:08: Last modified
  • /data: Directory name
