Lab Instructions

File Permission Requirements

‚òê Copy /etc/fstab to /var/tmp/fstab
‚òê Owner and group of /var/tmp/fstab must be root
‚òê File cannot be executed by any user
‚òê User natasha can read and write the file
‚òê User harry cannot read or write the file
‚òê All other users can read the file

Understanding File Permissions

Permission
Meaning
rw-
Read and write permissions
r--
Read-only permission
---
No permissions
setfacl
Command to set Access Control Lists (ACLs) for specific users
getfacl
Command to view ACL permissions

Step 1: Copy the File

Copy /etc/fstab to /var/tmp/fstab:

Type: cp /etc/fstab /var/tmp/fstab

Step 2: Set Owner and Group

Ensure owner and group are root:

Type: chown root:root /var/tmp/fstab

Step 3: Remove Execute Permissions

Remove all execute permissions:

Type: chmod a-x /var/tmp/fstab

Step 4: Configure User Permissions

Set permissions for natasha:

Type: setfacl -m u:natasha:rw- /var/tmp/fstab

Set permissions for harry:

Type: setfacl -m u:harry:--- /var/tmp/fstab

Set permissions for others:

Type: chmod o+r /var/tmp/fstab

Step 5: Verify Permissions

Check file permissions:

Type: ls -l /var/tmp/fstab

View ACL permissions:

Type: getfacl /var/tmp/fstab

Expected Output

The getfacl output should show:

user:natasha:rw- user:harry:---

The ls -l output should show:

-rw-r--r--+ 1 root root [size] [date] /var/tmp/fstab
