LVM Expansion Concepts

Physical Volume (PV) The base storage unit that can be a partition or whole disk. Created with pvcreate.
Volume Group (VG) A pool of physical volumes that provides storage for logical volumes. Extended with vgextend.
Logical Volume (LV) The usable "partition" that can be resized. Extended with lvextend.
Filesystem Resize After extending LV, may need resize2fs or xfs_growfs to use new space.

Step 1: Create Partition

Create a new 5GB partition with Linux LVM type (8e):

Run fdisk on the disk (e.g., /dev/hda):

Type: fdisk /dev/hda

Then follow these steps in fdisk:

n (new partition) p (primary) (partition number, e.g., 9) (start sector - default) +5G (size) t (change type) 8e (Linux LVM) w (write changes)

Step 2: Update Partition Table

Make the kernel aware of the new partition:

Run partprobe:

Type: partprobe /dev/hda

Step 3: Create Physical Volume

Initialize the new partition as a physical volume:

Create PV:

Type: pvcreate /dev/hda9

Step 4: Extend Volume Group

Add the new physical volume to your volume group:

Extend VG:

Type: vgextend test0 /dev/hda9

Step 5: Extend Logical Volume

Increase the size of your logical volume by 5GB:

Extend LV:

Type: lvextend -L+5G /dev/test0/lvtestvolume

Or alternatively: lvextend -L+5120M /dev/test0/lvtestvolume

Step 6: Verify Changes

Check that the logical volume was successfully extended:

Display LV information:

Type: lvdisplay /dev/test0/lvtestvolume

Should show the increased size (original + 5GB)

Filesystem Considerations

If you're using ext2/ext3/ext4, you may need to run:

resize2fs /dev/test0/lvtestvolume

For XFS, use:

xfs_growfs /data
