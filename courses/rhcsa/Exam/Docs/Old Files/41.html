Step 1: Configure Hostname

There are two ways to configure the hostname:

Method 1: Edit configuration file

Edit /etc/sysconfig/network and add:

NETWORKING=yes HOSTNAME=dtop5.dn.ws.com GATEWAY=172.28.10.1

Method 2: Use hostname command

Type: hostname dtop5.dn.ws.com

Understanding Hostname Configuration

The hostname identifies your system on the network. For the changes to persist after reboot, you must edit the configuration file. The hostname command only changes it temporarily.

Step 2: Configure Network Interface

Edit the network interface configuration file:

Navigate to network scripts directory:

Type: cd /etc/sysconfig/network-scripts/

Edit the interface configuration (typically ifcfg-eth0):

Type: vim ifcfg-eth0

Ensure the file contains these settings:

DEVICE=eth0 BOOTPROTO=none ONBOOT=yes IPADDR=172.28.10.5 NETMASK=255.255.255.0 GATEWAY=172.28.10.1 DNS1=172.28.10.1

Understanding Network Interface Configuration

The interface configuration file controls how your network interface behaves. Key parameters:

  • IPADDR: Sets the static IP address
  • GATEWAY: Defines the default gateway
  • DNS1: Primary DNS server
  • ONBOOT=yes: Brings interface up at boot

Step 3: Configure Additional Files

Edit hosts file:

Type: vim /etc/hosts

Ensure it contains an entry for your hostname:

127.0.0.1 localhost localhost.localdomain localhost4 localhost4.localdomain4 ::1 localhost localhost.localdomain localhost6 localhost6.localdomain6 172.28.10.5 dtop5.dn.ws.com dtop5

Edit resolv.conf:

Type: vim /etc/resolv.conf

Ensure it contains:

search dn.ws.com nameserver 172.28.10.1

Step 4: Apply Changes

Restart network service:

Type: systemctl restart network

Verify configuration:

Type: hostname (should show dtop5.dn.ws.com)

Type: ip addr show eth0 (should show 172.28.10.5)

Type: ip route show (should show gateway 172.28.10.1)

Type: cat /etc/resolv.conf (should show nameserver 172.28.10.1)

Verification Commands

After making changes, always verify with these commands:

  • hostname - Shows current hostname
  • ip addr - Displays IP addresses
  • ip route - Shows routing table including gateway
  • cat /etc/resolv.conf - Shows DNS configuration
