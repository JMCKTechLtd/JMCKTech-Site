Disk Management Concepts

Partitioning Use fdisk or parted to create partitions. Common steps: n (new), p (primary), set size, w (write).
Filesystems ext4 is the default Linux filesystem. Create with mkfs.ext4 or mkfs -t ext4.
Mounting Temporary mounts use mount command. Persistent mounts require /etc/fstab entries.
/etc/fstab Contains 6 fields: device, mount point, fs type, options, dump, fsck order.

Step 1: Create Partition

Start fdisk utility:

Type: fdisk /dev/vda

Then follow these steps in fdisk:

Command (m for help): n Partition type: p primary (0 primary, 0 extended, 4 free) e extended Select (default p): p Partition number (1-4, default 1): 5 First sector (2048-20971519, default 2048): [Enter] Last sector, +sectors or +size{K,M,G} (2048-20971519, default 20971519): +512M Command (m for help): w

Update kernel partition table:

Type: partprobe /dev/vda

About Partition Creation

Key fdisk commands:

  • n - Create new partition
  • p - Create primary partition
  • +512M - Specify partition size
  • w - Write changes to disk
  • q - Quit without saving

Always verify with lsblk or fdisk -l after changes.

Step 2: Format Partition

Create ext4 filesystem:

Type: mkfs -t ext4 /dev/vda5

About Filesystem Creation

Common filesystem types:

  • ext4 - Standard Linux filesystem
  • xfs - High-performance filesystem
  • vfat - Windows-compatible

Formatting options:

  • -t - Specify filesystem type
  • -L - Set volume label
  • -q - Quiet operation

Step 3: Create Mount Point

Create directory for mounting:

Type: mkdir -p /mnt/data

About Mount Points

Mount points:

  • Are empty directories where filesystems attach
  • Should follow FHS conventions (/mnt for temporary, /media for removable)
  • Must exist before mounting
  • Should have appropriate permissions (typically 755)

Step 4: Configure /etc/fstab

Edit fstab file:

Type: vim /etc/fstab

Add this line at the end:

/dev/vda5 /mnt/data ext4 defaults 0 0

Understanding /etc/fstab

fstab fields:

  1. Device (can use UUID with UUID=...)
  2. Mount point (must exist)
  3. Filesystem type
  4. Mount options (defaults is typical)
  5. Dump backup flag (0 to disable)
  6. Filesystem check order (0 for no check)

Step 5: Mount and Verify

Mount all filesystems:

Type: mount -a

Verify mounting:

Type: mount | grep /dev/vda5

Should show: /dev/vda5 on /mnt/data type ext4 (rw,relatime,seclabel,data=ordered)

Verification Commands

After configuration, verify with:

  • mount - Show mounted filesystems
  • lsblk - List block devices with mount points
  • df -h - Show disk space usage
  • cat /etc/fstab - Verify fstab entry
