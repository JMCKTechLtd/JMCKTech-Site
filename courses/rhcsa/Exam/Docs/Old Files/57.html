IP Forwarding Concept

Your system is going to be used as a router for two networks (192.168.0.0 and 192.168.1.0). To forward packets between these networks, you need to enable IP forwarding.

There are two aspects to configure:

  1. Enable immediately for the current session
  2. Configure to persist after reboot

Step 1: Enable IP Forwarding Immediately

To enable IP forwarding for the current session:

Type: echo "1" > /proc/sys/net/ipv4/ip_forward

Understanding Immediate IP Forwarding

The /proc/sys/net/ipv4/ip_forward file controls whether the Linux kernel will forward IPv4 packets between network interfaces. Setting this to "1" enables forwarding.

This change is temporary and will be lost after reboot.

Step 2: Configure Persistent IP Forwarding

To make IP forwarding persist after reboot:

Edit sysctl configuration file:

Type: vim /etc/sysctl.conf

Add or modify this line in the file:

net.ipv4.ip_forward = 1

Understanding Persistent IP Forwarding

The /etc/sysctl.conf file contains kernel parameters that are loaded at boot time. Adding net.ipv4.ip_forward = 1 ensures IP forwarding is enabled automatically when the system boots.

To apply changes without reboot, you can also run: sysctl -p

Step 3: Verify Configuration

Check current IP forwarding status:

Type: cat /proc/sys/net/ipv4/ip_forward

Should return: 1

Check sysctl configuration:

Type: grep ip_forward /etc/sysctl.conf

Should return: net.ipv4.ip_forward = 1
