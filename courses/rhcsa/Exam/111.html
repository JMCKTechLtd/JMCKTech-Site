0 Lab Introduction Completed

Welcome to the RHCSA Logical Volume Management Lab. In this lab, you will create and configure a logical volume according to specific requirements.

Follow the steps in order. You must complete each step before moving to the next one.

1 Create a Partition for LVM Current Step

First, we need to create a partition that we can use for our logical volume. We'll use fdisk to create a new partition.

Type this command exactly:
fdisk -cu /dev/vda

Once in fdisk, follow these steps:

  1. Type n to create a new partition
  2. Select p for primary partition
  3. Accept the default partition number
  4. Accept the default first sector
  5. Type +1G for the size (1GB)
  6. Type t to change the partition type
  7. Type 8e for Linux LVM
  8. Type w to write changes and exit

Understanding LVM Partitions

LVM requires partitions with the type set to 'Linux LVM' (8e). This tells the system that the partition will be used for Logical Volume Management.

2 Update Partition Table Pending

After creating the partition, we need to update the kernel's partition table.

Type this command exactly:
partx -a /dev/vda

This command informs the kernel about the new partition without requiring a reboot.

3 Create Physical Volume Pending

Now we'll create a physical volume from our new partition. Replace /dev/vdaX with your actual partition.

Type this command exactly:
pvcreate /dev/vdaX

Note: Replace X with your actual partition number (likely 5, 6, or 7 depending on your system).

4 Create Volume Group Pending

Now let's create the volume group with the specified extent size.

Type this command exactly:
vgcreate datastore /dev/vdaX -s 16M

The -s 16M option sets the physical extent size to 16MB as required.

Understanding Volume Groups

A volume group pools storage from one or more physical volumes. The extent size determines the smallest unit of allocation within the volume group.

5 Create Logical Volume Pending

Now we'll create the logical volume with 50 extents (800MB).

Type this command exactly:
lvcreate -l 50 -n database datastore

The -l 50 option creates a logical volume with 50 extents, and -n database names it "database".

6 Format with ext3 Filesystem Pending

Now we need to format the logical volume with the ext3 filesystem.

Type this command exactly:
mkfs.ext3 /dev/datastore/database

7 Create Mount Point Pending

Create the directory where we'll mount the logical volume.

Type this command exactly:
mkdir -p /nmt/database

8 Configure Automatic Mounting Pending

Now we need to configure the system to automatically mount the logical volume at boot.

Edit the fstab file:
vim /etc/fstab

Add this line to the file:

/dev/datastore/database /nmt/database ext3 defaults 0 0

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

9 Mount and Verify Pending

Finally, let's mount the filesystem and verify our configuration.

Type these commands one by one:
mount -a

This mounts all filesystems in /etc/fstab

df -hT

Should show the mounted logical volume at /nmt/database

lvdisplay /dev/datastore/database

Should show details about the logical volume

Verification Commands

After creating a logical volume, always verify with these commands:

  • df -hT - Shows mounted filesystems
  • lvdisplay - Shows logical volume details
  • vgdisplay - Shows volume group details
  • pvdisplay - Shows physical volume details
