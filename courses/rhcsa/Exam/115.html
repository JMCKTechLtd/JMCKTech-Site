Welcome to the RHCSA Collaborative Directory Lab. In this lab, you will create a shared directory with special permissions that allow group collaboration.

Follow the steps in order. You must complete each step before moving to the next one.

1 Create the Shared Directory Current Step

First, let's create the directory that will be used for collaboration.

Type this command exactly:
mkdir -p /mnt/shares

The -p option ensures parent directories are created if they don't exist.

Understanding Collaborative Directories

Collaborative directories allow multiple users in the same group to share files while maintaining proper ownership and permissions.

2 Set Group Ownership Pending

Now let's set the group ownership of the directory to sharegrp.

Type this command exactly:
chgrp sharegrp /mnt/shares/

This changes the group ownership to sharegrp, allowing members of that group to access the directory.

3 Set Directory Permissions Pending

Now we need to set the permissions to allow group members full access while restricting others.

Type this command exactly:
chmod 2770 /mnt/shares/

The 2770 permission set:

  • 2 - Sets the SGID bit
  • 7 - Owner has read, write, execute
  • 7 - Group has read, write, execute
  • 0 - Others have no permissions

Understanding Special Permissions

The SGID (Set Group ID) bit ensures that files created in the directory inherit the directory's group ownership rather than the user's primary group.

4 Verify Directory Configuration Pending

Let's verify that our directory is configured correctly.

Type this command exactly:
ls -ld /mnt/shares/

This should show the directory with drwxrws--- permissions and sharegrp group ownership.

5 Test the Collaborative Functionality Pending

Now let's test that the collaborative features work as expected.

Type these commands one by one:
su - harry -c "touch /mnt/shares/harry_file"

Creates a file as user harry

su - natasha -c "touch /mnt/shares/natasha_file"

Creates a file as user natasha

ls -l /mnt/shares/

Should show both files with sharegrp group ownership

Testing Collaboration

When the SGID bit is set, files created by any group member automatically belong to the directory's group, enabling seamless collaboration.
