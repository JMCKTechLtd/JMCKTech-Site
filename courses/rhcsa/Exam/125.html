0 Lab Introduction Completed

In this RHCSA-style task you will manage SELinux, firewall, and Apache configuration so that a web server can listen on TCP port 82 and be accessed successfully.

Follow the steps in order. The terminal on the right simulates the commands needed to complete the task.

1 Check current SELinux HTTP ports Current Step

First, inspect which TCP ports are currently associated with the http_port_t SELinux type.

Type this command exactly:
semanage port -l | grep http

http_port_t

SELinux restricts which ports a domain (like httpd_t) may bind to. Only ports labelled http_port_t are allowed for Apache by default.

2 Add port 82 to http_port_t Pending

Add TCP port 82 to the list of allowed HTTP ports in SELinux.

Type this command exactly:
semanage port -a -t http_port_t -p tcp 82

If the port already exists, you would use semanage port -m ... instead. For this lab, we assume it is not present yet.

3 Verify SELinux port mapping Pending

Verify that 82 now appears in the list of ports for http_port_t.

semanage port -l | grep http

4 Set Apache to listen on port 82 Pending

Edit the Apache configuration so that it listens on port 82.

Type this command exactly:
vim /etc/httpd/conf/httpd.conf

In the file, ensure you have a line:

Listen 82

Then save and exit with :wq.

5 Open port 82 in the firewall (permanent) Pending

Permanently allow TCP port 82 in the public zone.

firewall-cmd --permanent --zone=public --add-port=82/tcp

6 Reload firewall configuration Pending

Apply the new firewall rule by reloading the configuration.

firewall-cmd --reload

7 Restart Apache Pending

Restart the Apache service so it begins listening on the new port 82.

systemctl restart httpd

8 Test HTTP on port 82 Pending

Finally, confirm that the server is reachable on port 82 using wget.

wget http://node1.domain15.example.com:82

RHCSA-style verification

In the exam, a simple fetch like curl or wget is often used to check that the service is reachable and returning content.
