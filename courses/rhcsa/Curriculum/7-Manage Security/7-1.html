RHCSA Lab 7.1: Configure Firewall Rules (firewalld)

Lab Instructions

Step 1: Verify firewalld Status

Type exactly: sudo firewall-cmd --state

Understanding firewalld

What Dynamic firewall management daemon
Why Provides flexible, zone-based network security
Where Runs as a systemd service (firewalld.service)
When Default firewall solution on RHEL 7+

How it works: firewalld uses zones and services to manage firewall rules. The firewall-cmd is the primary configuration tool. Check status with --state and runtime configuration with --list-all.

Step 2: List Current Configuration

Type exactly: sudo firewall-cmd --list-all

Then type: sudo firewall-cmd --list-all-zones

Understanding Zones

What Predefined trust levels for network connections
Why Different security levels for different networks
Where Defined in /usr/lib/firewalld/zones/
When Assigned to network interfaces
Common Zones:
  • public: Default zone for untrusted networks
  • trusted: Allow all incoming traffic
  • internal: For internal networks
  • dmz: For publicly accessible services
  • block: Reject all incoming connections

Step 3: Add a Service to the Firewall

Type exactly: sudo firewall-cmd --add-service=http

Then verify with: sudo firewall-cmd --list-services

Understanding Services

What Predefined rules for common applications
Why Simplify firewall configuration
Where Defined in /usr/lib/firewalld/services/
When When allowing specific application traffic
Common Services:
  • http: Port 80 (web traffic)
  • https: Port 443 (secure web)
  • ssh: Port 22 (secure shell)
  • ftp: Port 21 (file transfer)
  • samba: Windows file sharing

Note: Changes made with --add-service are temporary. Add --permanent to make them persistent across reboots.

Step 4: Add a Port and Make Rules Permanent

Type exactly: sudo firewall-cmd --add-port=8080/tcp

Then make permanent: sudo firewall-cmd --runtime-to-permanent

Verify with: sudo firewall-cmd --list-ports

Runtime vs Permanent Configuration

What Runtime: Immediate but temporary
Why Test changes before making permanent
Where Runtime: memory; Permanent: config files
When Always test runtime first

How it works: Runtime changes take effect immediately but are lost on reload/reboot. Permanent changes are saved to configuration files but require a reload to take effect. --runtime-to-permanent copies runtime changes to permanent configuration.

Step 5: Change Default Zone and Reload

Type exactly: sudo firewall-cmd --set-default-zone=internal

Then reload: sudo firewall-cmd --reload

Verify with: sudo firewall-cmd --get-default-zone

Managing Zones

What Default zone applies to all interfaces
Why Define base security level
Where Stored in firewalld.conf
When When network environment changes

Important: The reload command applies permanent configuration changes. After changing the default zone, existing connections might be affected. Always verify with --get-default-zone.
