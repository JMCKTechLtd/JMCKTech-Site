RHCSA Lab 7.3: Apply File Context and Boolean Adjustments

Lab Instructions

Step 1: View SELinux File Contexts

Type exactly: ls -Z /var/www/html

Understanding SELinux Contexts

What Security labels applied to files and processes
Why Defines access controls for SELinux
Where Applied to all files, directories, and processes
When Viewed/modified during security configuration

How it works: The -Z option shows SELinux contexts. Contexts follow the format: user:role:type:level. The most important part is the type which defines access rules.

Step 2: Change File Context Temporarily

Type exactly: chcon -t httpd_sys_content_t /mydata/web_file.txt

Then verify with: ls -Z /mydata/web_file.txt

Understanding chcon

What Changes file context directly
Why Quick context changes for testing
Where On files needing temporary context changes
When Before establishing permanent policy

How it works: chcon changes the SELinux context directly on the file. Changes made with chcon may be lost if the file system is relabeled or if restorecon is run.

Step 3: Restore Default File Contexts

Type exactly: restorecon -v /mydata/web_file.txt

Understanding restorecon

What Restores default SELinux contexts
Why Ensures proper security labeling
Where On files with incorrect contexts
When After system changes or policy updates

How it works: restorecon resets the SELinux context based on the system policy. The -v option shows verbose output. This is the preferred way to set contexts as it uses the policy database.

Step 4: Set Permanent File Context

Type exactly: semanage fcontext -a -t httpd_sys_content_t "/mydata(/.*)?"

Then apply with: restorecon -Rv /mydata

Understanding semanage fcontext

What Manages SELinux policy rules
Why Creates persistent context changes
Where In policy for custom file locations
When Setting up services with non-standard paths

How it works: semanage fcontext adds a rule to the policy. The -a adds, -t specifies type, and the regex pattern "/mydata(/.*)?" matches the directory and all contents. restorecon -R recursively applies the new policy.

Step 5: Modify SELinux Booleans

Type exactly: getsebool -a | grep httpd

Then set a boolean: setsebool -P httpd_can_network_connect_db on

Understanding SELinux Booleans

What Toggle switches for SELinux policies
Why Adjust policy behavior without custom rules
Where System-wide policy settings
When Enabling/disabling specific features

How it works: Booleans allow runtime policy adjustments. getsebool shows current values. setsebool with -P makes changes persistent across reboots. This is often needed for services like web servers or databases.
