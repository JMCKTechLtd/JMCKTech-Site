RHCSA Lab 7.4: Configure SSH Key-based Authentication

Lab Instructions

Step 1: Generate SSH Key Pair

Type exactly: ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

When prompted, press Enter to accept default file location

When prompted for passphrase, press Enter twice (empty passphrase for this lab)

Understanding SSH Key Pairs

What Public-private key pair for authentication
Why More secure than password authentication
Where Stored in ~/.ssh/ directory
When For secure remote access to systems

How it works: ssh-keygen creates a pair of cryptographic keys. The private key (id_rsa) stays on your machine, while the public key (id_rsa.pub) is placed on remote servers.

Step 2: Copy Public Key to Remote Server

Type exactly: ssh-copy-id user@remote-server

When prompted, type "yes" to continue connecting

When prompted for password, enter the remote user's password

Understanding SSH Key Deployment

What Copies public key to remote server
Why Enables key-based authentication
Where Adds key to ~/.ssh/authorized_keys on server
When First time setting up key authentication

How it works: ssh-copy-id securely copies your public key to the remote server's authorized_keys file. After this, you can log in without a password (unless you set a passphrase).

Security Note: In production, always use a passphrase for your private key. This lab skips it for simplicity.

Step 3: Test Key-based Authentication

Type exactly: ssh user@remote-server

Understanding Key-based Login

What Passwordless authentication
Why More secure and convenient than passwords
Where Works with any SSH-enabled server
When After deploying public key

How it works: The SSH client presents your private key, and the server verifies it against the authorized public key. If they match, you're authenticated without a password prompt.

Step 4: Secure SSH Configuration (Optional)

Type exactly: sudo vi /etc/ssh/sshd_config

Find and uncomment/modify these lines:

PasswordAuthentication no

PubkeyAuthentication yes

Then save and exit (ESC, then :wq)

Finally, restart SSH: sudo systemctl restart sshd

Understanding SSH Server Security

What Server-side SSH configuration
Why Enforce key-based auth and disable passwords
Where /etc/ssh/sshd_config
When After verifying key-based auth works

How it works: These changes disable password authentication (forcing key-based auth only) and ensure public key authentication is enabled. Always test key-based auth before disabling passwords!

Warning: Don't disable password authentication until you've confirmed key-based authentication works, or you might lock yourself out!
