Welcome to the RHCSA SELinux Management Lab. In this lab, you will learn to manage SELinux settings on a Red Hat Enterprise Linux system.

Follow the steps in order. You must complete each step before moving to the next one.

1 Check SELinux Status Current Step

First, let's check the current SELinux status using the getenforce command.

Type this command exactly:
getenforce

Understanding SELinux States

Enforcing SELinux is active and enforcing policies
Permissive SELinux logs violations but doesn't enforce
Disabled SELinux is completely turned off
getenforce Shows the current SELinux mode

How it works: SELinux provides mandatory access control (MAC) for Linux systems. getenforce returns the current operational mode (Enforcing, Permissive, or Disabled).

2 Change SELinux Mode Temporarily Pending

Now let's change the SELinux mode temporarily using the setenforce command.

Type these commands in order:
setenforce 0

Switches SELinux to Permissive mode

getenforce

Verifies the mode change

setenforce 1

Switches SELinux back to Enforcing mode

Understanding setenforce

What Changes SELinux mode temporarily
Why For troubleshooting without reboot
Where Only works between Enforcing/Permissive
When When testing policy changes

How it works: setenforce 0 switches to Permissive mode, setenforce 1 returns to Enforcing. Changes don't persist after reboot.

3 View SELinux Contexts Pending

Now let's view SELinux contexts for files and processes.

Type these commands in order:
ls -Z /etc/passwd

Shows SELinux context for the passwd file

ps -Z

Shows SELinux contexts for running processes

Understanding SELinux Contexts

Files Have security contexts (user:role:type:level)
Processes Also have security contexts
-Z option Shows SELinux contexts
semanage Manages policy settings

How it works: The -Z flag shows SELinux contexts. File contexts determine access rules, process contexts determine permissions.

4 Change File Context Pending

Finally, let's change a file context using semanage and apply it with restorecon.

Type these commands in order:
semanage fcontext -a -t httpd_sys_content_t "/custom(/.*)?"

Adds a new file context rule for the /custom directory

restorecon -Rv /custom

Applies the new context recursively and verbosely

Understanding semanage

What Policy management tool
Why To modify SELinux settings persistently
Where Changes stored in policy
When When default contexts need adjustment

How it works: semanage fcontext adds file context rules. restorecon applies the new context. This is persistent across reboots.
