RHCSA Lab 5.1: Schedule Tasks (cron, at)

Lab Instructions

Step 1: Understanding cron

Type exactly: crontab -l

Understanding cron

What A time-based job scheduler in Unix-like systems
Why Automate repetitive tasks at specific times
Where System-wide (/etc/crontab) or per-user (crontab -e)
When For recurring tasks (daily, weekly, etc.)

How it works: The cron daemon (crond) runs in the background and executes scheduled commands. crontab -l lists your current cron jobs.

Cron time format:

* * * * * command-to-execute

Step 2: Creating a cron job

Type exactly: crontab -e

Then add this line exactly: * * * * * echo "Cron test" >> ~/cron_test.log

Save and exit the editor (in this lab, just type the command)

Editing crontab

What User-specific cron job configuration
Why Personal automated tasks without root access
Where Stored in /var/spool/cron/username
When When you need user-level automation

How it works: crontab -e opens your personal crontab in an editor. The example job runs every minute and appends "Cron test" to a log file.

Important: In a real system, this job would run every minute. For this lab, we're simulating the behavior.

Step 3: Verifying cron jobs

Type exactly: crontab -l

Then type: ls ~/cron_test.log

Managing cron jobs

What Viewing and verifying cron jobs
Why Ensure jobs are scheduled correctly
Where Output goes to files or email
When After creating or modifying jobs

How it works: crontab -l lists your cron jobs. The log file should appear after the job runs (simulated in this lab).

Step 4: Using the at command

Type exactly: echo "echo 'AT test' >> ~/at_test.log" | at now + 1 minute

Then type: atq

Understanding at

What One-time task scheduler
Why Run a command once at a specific time
Where Jobs stored in /var/spool/at/
When For non-recurring future tasks

How it works: at schedules a one-time job. atq lists pending jobs. The example schedules a job to run in 1 minute (simulated in this lab).

Note: In a real system, you'd need to ensure the atd service is running (systemctl status atd).

Step 5: Managing at jobs

Type exactly: atrm 1

Then verify with: atq

Removing at jobs

What Deleting scheduled at jobs
Why Cancel jobs that are no longer needed
Where Uses job numbers from atq
When When you need to cancel a scheduled job

How it works: atrm removes jobs by their ID (shown in atq output). After removal, atq should show no jobs.
