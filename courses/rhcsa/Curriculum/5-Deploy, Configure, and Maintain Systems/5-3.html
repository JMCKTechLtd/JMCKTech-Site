RHCSA Lab 5.3: Manage Services (systemctl, service)

Lab Instructions

Step 1: Check Service Status

Type exactly: systemctl status sshd

Understanding Service Status

What Current state of a service (running, stopped, etc.)
Why Verify if a service is operational
Where On any systemd-managed Linux system
When Before making changes to services

How it works: systemctl status shows detailed information about a service including its current state, recent logs, and whether it's enabled to start at boot.

Step 2: Start and Stop Services

Type exactly: sudo systemctl stop sshd

Then type: systemctl status sshd (verify it's stopped)

Then type: sudo systemctl start sshd

Understanding Service Control

What Starting and stopping system services
Why Manage service availability without rebooting
Where On any system with systemd
When When you need to temporarily enable/disable services

How it works: systemctl start activates a service immediately, while systemctl stop deactivates it. These changes don't persist across reboots.

Step 3: Enable and Disable Services

Type exactly: sudo systemctl disable sshd

Then type: systemctl is-enabled sshd (verify it's disabled)

Then type: sudo systemctl enable sshd

Understanding Service Persistence

What Configuring services to start at boot
Why Ensure critical services start automatically
Where Managed in /etc/systemd/system/
When When configuring system services

How it works: systemctl enable creates symlinks to start the service at boot, while systemctl disable removes them. This doesn't affect the current running state.

Step 4: Legacy Service Commands

Type exactly: sudo service sshd restart

Then type: service --status-all

Understanding Legacy Service Management

What Traditional service management commands
Why Compatibility with older systems/scripts
Where On systems with SysV init compatibility
When When working with older systems

How it works: The service command provides backward compatibility with SysV init scripts. service sshd restart stops and starts the service, while --status-all shows all services.
