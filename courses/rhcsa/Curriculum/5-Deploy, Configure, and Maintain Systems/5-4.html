Welcome to the RHCSA Networking Configuration Lab. In this lab, you will learn essential network configuration commands for Red Hat Enterprise Linux.

Follow the steps in order. You must complete each step before moving to the next one.

1 Network Interface Information Current Step

First, let's examine the current network interface configuration using the ip command.

Type this command exactly:
ip addr show

Understanding Network Interfaces

What Physical and virtual network interfaces
Why Identify available network connections
Where System-wide network configuration
When Initial setup or troubleshooting

How it works: The ip command shows detailed network information. Common interfaces include eth0 (Ethernet) and wlan0 (Wi-Fi). Look for inet addresses (IPv4) and state (UP/DOWN).

2 NetworkManager Control Pending

Now let's examine NetworkManager connections using the nmcli command.

Type this command exactly:
nmcli connection show

Understanding NetworkManager

What NetworkManager service (nmcli)
Why Manage network connections consistently
Where System service controlling all interfaces
When Configuring or modifying network settings

How it works: nmcli is the command-line interface to NetworkManager. It shows connection profiles with their names, UUIDs, devices, and types (ethernet, wifi, etc.).

3 Configure Static IP Pending

Now let's configure a static IP address for the "Wired connection 1" connection.

Type this command exactly:
nmcli connection modify "Wired connection 1" ipv4.addresses 192.168.1.100/24

Then activate the connection:

nmcli connection up "Wired connection 1"

Understanding Static IP Configuration

What Manual IP address assignment
Why Consistent addressing for servers
Where Connection profile settings
When Setting up servers or network devices

How it works: The modify subcommand changes connection properties. ipv4.addresses sets the static IP (192.168.1.100) with netmask (/24). connection up activates the changes.

4 Hostname Management Pending

Now let's set and verify the system hostname.

Type this command exactly:
hostnamectl set-hostname rhcsa-lab

Then verify the change:

hostnamectl status

Understanding Hostname Control

What System hostname configuration
Why Identify system on the network
Where System-wide setting
When Initial setup or renaming

How it works: hostnamectl manages the system hostname. Changes are persistent across reboots. The status shows static (configured), transient (DHCP), and pretty (formatted) hostnames.

5 Connectivity Testing Pending

Finally, let's test network connectivity to verify our configuration.

Type this command exactly:
ping -c 4 8.8.8.8

Understanding Network Testing

What Basic connectivity verification
Why Confirm network functionality
Where Any system with network access
When After configuration changes

How it works: ping sends ICMP packets to test reachability. -c 4 sends exactly 4 packets. Success shows response times; failure may indicate configuration issues.
