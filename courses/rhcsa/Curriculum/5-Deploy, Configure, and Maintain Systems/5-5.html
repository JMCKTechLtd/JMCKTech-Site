RHCSA Lab 5.5: Manage Users & Groups

Lab Instructions

Step 1: Create a New User

Type exactly: sudo useradd -m -s /bin/bash rhcsa_user

Then type: sudo passwd rhcsa_user and set password to "redhat"

Understanding User Creation

What Creating a new system user account
Why Each user needs their own account for security and resource management
Where User info stored in /etc/passwd and /etc/shadow
When When adding new users to the system

How it works: useradd creates the account, -m creates home directory, -s sets shell. passwd sets/changes the password. Always use strong passwords in production!

Step 2: Create a New Group

Type exactly: sudo groupadd rhcsa_group

Understanding Group Management

What Creating a new user group
Why Groups simplify permission management for multiple users
Where Group info stored in /etc/group
When When you need to share resources among users

How it works: groupadd creates a new group. Groups are essential for managing file permissions and system access.

Step 3: Add User to Group

Type exactly: sudo usermod -aG rhcsa_group rhcsa_user

Understanding User Modification

What Modifying user account properties
Why To change user attributes like groups, shell, or home directory
Where Updates /etc/group and /etc/passwd
When When user attributes need modification

How it works: usermod modifies user accounts. -aG appends the user to supplementary groups without removing existing group memberships.

Step 4: Verify User and Group

Type exactly: id rhcsa_user

Then type: groups rhcsa_user

Understanding User Verification

What Checking user and group information
Why Verify correct configuration of users and groups
Where Queries system user/group databases
When After creating or modifying users/groups

How it works: id shows user and group IDs, groups shows group memberships. These commands help verify your changes were applied correctly.
