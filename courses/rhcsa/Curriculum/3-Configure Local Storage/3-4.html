Welcome to the RHCSA Filesystem Mounting Lab. In this lab, you will learn how to manage filesystems in Linux - from temporary mounts to persistent configurations.

Follow the steps in order. You must complete each step before moving to the next one.

1 View Mounted Filesystems Current Step

First, let's view the currently mounted filesystems using the mount command.

Type this command exactly:
mount

Understanding Mounted Filesystems

What Shows currently mounted filesystems
Why Verify what's mounted and where
Where Works on any Linux system
When Before/after mounting operations

How it works: The mount command without arguments displays all mounted filesystems with their mount points, types, and options.

2 Mount a Filesystem Temporarily Pending

Now let's mount a filesystem temporarily. This mount will persist until the next reboot.

Type this command exactly:
sudo mount /dev/sdb1 /mnt

Then verify the mount was successful:

mount | grep /mnt

Understanding Temporary Mounts

What Mounts a filesystem until reboot
Why Temporary access to storage
Where Any available block device
When For temporary storage needs

How it works: mount attaches the filesystem on /dev/sdb1 to the directory /mnt. The sudo command is needed for root privileges.

3 Unmount a Filesystem Pending

Now let's safely unmount the filesystem we just mounted.

Type this command exactly:
sudo umount /mnt

Then verify the unmount was successful:

mount | grep /mnt

Understanding Unmounting

What Safely detaches a filesystem
Why Prevent data corruption
Where Any mounted filesystem
When Before removing storage

How it works: umount (note the spelling) detaches the filesystem from its mount point. The filesystem must not be in use (no open files or processes using it).

4 Configure Persistent Mounts (/etc/fstab) Pending

Now let's configure a persistent mount that will survive reboots by editing the /etc/fstab file.

Type this command exactly:
sudo vim /etc/fstab

Once in the editor, add this line to the file (use Tab between fields):

/dev/sdb1 /mnt ext4 defaults 0 0

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Then test the configuration:

sudo mount -a

Understanding /etc/fstab

What Filesystem table configuration
Why Persistent mounts across reboots
Where /etc/fstab
When System boot or mount -a

How it works: The fstab file defines how filesystems should be mounted. Fields are: device, mount point, type, options, dump, fsck order.

Field Example Description
Device /dev/sdb1 Block device or UUID
Mount point /mnt Directory to mount to
Filesystem type ext4 Type of filesystem
Options defaults Mount options (ro,rw,etc)
Dump 0 Backup utility flag
Fsck order 0 Filesystem check order
