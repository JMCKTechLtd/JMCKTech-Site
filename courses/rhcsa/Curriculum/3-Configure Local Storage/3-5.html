Welcome to the RHCSA Swap Space Management Lab. In this lab, you will learn how to create, manage, and configure swap space on a Red Hat Enterprise Linux system.

Follow the steps in order. You must complete each step before moving to the next one.

1 Checking Current Swap Space Current Step

First, let's check the current swap space on the system.

Type these commands exactly:
swapon --show

Then type:

free -h

Understanding Swap Space

What Disk space used when RAM is full
Why Prevents system crashes when memory is exhausted
Where Configured in /etc/fstab or created manually
When Needed when physical memory is insufficient

How it works: swapon --show displays active swap areas. free -h shows memory usage including swap in human-readable format. Swap can be a dedicated partition or a swap file.

2 Creating a Swap File Pending

Now let's create a 512MB swap file.

Type these commands exactly:
sudo fallocate -l 512M /swapfile

Then type:

sudo chmod 600 /swapfile

Then type:

sudo mkswap /swapfile

Understanding Swap File Creation

What Creating a file-based swap space
Why Flexible alternative to swap partitions
Where Typically in root directory (/swapfile)
When When you need temporary swap space

How it works: fallocate creates the file, chmod sets secure permissions, and mkswap formats it as swap space. Swap files are useful when you can't repartition the disk.

3 Activating and Deactivating Swap Pending

Now let's activate and deactivate the swap file.

Type these commands exactly:
sudo swapon /swapfile

Then type:

swapon --show

Then type:

sudo swapoff /swapfile

Understanding Swap Management

What Enabling/disabling swap space
Why Temporary activation or system maintenance
Where Works on both partitions and files
When When adjusting memory resources

How it works: swapon activates swap space, while swapoff deactivates it. Changes are temporary unless added to /etc/fstab. Always verify with swapon --show.

4 Making Swap Persistent Pending

Finally, let's make the swap file persistent across reboots.

Type these commands exactly:
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

Then type:

sudo swapon -a

Then type:

swapon --show

Understanding Persistent Swap

What Automatically activated swap at boot
Why Ensures swap is available after reboot
Where Configured in /etc/fstab
When For permanent swap space

How it works: The /etc/fstab entry tells the system to activate this swap space at boot. swapon -a activates all swap spaces listed in /etc/fstab. This ensures your swap space survives reboots.
