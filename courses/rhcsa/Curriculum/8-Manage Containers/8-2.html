RHCSA Lab 8.2: Run, List, and Remove Containers (Podman)

Lab Instructions

Step 1: Run Your First Container

Type exactly: podman run -d --name web_server httpd

Understanding Container Execution

What Running a container from an image
Why Isolate applications with their dependencies
Where On your local system with Podman
When When you need to deploy an application

How it works: podman run creates and starts a container. -d runs it in detached mode, --name assigns a name, and httpd is the Apache HTTP Server image.

Step 2: List Running Containers

Type exactly: podman ps

Understanding Container Listing

What View active containers
Why Monitor running containers and their status
Where On your local system with Podman
When After starting containers or troubleshooting

How it works: podman ps shows running containers. Add -a to see all containers (including stopped ones). The output shows container ID, image, command, status, and ports.

Step 3: Stop a Container

Type exactly: podman stop web_server

Understanding Container Lifecycle

What Gracefully stopping a container
Why Properly shut down applications
Where On your running container
When When you need to stop a service

How it works: podman stop sends a SIGTERM signal to the main process in the container, allowing it to shut down gracefully. Use podman kill for immediate termination.

Step 4: Remove a Container

Type exactly: podman rm web_server

Understanding Container Removal

What Deleting a container
Why Clean up unused containers
Where On your local system storage
When When a container is no longer needed

How it works: podman rm removes a stopped container. Add -f to force removal of a running container. The container filesystem is deleted, but the image remains.

Step 5: Run and Auto-Remove

Type exactly: podman run --rm -it alpine sh

Then type: exit in the container shell

Understanding Ephemeral Containers

What Temporary containers that auto-remove
Why For one-time tasks without leaving traces
Where On your local system with Podman
When For testing or temporary access

How it works: --rm automatically removes the container when it exits. -it allocates a pseudo-TTY and keeps STDIN open. alpine is a minimal Linux image, and sh starts a shell.
