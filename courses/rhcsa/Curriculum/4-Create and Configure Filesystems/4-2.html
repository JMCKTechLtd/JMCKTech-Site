RHCSA Lab 4.2: Configure ACLs (Access Control Lists)

Lab Instructions

Step 1: Verify ACL Support

Type exactly: mount | grep acl

Understanding ACL Support

What Extended file permissions beyond standard Unix permissions
Why Allows more granular access control for multiple users/groups
Where Filesystems must be mounted with acl option
When Needed when standard permissions are insufficient

How it works: The mount command shows mounted filesystems. The acl option must be present for ACL support. Most modern Linux distributions mount filesystems with ACL support by default.

Step 2: View ACLs

Type exactly: getfacl /etc/passwd

Understanding getfacl

What Displays Access Control Lists for files/directories
Why View current ACL settings
Where Works on any file/directory with ACL support
When Before modifying ACLs or troubleshooting access

How it works: getfacl shows the file's standard permissions first, followed by any extended ACL entries. Each entry specifies a user or group and their permissions.

Step 3: Set ACL for a User

Type exactly: touch testfile

Then type: setfacl -m u:student:rw testfile

Finally type: getfacl testfile

Understanding setfacl

What Modifies Access Control Lists
Why Grant/revoke permissions for specific users/groups
Where On files/directories with ACL support
When When standard permissions aren't sufficient

How it works: The -m option modifies ACLs. u:student:rw grants read-write permissions to user 'student'. The format is u|g:name:permissions.

Step 4: Set ACL for a Group

Type exactly: setfacl -m g:developers:rx testfile

Then type: getfacl testfile

Understanding Group ACLs

What ACL entries for groups
Why Control access for entire groups
Where Same as user ACLs
When When multiple users need the same access

How it works: Similar to user ACLs but with g: prefix. g:developers:rx grants read-execute to the 'developers' group. Multiple ACL entries can exist for different groups.

Step 5: Remove ACL Entry

Type exactly: setfacl -x g:developers testfile

Then type: getfacl testfile

Understanding ACL Removal

What Removing specific ACL entries
Why Revoke previously granted permissions
Where On files/directories with ACLs
When When access needs to be revoked

How it works: The -x option removes ACL entries. Specify the entry type (u or g) and name, but not permissions. To remove all ACLs, use setfacl -b.

Step 6: Default ACLs

Type exactly: mkdir testdir

Then type: setfacl -m d:u:student:rwx testdir

Finally type: getfacl testdir

Understanding Default ACLs

What ACLs inherited by new files/directories
Why Automatically apply ACLs to new items
Where On directories only
When When consistent permissions are needed for new items

How it works: The d: prefix sets default ACLs. New files/directories in this directory will inherit these ACLs. Default ACLs don't affect existing items.
