RHCSA Lab 4.4: Create & Manage Hard/Soft Links

Lab Instructions

Step 1: Create a Sample File

Type exactly: echo "This is original content" > original_file.txt

Then type: ls -l original_file.txt

Understanding Files and Inodes

What Files store data, inodes store metadata
Why Need original file before creating links
Where Files stored in filesystem with unique inodes
When Before creating any type of link

How it works: The echo command creates a file with content. ls -l shows file details including permissions, size, and modification time.

Step 2: Create a Hard Link

Type exactly: ln original_file.txt hard_link.txt

Then type: ls -li original_file.txt hard_link.txt

Understanding Hard Links

What Additional name for the same inode
Why Multiple access points to same data
Where Must be on same filesystem
When Need multiple names for same file

How it works: ln creates a hard link. Notice identical inode numbers and link counts in ls -li output. Changes to one affect all links.

Step 3: Create a Soft Link (Symbolic Link)

Type exactly: ln -s original_file.txt soft_link.txt

Then type: ls -li original_file.txt soft_link.txt

Understanding Soft Links

What Pointer to another file path
Why Link across filesystems or to directories
Where Anywhere in the filesystem
When Need flexible references

How it works: ln -s creates a symbolic link. Note different inode numbers. The link shows as soft_link.txt -> original_file.txt in ls -l output.

Step 4: Compare Link Behavior

Type exactly: rm original_file.txt

Then type: cat hard_link.txt

Then type: cat soft_link.txt

Understanding Link Differences

Hard Links Survive original deletion (same inode)
Soft Links Break if original deleted (dangling link)
Hard Link Limits Can't link directories or cross filesystems
Soft Link Advantages Can link to directories and across filesystems

Key Differences: After deleting the original, the hard link still works (same data), while the soft link fails (points to non-existent path).
