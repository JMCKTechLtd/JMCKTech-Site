RHCSA Lab 4.1: File Permissions & Ownership

Lab Instructions

Step 1: Viewing Permissions

Type exactly: ls -l /var/tmp/permission_lab/file1.txt

Understanding Permission Bits

What File mode bits (rwxrwxrwx)
Why Controls access to files/directories
Where Displayed with ls -l
When Always visible for all files

How it works: Permissions are shown as 10 characters (e.g., -rw-r--r--). The first character is file type, followed by three sets of rwx (owner, group, others).

Typical permission visualization

Step 2: Changing Permissions (chmod)

Type exactly: chmod 755 /var/tmp/permission_lab/file1.txt

Then verify with: ls -l /var/tmp/permission_lab/file1.txt

Understanding chmod

What Change file mode bits
Why Modify access permissions
Where Applied to files/directories
When When access needs adjustment

How it works: chmod accepts numeric (octal) or symbolic modes. 755 means rwxr-xr-x (7=rwx, 5=r-x). Owner gets all, group/others get read+execute.

RHCSA Tip: You must know both numeric (755) and symbolic (u=rwx,go=rx) formats for the exam.

Step 3: Changing Ownership (chown)

Type exactly: sudo chown root:root /var/tmp/permission_lab/file2.txt

Then verify with: ls -l /var/tmp/permission_lab/file2.txt

Understanding chown

What Change file owner/group
Why Transfer ownership or change group
Where Applied to files/directories
When When ownership needs changing

How it works: chown user:group file changes both owner and group. Requires root privileges. The sudo command elevates privileges temporarily.

Step 4: Changing Group (chgrp)

Type exactly: sudo chgrp wheel /var/tmp/permission_lab/file3.txt

Then verify with: ls -l /var/tmp/permission_lab/file3.txt

Understanding chgrp

What Change file group only
Why When only group needs changing
Where Alternative to chown
When More specific than chown

How it works: chgrp changes only the group ownership. You must be the file owner or root to change the group.

Step 5: Understanding umask

Type exactly: umask

Then type: umask 0022

Understanding umask

What Default permission mask
Why Sets default permissions for new files
Where Shell environment setting
When Affects all new file creations

How it works: umask subtracts permissions from the default (666 for files, 777 for directories). 0022 results in 644 (rw-r--r--) for files and 755 (rwxr-xr-x) for directories.

RHCSA Tip: Know how to calculate permissions using umask (e.g., 666 - 022 = 644).
