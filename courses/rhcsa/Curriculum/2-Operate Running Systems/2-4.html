1 Listing Processes Current Step

First, let's list all running processes to see what's currently running on the system.

Type this command exactly:
ps aux

Understanding Process Management

What Processes are running program instances
Why Need to manage system resources and troubleshoot
Where Viewable with ps, top, htop commands
When When managing system performance or stopping misbehaving programs

How it works: ps aux shows all running processes with details. Each process has a unique PID (Process ID) that identifies it.

2 Killing by PID Pending

Now let's terminate a specific process by its Process ID (PID).

Type this command exactly:
kill 1256

Understanding the kill Command

What Terminates processes by PID
Why Precise control over specific processes
Where Works on any running process you own (or as root)
When When you know the exact PID to terminate

How it works: kill sends signal 15 (TERM) by default, asking the process to terminate gracefully. Use kill -9 for force termination.

Common Signals

1 (SIGHUP) - Hangup, reload configuration
2 (SIGINT) - Interrupt (Ctrl+C)
9 (SIGKILL) - Force kill (cannot be ignored)
15 (SIGTERM) - Graceful termination (default)
18 (SIGCONT) - Continue stopped process
19 (SIGSTOP) - Stop/pause process

3 Killing by Name (pkill) Pending

Now let's terminate processes by name using pkill.

Type this command exactly:
pkill sleep

Understanding pkill

What Kills processes by name pattern
Why Convenient when you don't know the PID
Where Works on process names or other attributes
When When targeting multiple processes with similar names

How it works: pkill matches processes by name (or other criteria) and sends them the specified signal (TERM by default). Use pgrep first to see what it would match.

Warning: pkill can match more processes than intended! Always verify with pgrep first.

4 Killing All Instances (killall) Pending

Finally, let's terminate all instances of a specific program using killall.

Type this command exactly:
killall -v bash

Understanding killall

What Kills all processes matching exact name
Why Quick way to terminate all instances of a program
Where Useful for services with multiple workers
When When you want to stop all copies of a specific program

How it works: killall matches processes by exact name (unlike pkill which uses patterns). The -v option shows verbose output about what was killed.

Danger: killall can be dangerous - on some systems, killall kills ALL processes (like kill -9 -1)!
