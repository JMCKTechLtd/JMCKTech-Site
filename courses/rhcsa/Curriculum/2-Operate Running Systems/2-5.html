RHCSA Lab 2.5: System Log Files (/var/log, journalctl)

Lab Instructions

Step 1: Exploring /var/log

Type exactly: ls -l /var/log

Understanding System Log Files

What System log files stored in /var/log
Why Record system events for troubleshooting
Where /var/log directory (requires root for some files)
When Continuously updated by system services

Key log files:

  • messages: General system messages
  • secure: Authentication logs
  • boot.log: Boot process messages
  • dmesg: Kernel ring buffer messages

Step 2: Viewing Log Contents

Type exactly: sudo tail -20 /var/log/messages

Reading Log Files

What Tools to view log contents
Why Monitor and troubleshoot system issues
Where Command line with appropriate permissions
When When investigating system behavior

Common tools:

  • tail: View end of file (use -n for line count)
  • head: View beginning of file
  • less: Interactive file viewer
  • grep: Filter log entries

Step 3: Using journalctl

Type exactly: journalctl -n 10

Then type: journalctl --since "1 hour ago"

Understanding journalctl

What Systemd's journal logging system
Why Centralized logging with metadata
Where Binary logs stored in /var/log/journal
When Modern systems using systemd

Key features:

  • Structured logging with metadata
  • Persistent or volatile storage
  • Powerful filtering options
  • Integrated with systemd services

Step 4: Filtering Logs

Type exactly: journalctl -p err

Then type: journalctl -u sshd

Advanced Log Filtering

What Filtering logs by priority, service, etc.
Why Focus on relevant log entries
Where journalctl with filtering options
When Troubleshooting specific issues

Common filters:

  • -p: Priority (emerg, alert, crit, err, warning, notice, info, debug)
  • -u: Unit/service name
  • --since/--until: Time ranges
  • -k: Kernel messages only
