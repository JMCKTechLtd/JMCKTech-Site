1 Output Redirection (>) Current Step

First, let's learn output redirection using the > operator.

Type this command exactly:
echo "Hello RHCSA" > hello.txt

Then verify the file was created:

cat hello.txt

Understanding Output Redirection

What Sends command output to a file instead of screen
Why To save output for later use or logging
Where Works with most commands that produce output
When When you need to capture command output

How it works: The > operator creates/overwrites a file. echo writes to the file instead of the terminal. cat displays file contents.

Operator Description Example
> Overwrite file ls > files.txt
>> Append to file date >> log.txt

2 Input Redirection (<) Pending

Now let's learn input redirection using the < operator.

Type this command exactly:
wc -l < hello.txt

Understanding Input Redirection

What Uses file contents as command input
Why To process files without typing content
Where Works with commands expecting stdin
When When feeding files to commands

How it works: The < operator makes the command read from the file instead of keyboard input. wc -l counts lines from the redirected input.

3 Pipes (|) Pending

Now let's learn how to use pipes to connect commands.

Type this command exactly:
cat hello.txt | grep "RHCSA"

Understanding Pipes

What Connects command output to another's input
Why To chain commands together
Where Between any commands producing/consuming text
When Creating processing pipelines

How it works: The | takes cat's output and feeds it to grep, which filters for lines containing "RHCSA".

4 Error Redirection (2>) Pending

Finally, let's learn error redirection using the 2> operator.

Type this command exactly:
ls /nonexistent 2> error.log

Then check the error log:

cat error.log

Understanding Error Redirection

What Redirects error messages separately
Why To handle errors differently than normal output
Where When commands may produce errors
When Debugging or logging errors

How it works: File descriptor 2 (stderr) is redirected to a file. Normal output would still appear on screen unless also redirected.

Descriptor Stream Example
0 stdin (input) < input.txt
1 stdout (output) > output.txt
2 stderr (errors) 2> errors.txt
