1 Basic grep Usage Current Step

First, let's learn the basic usage of grep to search for text patterns in files.

Type this command exactly:
grep "error" sample.log

Then type:

grep -i "warning" sample.log

Understanding grep Basics

What Searches for patterns in text
Why To find specific information in files
Where Works on text files or command output
When When you need to filter or search text

How it works: grep scans each line of input for matches to the pattern. The -i option makes the search case-insensitive.

Option Description Example
-i Case-insensitive search grep -i "hello" file.txt
-v Invert match (non-matching lines) grep -v "success" log.txt
-n Show line numbers grep -n "error" log.txt

2 Regular Expression Basics Pending

Now let's learn about regular expression anchors to match text at specific positions.

Type this command exactly:
grep "^2023" sample.log

Then type:

grep "failed$" sample.log

Understanding Regular Expressions

What Patterns that match text
Why For powerful text searching
Where Used with grep, sed, awk, etc.
When When simple text matching isn't enough

How it works: The ^ matches start of line, $ matches end of line. These anchors help find text in specific positions.

Sample log file content:
2023-01-10 10:00:01 System started
2023-01-10 10:05:23 Warning: disk space low
2023-01-10 10:10:45 Error: connection failed
2023-01-10 10:15:12 Operation completed successfully

3 Character Classes Pending

Now let's explore character classes for matching specific sets of characters.

Type this command exactly:
grep "[0-9]%" sample.log

Then type:

grep "[AEIOU]" sample.log

Understanding Character Classes

What Match any character from a set
Why To match variations of a pattern
Where Inside regular expressions
When When you need flexible matching

How it works: [0-9] matches any digit, [AEIOU] matches any uppercase vowel. Use [^...] to negate the match.

Pattern Matches Example
[abc] a, b, or c gr[ae]y (gray or grey)
[a-z] Any lowercase letter [a-z]@domain.com
[^0-9] Any non-digit [^0-9]%

4 Extended Regular Expressions Pending

Finally, let's explore extended regular expressions for more complex pattern matching.

Type this command exactly:
grep -E "error|warning" sample.log

Then type:

grep -E "[0-9]{3}-[0-9]{2}-[0-9]{4}" data.txt

Understanding Extended Regex

What More powerful pattern matching
Why For complex matching requirements
Where With grep -E or egrep
When When basic regex isn't sufficient

How it works: -E enables extended regex features. | means OR, {n} matches exactly n occurrences.

Pattern Meaning Example
| Alternation (OR) error|warning
+ One or more occurrences [0-9]+
{n} Exactly n occurrences [0-9]{3}
