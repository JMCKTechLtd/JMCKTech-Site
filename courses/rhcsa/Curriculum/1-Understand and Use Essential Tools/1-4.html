RHCSA Lab 1.4: Manage Files & Directories

Lab Instructions

Step 1: Listing Files

Type exactly: ls -l

Then type: ls -a

Understanding ls Command

What Lists directory contents
Why View files and directories
Common Options -l (long format), -a (show hidden)
Output Meaning Permissions, owner, size, date

How it works: ls displays directory contents. -l shows detailed information, -a includes hidden files (starting with .). Combine options like ls -la.

Initial file structure in your lab environment:
./documents/
./file1.txt
./file2.txt
./.hidden_file

Step 2: Creating and Copying Files

Type exactly: touch newfile.txt

Then type: cp newfile.txt copied_file.txt

Then type: cp -r documents documents_backup

Understanding touch and cp Commands

touch Creates empty files or updates timestamps
cp Copies files/directories
Common Options -r (recursive for directories)
Paths Can use relative or absolute paths

How it works: touch creates files if they don't exist. cp requires source and destination. For directories, use -r option.

Step 3: Moving and Renaming Files

Type exactly: mv copied_file.txt documents/

Then type: mv documents/copied_file.txt documents/renamed_file.txt

Understanding mv Command

What Moves or renames files/directories
Renaming Same command as moving within same directory
Caution Overwrites existing files without warning
Options -i (interactive), -n (no overwrite)

How it works: mv changes file location or name. Syntax is similar to cp but doesn't need -r for directories.

Step 4: Removing Files and Directories

Type exactly: rm file1.txt

Then type: rm -r documents_backup

Understanding rm Command

What Removes files/directories
Danger Permanent deletion (no trash bin)
Common Options -r (recursive), -i (interactive)
Safety Consider -i or --preserve-root

How it works: rm deletes files permanently. Use -r for directories. Be extremely careful with wildcards (rm *).

Step 5: Finding Files

Type exactly: find . -name "*.txt"

Then type: find /etc -type f -name "*.conf"

Understanding find Command

What Searches for files/directories
Power Can search by name, type, size, time, etc.
Common Options -name, -type, -size, -exec
Performance Searches recursively through directories

How it works: find searches directory trees. Basic syntax: find [path] [criteria] [action]. Can combine multiple tests.
