RHCSA Lab 1.5: Create and Edit Text Files

Lab Instructions

Step 1: Creating Files with Nano

Type exactly: nano my_first_file.txt

Nano Editor Basics

After opening nano, follow these steps:

  1. Type some text (e.g., "Hello, this is my first file")
  2. Press Ctrl+O to save (Write Out)
  3. Press Enter to confirm filename
  4. Press Ctrl+X to exit

Understanding Nano

What Simple, beginner-friendly text editor
Why Easy to use with on-screen help
Where Available on most Linux systems
When Quick edits or when learning

How it works: Nano shows commands at the bottom of the screen. Ctrl+G displays help. Files are saved in current directory unless full path is specified.

Step 2: Viewing File Contents

Type exactly: cat my_first_file.txt

Then type: ls -l my_first_file.txt

Understanding File Operations

What cat and ls -l
Why View contents and file metadata
Where Anywhere in the filesystem
When After creating or modifying files

How it works: cat concatenates and displays file contents. ls -l shows detailed file information including permissions, owner, size, and modification time.

Step 3: Editing with Vim

Type exactly: vim my_second_file.txt

Vim Editor Basics

After opening vim, follow these steps:

  1. Press i to enter Insert mode
  2. Type some text (e.g., "Learning Vim for RHCSA")
  3. Press Esc to return to Normal mode
  4. Type :wq and press Enter to save and quit

Understanding Vim

What Powerful modal text editor
Why Efficient text manipulation
Where Installed by default on RHEL
When Advanced editing or system administration

How it works: Vim has modes: Normal (navigation), Insert (editing), and Command-line (commands). Start in Normal mode, i to insert, Esc to return to Normal mode, :wq to write and quit.

Step 4: Advanced Vim Operations

Type exactly: vim my_second_file.txt

Vim Advanced Operations

After opening the file, perform these actions:

  1. Navigate to a line and press dd to delete a line
  2. Press u to undo the deletion
  3. Type :set number to show line numbers
  4. Type :10 to jump to line 10 (creates new lines if needed)
  5. Type :wq to save and quit

Understanding Vim Efficiency

What Advanced Vim operations
Why Faster editing workflow
Where Useful in large files
When Frequent editing tasks

How it works: Vim's power comes from combining commands. dd deletes a line, u undoes, Ctrl+r redoes. :set number shows line numbers. :10 jumps to line 10.
