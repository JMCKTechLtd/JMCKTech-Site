Understanding Podman

What A daemonless container engine (Docker alternative)
Why Runs containers securely without root daemon
Where Available in RHEL 8+ repositories
When When you need to manage OCI containers

How it works: Podman is compatible with Docker CLI commands but runs containers without a daemon. The --version flag shows the installed version.

Understanding Image Pull

What Downloads container image from registry
Why To get images for container deployment
Where From configured container registries
When Before running a container for the first time

How it works: Podman pulls images from registries like Docker Hub, Red Hat Registry, or Quay.io. UBI (Universal Base Image) is Red Hat's free-to-use container base image.

Now let's list the container images available locally.

Understanding Image Management

What Lists locally available container images
Why Verify images are downloaded correctly
Where Stored in local storage (usually ~/.local/share/containers)
When After pulling or building images

How it works: The images command shows repository, tag, image ID, size, and creation date. Images are stored in layers for efficiency.

Understanding Image Tags

What Version identifier for container images
Why To specify exact image versions
Where Appended after repository name with colon
When When you need a specific image version

How it works: Tags typically represent versions (8.6), stability (latest), or variants (minimal). If no tag is specified, latest is assumed.

Finally, let's search the registry for available UBI images.

Type this command exactly: 
podman search ubi --limit=3

Understanding Image Search

What Finds images in configured registries
Why Discover available container images
Where Searches registry.access.redhat.com by default
When When you need to find specific software

How it works: The search command queries registries for matching images. --limit restricts results. Official images are marked "OK" in output.
