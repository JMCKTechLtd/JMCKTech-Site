Understanding /etc/hosts

What: Static hostname resolution file

Why: Maps IP addresses to hostnames locally

Format: Each line contains an IP address followed by hostnames (primary first, then aliases).

127.0.0.1 localhost localhost.localdomain localhost4 localhost4.localdomain4 ::1 localhost localhost.localdomain localhost6 localhost6.localdomain6

Now let's add a static host entry to the hosts file.

Type the command shown below:

sudo nano /etc/hosts

Add this line to the file:

192.168.1.100 server1.example.com server1

To save and exit nano:

  1. Press Ctrl+O to write the file
  2. Press Enter to confirm the filename
  3. Press Ctrl+X to exit

Editing System Files

How it works: sudo elevates privileges. nano is a simple text editor. After saving, verify changes with cat.

4 Configure DNS Resolution Current Step

Now let's configure DNS resolution by editing the resolv.conf file.

First, view the current configuration:

cat /etc/resolv.conf

Then edit the file:

sudo nano /etc/resolv.conf

Add these lines to the file:

nameserver 8.8.8.8 nameserver 8.8.4.4

Understanding /etc/resolv.conf

What: DNS resolver configuration

Why: Specifies DNS servers to use

Format: Each nameserver line specifies a DNS server (max 3). They're queried in order.

[user@lab ~]$ ping -c 3 server1 PING server1.example.com (192.168.1.100) 56(84) bytes of data. 64 bytes from server1.example.com (192.168.1.100): icmp_seq=1 ttl=64 time=0.123 ms 64 bytes from server1.example.com (192.168.1.100): icmp_seq=2 ttl=64 time=0.456 ms 64 bytes from server1.example.com (192.168.1.100): icmp_seq=3 ttl=64 time=0.789 ms Step 3 completed.
