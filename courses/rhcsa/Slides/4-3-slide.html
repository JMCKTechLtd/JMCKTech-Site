File Attributes Reference

Common File Attributes

Attribute Description
a (append only) File can only be opened in append mode for writing
i (immutable) File cannot be modified, deleted, or renamed
A (no atime updates) Access time is not modified when file is read
c (compressed) File is compressed on disk (transparently)
d (no dump) File is not backed up by dump command
e (extent format) File uses extents for mapping blocks

Practical Uses

Immutable Files Protect critical configuration files from accidental changes
Append-Only Secure log files from tampering while allowing new entries
No ATIME Improve performance by reducing disk writes
Compression Save disk space for rarely accessed files

Security Considerations

Important Notes

Security Limitations

File attributes provide protection against accidental changes but aren't a security feature. A user with root access can remove the attributes.

Best Practices
  • Use attributes to protect against accidental deletion/modification
  • Combine with proper file permissions for layered security
  • Document which files have special attributes
  • Regularly audit file attributes on critical systems

Common Use Cases

/etc/passwd, /etc/shadow Often made immutable on production systems
Log files Append-only to prevent tampering with existing entries
System binaries Immutable to prevent malware modification
Configuration files Immutable after initial setup to prevent drift
