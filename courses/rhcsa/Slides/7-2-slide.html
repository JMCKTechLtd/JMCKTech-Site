4 Change File Context Current Step

Finally, let's change a file context using semanage and apply it with restorecon.

semanage fcontext -a -t httpd_sys_content_t "/custom(/.*)?"

Adds a new file context rule for the /custom directory

restorecon -Rv /custom

Applies the new context recursively and verbosely

What Policy management tool
Why To modify SELinux settings persistently

5 SELinux Contexts Explained Current Step

Let's understand the SELinux context format:

system_u:object_r:passwd_file_t:s0

This context has four components:

  • User: system_u (system user)
  • Role: object_r (object role)
  • Type: passwd_file_t (file type)
  • Level: s0 (sensitivity level)

SELinux uses these contexts to enforce access control policies.

[user@lab ~]$ semanage fcontext -a -t httpd_sys_content_t "/custom(/.*)?" Added file context for /custom(/.*)? [user@lab ~]$ restorecon -Rv /custom Relabeled /custom from default_t to httpd_sys_content_t Step 4 completed. All requirements met. 6 SELinux Best Practices Current Step

Important SELinux management tips:

  • Use setenforce for temporary changes during troubleshooting
  • Use semanage for persistent policy changes
  • Always verify changes with getenforce and ls -Z
  • Use Permissive mode to test new configurations before enforcing
  • Check SELinux logs in /var/log/audit/audit.log for issues

Remember: SELinux provides an additional layer of security beyond standard Linux permissions.
