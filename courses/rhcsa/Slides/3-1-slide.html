2 Creating MBR Partitions with fdisk Current Step

Now let's create a partition using the traditional fdisk tool.

Type the command shown below:

sudo fdisk /dev/sdb

Once in fdisk, follow these steps:

  1. Type n (for new partition)
  2. Type p (for primary)
  3. Type 1 (for partition number)
  4. Accept defaults for first/last sectors
  5. Type w (to write changes)

Understanding fdisk

What Traditional MBR partition editor
Why For disks ≤2TB with MBR partition tables
Where Works with MBR-formatted disks
When Legacy systems or small disks

How it works: fdisk is interactive. Common commands: n (new), d (delete), p (print), w (write), q (quit). Changes take effect after writing.

3 Creating GPT Partitions with gdisk Current Step

Now let's create a GPT partition using gdisk.

Type the command shown below:

sudo gdisk /dev/sdc

Once in gdisk, follow these steps:

  1. Type n (for new partition)
  2. Accept defaults for partition number, first/last sectors
  3. Set type: 8200 (Linux swap)
  4. Type w (to write changes)

Understanding gdisk

What GPT partition editor (modern alternative)
Why For disks >2TB or modern systems
Where Works with GPT-formatted disks
When UEFI systems or large disks

How it works: Similar to fdisk but for GPT. Common commands: n, d, p, w, q. Type codes: 8300 (Linux filesystem), 8200 (Linux swap).

4 Using parted for Advanced Partitioning Current Step

Finally, let's use parted for more advanced partitioning operations.

Type the command shown below:

sudo parted /dev/sdd

Once in parted, follow these steps:

  1. Type mklabel gpt
  2. Type mkpart primary ext4 1MiB 500MiB
  3. Type print to verify
  4. Type quit to exit

Understanding parted

What Advanced partition manipulation tool
Why Scriptable and supports both MBR/GPT
Where Useful for precise partition sizing
When Automation or complex partitioning

How it works: parted allows precise control. Commands: mklabel (create partition table), mkpart (create partition), rm (delete), print (show info). Changes take effect immediately.
