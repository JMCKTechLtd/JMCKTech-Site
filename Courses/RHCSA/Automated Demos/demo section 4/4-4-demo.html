Welcome to the RHCSA Links Lab. In this lab, you will learn the differences between hard and soft links.

1 Create Original File Current Step

First, let's create a file with content.

Create file:
echo "This is original content" > original_file.txt
Verify:
ls -l original_file.txt

Understanding Inodes

How it works: Every file has an inode number. Hard links share the same inode, while soft links point to the filename.

2 Create Hard Link Pending

Now let's create a hard link.

Create hard link:
ln original_file.txt hard_link.txt
View inodes:
ls -li original_file.txt hard_link.txt

Understanding Hard Links

How it works: Hard links share the same inode. Deleting the original file doesn't affect the hard link. Can't cross filesystems.

3 Create Soft Link Pending

Now let's create a soft (symbolic) link.

Create soft link:
ln -s original_file.txt soft_link.txt
View inodes:
ls -li original_file.txt soft_link.txt

Understanding Soft Links

How it works: Soft links have different inodes and point to the filename. They break if the original is deleted. Can cross filesystems.

4 Test Link Behavior Pending

Now let's delete the original and see what happens.

Delete original:
rm original_file.txt
Test hard link:
cat hard_link.txt
Test soft link:
cat soft_link.txt

Understanding Link Behavior

How it works: Hard link still works (same inode). Soft link breaks (points to non-existent file).
