Welcome to the RHCSA File Permissions Lab. In this lab, you will learn how to manage file permissions and ownership.

1 View File Permissions Current Step

First, let's view the current permissions of a file.

Type this command exactly:
ls -l /var/tmp/permission_lab/file1.txt

Understanding File Permissions

How it works: The ls -l command displays permissions in the format: -rwxrwxrwx (owner, group, others). r=read, w=write, x=execute.

2 Change File Permissions Pending

Now let's change the file permissions using chmod.

First, change permissions:
chmod 755 /var/tmp/permission_lab/file1.txt
Then verify the change:
ls -l /var/tmp/permission_lab/file1.txt

Understanding chmod

How it works: chmod uses numeric (755) or symbolic (u+x) modes. 755 = rwxr-xr-x (owner=7, group=5, others=5).

3 Change File Ownership Pending

Now let's change the file owner and group.

Change owner and group:
sudo chown root:root /var/tmp/permission_lab/file2.txt
Then verify:
ls -l /var/tmp/permission_lab/file2.txt

Understanding chown

How it works: chown user:group file changes ownership. Use user: to change only user, :group for only group.

4 Change Group Ownership Pending

Now let's use chgrp to change only the group.

Change group:
sudo chgrp wheel /var/tmp/permission_lab/file3.txt
Then verify:
ls -l /var/tmp/permission_lab/file3.txt

Understanding chgrp

How it works: chgrp changes only the group ownership. Useful when you don't want to change the user owner.

5 Configure umask Pending

Finally, let's check and configure the umask.

Check current umask:
umask
Set umask:
umask 0022

Understanding umask

How it works: umask sets default permissions for new files. 0022 means new files get 644 (666-022) and directories get 755 (777-022).
