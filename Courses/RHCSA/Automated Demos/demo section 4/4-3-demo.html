Welcome to the RHCSA File Attributes Lab. In this lab, you will learn how to work with special file attributes.

1 View File Attributes Current Step

First, let's view the attributes of an existing file.

Type this command exactly:
lsattr /etc/passwd

Understanding lsattr

How it works: lsattr lists file attributes. Common attributes: i (immutable), a (append-only), A (no atime updates).

2 Set Immutable Attribute Pending

Now let's create a file and make it immutable.

Create file:
touch testfile
Set immutable:
sudo chattr +i testfile
Verify:
lsattr testfile

Understanding Immutable Attribute

How it works: The +i flag makes a file immutable. It cannot be modified, deleted, or renamed, even by root.

3 Set Append-Only Attribute Pending

Now let's create a log file and make it append-only.

Create file:
touch audit.log
Set append-only:
sudo chattr +a audit.log
Verify:
lsattr audit.log

Understanding Append-Only Attribute

How it works: The +a flag allows data to be appended but prevents modification or deletion. Perfect for log files.

4 Remove Attributes Pending

Finally, let's remove the attributes.

Remove immutable:
sudo chattr -i testfile
Remove append-only:
sudo chattr -a audit.log
Verify:
lsattr testfile audit.log

Understanding Attribute Removal

How it works: Use chattr -i or chattr -a to remove attributes. Only root can remove these attributes.
