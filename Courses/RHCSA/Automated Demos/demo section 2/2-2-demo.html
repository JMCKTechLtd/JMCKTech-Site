Welcome to the RHCSA Runlevels/Targets Lab. In this lab, you will learn how to manage system states using systemd targets.

1 Check Current Target Current Step

First, let's check the current default target.

Type this command exactly:
systemctl get-default

Understanding Systemd Targets

How it works: The systemctl get-default command shows the default target that boots automatically. Common targets include graphical.target (GUI with networking), multi-user.target (CLI with networking), rescue.target (single-user mode), and emergency.target (most minimal).

2 Change to Multi-User Target Pending

Now let's change to the multi-user target.

Type this command exactly:
systemctl isolate multi-user.target

Understanding Target Isolation

How it works: isolate stops services not needed for the new target and starts required ones. Multi-user target provides a console interface with networking but no GUI.

3 Change to Graphical Target Pending

Now let's change back to the graphical target.

Type this command exactly:
systemctl isolate graphical.target

Understanding Graphical Target

How it works: This target starts the display manager (GDM, SDDM, etc.) and all GUI-related services. It consumes more resources than multi-user target.

4 Set Default Target Pending

Now let's set the multi-user target as the default and verify the change.

First, type this command:
systemctl set-default multi-user.target
Then verify with:
systemctl get-default

Understanding Persistent Changes

How it works: set-default creates a symlink from the target to /etc/systemd/system/default.target. This affects future boots but not the current session.

5 Rescue Mode Pending

Finally, let's explore rescue mode and exit it.

First, type this command:
systemctl rescue
Then exit with:
exit

Understanding Rescue Mode

How it works: Rescue mode provides a single-user environment for system recovery. Use exit to return to normal system operation.
