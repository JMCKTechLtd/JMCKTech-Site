Welcome to the RHCSA Boot, Reboot & Shutdown Lab. In this lab, you will learn how to control system states using systemctl commands.

1 System Boot Process Current Step

First, let's examine the system boot process by listing all active systemd targets.

Type this command exactly:
systemctl list-units --type target

Understanding the Boot Process

How it works: The command shows all active systemd targets (similar to runlevels). The default target (usually graphical.target or multi-user.target) determines which services start at boot.

2 Change Default Target Pending

Now let's check the current default target and change it to multi-user.target.

First, type this command:
systemctl get-default
Then, type this command:
systemctl set-default multi-user.target

Understanding System Targets

How it works: get-default shows the current default target, while set-default changes it. Multi-user target provides a console interface without GUI.

3 System Reboot Pending

Now let's practice rebooting the system.

Type this command exactly:
systemctl reboot

Understanding System Reboot

How it works: The command cleanly stops services, unmounts filesystems, and restarts the system. Similar to shutdown -r now but more modern.

4 System Shutdown Pending

Now let's practice shutting down the system.

Type this command exactly:
systemctl poweroff

Understanding System Shutdown

How it works: The command cleanly stops services, unmounts filesystems, and powers off the system. Similar to shutdown -h now but more modern.

5 Emergency Procedures Pending

Finally, let's practice emergency procedures for system recovery.

First, type this command:
systemctl rescue
Then, type this command:
systemctl emergency

Understanding Rescue Modes

How it works: Rescue mode provides a single-user environment with network, while emergency mode provides the most basic environment without network. Use these for system recovery when normal boot isn't possible.
