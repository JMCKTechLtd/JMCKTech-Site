First, let's examine file attributes on an existing system file:

This command displays the Access Control List for the /etc/passwd file.

Common File Attributes

Attribute Description
a (append only) File can only be opened in append mode
i (immutable) Cannot be modified, deleted, or renamed
A (no atime) Access time not modified when read

2 Making a File Immutable Current Step

Now let's create a test file and make it immutable:

touch testfile
sudo chattr +i testfile
lsattr testfile

Immutable Files

The +i attribute makes a file immutable. Even root cannot modify or delete it until the attribute is removed. This is useful for protecting critical system files.

Security Note: Immutable files provide protection against accidental changes but aren't a security feature. A user with root access can remove the attribute.

3 Append-Only Files Current Step

Now let's create a log file and set it to append-only mode:

touch audit.log
sudo chattr +a audit.log
lsattr audit.log

Append-Only Files

The +a attribute allows data to be appended but prevents modifying existing content. Useful for log files where you want to ensure data isn't altered after writing.

Use Case: Protect log files or audit trails from tampering while still allowing new entries to be added.

4 Removing Attributes Current Step

Finally, let's remove the attributes we set earlier:

sudo chattr -i testfile
sudo chattr -a audit.log
lsattr testfile audit.log

Managing Attributes

Use chattr - to remove attributes. You can combine operations like chattr -i+a file to remove immutable and add append-only in one command.

Warning: Some attributes (like 'c' for compression) cannot be removed after being set. Be certain before applying attributes.

Key Takeaways

lsattr: View file attributes on ext2/3/4 filesystems

chattr +i: Make files immutable (cannot be modified/deleted)

chattr +a: Make files append-only (protect existing content)

chattr -: Remove attributes when needed

Remember: File attributes provide an additional layer of protection beyond standard permissions, useful for system security and compliance.
