Understanding SELinux Contexts

SELinux contexts are security labels applied to files and processes that define access controls. Contexts follow the format: user:role:type:level. The most important part is the type which defines access rules.

Why Defines access controls for SELinux

2 Change File Context Temporarily Current Step

Now let's temporarily change the SELinux context of a file using chcon.

Type the command shown below:

chcon -t httpd_sys_content_t /mydata/web_file.txt

Then verify the change with:

ls -Z /mydata/web_file.txt

Understanding chcon

The chcon command changes the SELinux context directly on the file. Changes made with chcon may be lost if the file system is relabeled or if restorecon is run.

What Changes file context directly
Why Quick context changes for testing
Where On files needing temporary context changes
When Before establishing permanent policy

3 Restore Default File Contexts Current Step

Now let's restore the default SELinux context using restorecon.

Type the command shown below:

restorecon -v /mydata/web_file.txt

Understanding restorecon

The restorecon command resets the SELinux context based on the system policy. The -v option shows verbose output. This is the preferred way to set contexts as it uses the policy database.

What Restores default SELinux contexts
Why Ensures proper security labeling
Where On files with incorrect contexts
When After system changes or policy updates

4 Set Permanent File Context Current Step

Now let's set a permanent SELinux context using semanage fcontext.

Type the command shown below:

semanage fcontext -a -t httpd_sys_content_t "/mydata(/.*)?"

Then apply the new policy with:

restorecon -Rv /mydata

Understanding semanage fcontext

The semanage fcontext command adds a rule to the policy. The -a adds, -t specifies type, and the regex pattern "/mydata(/.*)?" matches the directory and all contents. restorecon -R recursively applies the new policy.

What Manages SELinux policy rules
Why Creates persistent context changes
Where In policy for custom file locations
When Setting up services with non-standard paths

5 Modify SELinux Booleans Current Step

Finally, let's work with SELinux booleans to adjust policy behavior.

First, view HTTP-related booleans:

getsebool -a | grep httpd

Then set a boolean permanently:

setsebool -P httpd_can_network_connect_db on

Understanding SELinux Booleans

Booleans are toggle switches for SELinux policies that allow runtime policy adjustments without custom rules. getsebool shows current values. setsebool with -P makes changes persistent across reboots.

What Toggle switches for SELinux policies
Why Adjust policy behavior without custom rules
Where System-wide policy settings
When Enabling/disabling specific features
