2 Basic Connectivity Testing with ping Current Step

First, let's test basic network connectivity using the ping command.

Type this command exactly:

ping -c 4 127.0.0.1
Understanding ping
What Tests basic network connectivity (ICMP protocol)
Why Verify if a host is reachable and measure latency
Where Works across all network-connected systems
When First step in network troubleshooting

How it works: ping sends ICMP Echo Request packets and listens for Echo Reply packets. The -c option specifies the number of packets to send (4 in this case). 127.0.0.1 is the loopback address (your own machine).

[user@lab ~]$ ping -c 4 127.0.0.1 PING 127.0.0.1 (127.0.0.1) 56(84) bytes of data. 64 bytes from 127.0.0.1: icmp_seq=1 ttl=64 time=0.034 ms 64 bytes from 127.0.0.1: icmp_seq=2 ttl=64 time=0.042 ms 64 bytes from 127.0.0.1: icmp_seq=3 ttl=64 time=0.039 ms 64 bytes from 127.0.0.1: icmp_seq=4 ttl=64 time=0.041 ms --- 127.0.0.1 ping statistics --- 4 packets transmitted, 4 received, 0% packet loss, time 3077ms rtt min/avg/max/mdev = 0.034/0.039/0.042/0.003 ms Step 2 completed successfully. [user@lab ~]$ Next command: [user@lab ~]$ ss -tuln
[user@lab ~]$

3 Checking Active Connections with ss Current Step

Now let's check active network connections and listening ports using the ss command.

Type this command exactly:

ss -tuln
Understanding ss (Socket Statistics)
What Modern replacement for netstat (shows socket info)
Why View listening ports and active connections
Where Useful on any Linux system
When Troubleshooting network services

How it works: -t shows TCP, -u shows UDP, -l shows listening ports, -n shows numeric addresses. Output shows protocol, local address:port, and state.

Common Ports to Recognize
22 SSH
80 HTTP
443 HTTPS
53 DNS
[user@lab ~]$ ss -tuln Netid State Recv-Q Send-Q Local Address:Port Peer Address:Port tcp LISTEN 0 128 0.0.0.0:22 0.0.0.0:* tcp LISTEN 0 128 127.0.0.1:631 0.0.0.0:* udp UNCONN 0 0 0.0.0.0:5353 0.0.0.0:* Step 3 completed. Moving to step 4. [user@lab ~]$ Next: nc -l 12345 (first terminal)

4 Advanced Testing with netcat (nc) Current Step

Now we'll use netcat to create a simple TCP connection between two terminals.

First terminal - Type this command:

nc -l 12345
Understanding netcat (nc)
What "Swiss army knife" of networking
Why Test arbitrary TCP/UDP connections
Where Useful for testing firewalls and services
When Need to verify if a port is truly accessible

How it works: In this lab, we create a simple listener on port 12345 (-l) and connect to it from another terminal. Any text sent from one side appears on the other.

[user@lab ~]$ nc -l 12345 Listening on 0.0.0.0 12345 Netcat listener started successfully. [user@lab ~]$ Open a second terminal and connect to this listener.

7 Final Network Verification Current Step

Now let's check for active HTTP connections to complete our network testing.

Type this command exactly:

ss -t sport = :80

This command filters for TCP connections with source port 80 (HTTP).
