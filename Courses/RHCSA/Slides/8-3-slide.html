First, let's create a container with a volume mount to persist data beyond the container lifecycle.

Type the command shown below:

podman run -v /tmp/container-data:/data:Z alpine touch /data/testfile
What Persistent storage for containers
Why Preserve data beyond container lifecycle
Where Host directories or volumes
When When data persistence is needed

How it works: The -v flag mounts a host directory (/tmp/container-data) into the container (/data). The :Z option applies proper SELinux context. This command creates a test file in the mounted directory.

Now let's explore container networking by running a web server container with port mapping.

Type the commands shown below:

podman run -d --name web -p 8080:80 nginx
curl http://localhost:8080
What Network isolation and port mapping
Why Secure communication between containers/host
Where Virtual networks and port mappings
When For multi-container apps or external access

How it works: The -p flag maps host port 8080 to container port 80. -d runs in detached mode. curl verifies the nginx web server is accessible.

Finally, let's inspect the container's network configuration and list available networks.

What Virtual networks for containers
Why Isolate and connect container communications
Where Managed by Podman/container runtime
When For multi-container applications

How it works: podman inspect shows the container's network details. podman network ls lists available networks. By default, containers use a bridge network with NAT.
