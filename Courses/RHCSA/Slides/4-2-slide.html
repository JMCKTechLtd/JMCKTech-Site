3 Set ACL for a User Current Step

Create a test file and set an ACL for a specific user:

touch testfile
setfacl -m u:student:rw testfile
getfacl testfile

These commands create a test file, grant read-write permissions to user 'student', and display the updated ACL.

[user@lab ~]$ getfacl /etc/passwd # file: etc/passwd # owner: root # group: root user::rw- group::r-- other::r-- Step 2 completed. Moving to step 3. [user@lab ~]$ Type: touch testfile then setfacl -m u:student:rw testfile

4 Set ACL for a Group Current Step

Now set an ACL for a group on our test file:

setfacl -m g:developers:rx testfile
getfacl testfile

These commands grant read-execute permissions to the 'developers' group and display the updated ACL.

[user@lab ~]$ touch testfile [user@lab ~]$ setfacl -m u:student:rw testfile [user@lab ~]$ getfacl testfile # file: testfile # owner: user # group: user user::rw- user:student:rw- group::r-- mask::rw- other::r-- Step 3 completed. 6 Default ACLs Current Step

Finally, configure default ACLs on a directory:

These commands create a test directory, set a default ACL for user 'student', and display the directory's ACL.

7 Using Default ACLs Current Step

Default ACLs are inherited by new files and directories created within the directory.

Notice how the default ACL entries appear when viewing the directory's ACL.

[user@lab ~]$ mkdir testdir [user@lab ~]$ setfacl -m d:u:student:rwx testdir [user@lab ~]$ getfacl testdir # file: testdir # owner: user # group: user user::rwx group::r-x other::r-x default:user::rwx default:user:student:rwx default:group::r-x default:mask::rwx default:other::r-x Default ACLs will be inherited by new files/directories in this directory.
