Lab Instructions

Lab Objective
Learn to manage Logical Volume Manager (LVM) by creating physical volumes, volume groups, and logical volumes, then creating filesystems and extending storage.

Time Estimate: 20–25 minutes

LVM Configuration Requirements

Create Physical Volume: sudo pvcreate /dev/sdb
Display PV information: sudo pvdisplay
Create Volume Group: sudo vgcreate vg_data /dev/sdb
Display VG information: sudo vgdisplay vg_data
Create Logical Volume: sudo lvcreate -n lv_app -L 2G vg_data
Display LV information: sudo lvdisplay /dev/vg_data/lv_app
Create filesystem: sudo mkfs.xfs /dev/vg_data/lv_app
Verify filesystem: sudo blkid /dev/vg_data/lv_app
Extend LV: sudo lvextend -L +1G /dev/vg_data/lv_app
Grow filesystem: sudo xfs_growfs /dev/vg_data/lv_app

1 Physical Volumes (PVs) Current Step

First, let's create a physical volume from the disk device.

Type the command shown below:

sudo pvcreate /dev/sdb

Then display the physical volume information:

Understanding Physical Volumes

What Physical storage devices prepared for LVM
Why Foundation for LVM storage management
Where Typically disk partitions or whole disks
When First step in LVM configuration

How it works: pvcreate initializes a device for LVM. pvdisplay shows PV details including size, free space, and VG membership.

Now let's create a volume group from the physical volume.

Type the command shown below:

Then display the volume group information:

Understanding Volume Groups

What Pools of PVs that provide storage for LVs
Why Combine multiple PVs into flexible storage pools
Where Created from one or more PVs
When After PV creation, before LV creation

How it works: vgcreate combines PVs into a VG. vgdisplay shows VG details including size, free space, and PV composition. VGs can be extended by adding more PVs.

Now let's create a logical volume from the volume group.

Then display the logical volume information:

Understanding Logical Volumes

What Virtual partitions carved from VGs
Why Flexible, resizable storage units
Where Created within VGs
When After VG creation, before filesystem creation

How it works: lvcreate allocates space from a VG to create an LV. lvdisplay shows LV details including size, VG source, and device path. LVs can be resized as needed.

Now let's create a filesystem on the logical volume.

Type the command shown below:

sudo mkfs.xfs /dev/vg_data/lv_app

Then verify the filesystem:

sudo blkid /dev/vg_data/lv_app

Understanding Filesystems on LVs

What Filesystem creation on LVs
Why Make storage usable by the OS
Where On top of LVM logical volumes
When Final step before mounting

How it works: mkfs.xfs creates an XFS filesystem on the LV. blkid shows filesystem details including UUID. Other filesystems like ext4 can also be used.

[root@lab ~]# sudo lvcreate -n lv_app -L 2G vg_data Logical volume "lv_app" created. 5 Extending LVM Current Step

Finally, let's extend the logical volume and grow the filesystem.

Type the command shown below:

sudo lvextend -L +1G /dev/vg_data/lv_app

Then grow the filesystem to use the new space:

sudo xfs_growfs /dev/vg_data/lv_app

Understanding LVM Extension

What Online storage expansion
Why Increase capacity without downtime
Where On existing LVs with free space
When When more storage is needed

How it works: lvextend increases LV size (requires VG free space). xfs_growfs expands XFS filesystem to use new space. For ext4, use resize2fs instead.

[root@lab ~]# sudo mkfs.xfs /dev/vg_data/lv_app meta-data=/dev/vg_data/lv_app isize=512 agcount=4, agsize=131072 blks = sectsz=512 attr=2, projid32bit=1 = crc=1 finobt=1, sparse=1, rmapbt=0 = reflink=1 data = bsize=4096 blocks=524288, imaxpct=25 = sunit=0 swidth=0 blks naming =version 2 bsize=4096 ascii-ci=0, ftype=1 log =internal log bsize=4096 blocks=2560, version=2 = sectsz=512 sunit=0 blks, lazy-count=1 realtime =none extsz=4096 blocks=0, rtextents=0 [root@lab ~]# sudo blkid /dev/vg_data/lv_app /dev/vg_data/lv_app: UUID="a1b2c3d4-e5f6-7890" TYPE="xfs" Step 4 completed.
