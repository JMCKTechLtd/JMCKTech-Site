Welcome to the RHCSA Runlevels/Targets Lab. In this lab, you will learn how to manage system states using systemd targets.

Follow the steps in order. You must complete each step before moving to the next one.

1 Check Current Target Current Step

First, let's check the current default target.

Type this command exactly:
systemctl get-default

Understanding Systemd Targets

What Systemd targets replace traditional runlevels
Why To manage system states and services
Where Defined in /usr/lib/systemd/system/
When During boot or system state changes

How it works: The systemctl get-default command shows the default target that boots automatically. Common targets include:

Common Targets
  • graphical.target - GUI with networking (runlevel 5)
  • multi-user.target - CLI with networking (runlevel 3)
  • rescue.target - Single-user mode (runlevel 1)
  • emergency.target - Emergency shell (most minimal)

2 Change to Multi-User Target Pending

Now let's change to the multi-user target.

Type this command exactly:
systemctl isolate multi-user.target

Understanding Target Isolation

What Switching system state immediately
Why For maintenance or performance
Where Affects all running services
When When you need CLI-only access

How it works: isolate stops services not needed for the new target and starts required ones. Multi-user target provides a console interface with networking but no GUI.

Note: Since you're already root in this lab environment, you don't need to use sudo.

3 Change to Graphical Target Pending

Now let's change back to the graphical target.

Type this command exactly:
systemctl isolate graphical.target

Understanding Graphical Target

What Desktop environment target
Why For GUI-based workstations
Where Requires Xorg/Wayland and display manager
When Normal desktop usage

How it works: This target starts the display manager (GDM, SDDM, etc.) and all GUI-related services. It consumes more resources than multi-user target.

4 Set Default Target Pending

Now let's set the multi-user target as the default.

Type this command exactly:
systemctl set-default multi-user.target

Then verify the change:

systemctl get-default

Understanding Persistent Changes

What Changes boot-time target
Why For servers or headless systems
Where Creates symlink in /etc/systemd/system/
When When changing system purpose

How it works: set-default creates a symlink from the target to /etc/systemd/system/default.target. This affects future boots but not the current session.

5 Rescue Mode Pending

Finally, let's explore rescue mode.

Type this command exactly:
systemctl rescue

After seeing the simulated output, exit rescue mode:

exit

Understanding Rescue Mode

What Minimal environment with networking
Why For system recovery operations
Where Mounts filesystems read-only by default
When When system won't boot normally

How it works: Rescue mode provides a single-user root shell with minimal services running. Warning: In a real system, this would actually switch modes.

Exam Note: Know how to boot into rescue mode from GRUB for the RHCSA exam.
