Welcome to the RHCSA Hostname Resolution Lab. In this lab, you will configure hostname resolution by editing system files and testing the configuration.

Follow the steps in order. You must complete each step before moving to the next one.

1 View Current Hostname Resolution Current Step

First, let's view the current hostname resolution configuration.

Type this command exactly:
cat /etc/hosts

Understanding /etc/hosts

What Static hostname resolution file
Why Maps IP addresses to hostnames locally
Where /etc/hosts
When Checked before DNS queries

Format: Each line contains an IP address followed by hostnames (primary first, then aliases).

127.0.0.1 localhost localhost.localdomain localhost4 localhost4.localdomain4 ::1 localhost localhost.localdomain localhost6 localhost6.localdomain6

2 Add a Host Entry Pending

Now let's add a static host entry to the hosts file.

Type this command exactly:
sudo nano /etc/hosts

Add this line to the file:

192.168.1.100 server1.example.com server1

To save and exit nano:

  1. Press Ctrl+O to write the file
  2. Press Enter to confirm the filename
  3. Press Ctrl+X to exit
Verify your changes:
cat /etc/hosts

Editing System Files

What System configuration files
Why Need root privileges to modify
Where Typically in /etc
When When configuring system settings

How it works: sudo elevates privileges. nano is a simple text editor. After saving, verify changes with cat.

3 Test Hostname Resolution Pending

Now let's test that our hostname resolution is working correctly.

Type these commands one by one:
ping -c 3 server1
ping -c 3 server1.example.com

Testing Resolution

What Verification tools
Why Confirm entries work correctly
Where Available on all Linux systems
When After modifying resolution files

How it works: ping tests network connectivity. The -c 3 option sends 3 packets. Both hostname forms should resolve to the same IP.

4 Configure DNS Resolution Pending

Now let's configure DNS resolution by editing the resolv.conf file.

First, view the current configuration:
cat /etc/resolv.conf
Then edit the file:
sudo nano /etc/resolv.conf

Add these lines to the file:

nameserver 8.8.8.8 nameserver 8.8.4.4

To save and exit nano:

  1. Press Ctrl+O to write the file
  2. Press Enter to confirm the filename
  3. Press Ctrl+X to exit
Verify your changes:
cat /etc/resolv.conf

Understanding /etc/resolv.conf

What DNS resolver configuration
Why Specifies DNS servers to use
Where /etc/resolv.conf
When For all DNS queries not in /etc/hosts

Format: Each nameserver line specifies a DNS server (max 3). They're queried in order.

# Generated by NetworkManager search example.com nameserver 8.8.8.8 nameserver 8.8.4.4

5 Test DNS Resolution Pending

Finally, let's test that our DNS configuration is working correctly.

Type these commands one by one:
nslookup google.com
dig redhat.com

DNS Testing Tools

What DNS diagnostic utilities
Why Verify DNS configuration works
Where May need to install (bind-utils)
When After changing DNS settings

How they work: nslookup provides basic DNS queries. dig gives detailed DNS information including TTL and authoritative servers.
