Welcome to the RHCSA Package Management Lab. In this lab, you will learn to use rpm, dnf, and yum commands to manage software packages on a Red Hat Enterprise Linux system.

Follow the steps in order. You must complete each step before moving to the next one.

1 Querying Packages with rpm Current Step

First, let's query installed packages using the rpm command.

Type this command exactly:
rpm -qa | grep bash

This command will list all installed packages containing "bash" in their name.

Understanding rpm

What Low-level package manager (RPM Package Manager)
Why Install, query, verify, update, and remove packages
Where Works with local .rpm files
When When you need direct package manipulation

How it works: rpm -qa lists all installed packages. Piping to grep filters results. Common options: -q (query), -i (info), -e (erase), -V (verify).

2 Searching Packages with dnf Pending

Now let's search for packages in repositories using the dnf command.

Type this command exactly:
dnf search nano

This command will search repositories for packages related to "nano".

Understanding dnf

What Next-generation package manager (Dandified YUM)
Why Resolves dependencies automatically
Where Works with repositories
When Preferred for package management in RHEL 8+

How it works: dnf search finds packages in repositories. Other common commands: install, remove, update, info, list. Uses /etc/dnf/dnf.conf for configuration.

3 Installing a Package Pending

Now let's install a package using dnf.

Type this command exactly:
sudo dnf install -y nano

This command will install the nano text editor with administrative privileges.

Understanding Package Installation

What Adding software to the system
Why To make new software available
Where From configured repositories
When When you need new functionality

How it works: sudo elevates privileges. -y automatically answers yes to prompts. DNF resolves and installs dependencies automatically. Packages are installed to standard locations (e.g., /usr/bin/).

4 Removing a Package Pending

Now let's remove a package using dnf.

Type this command exactly:
sudo dnf remove -y nano

This command will remove the nano text editor and any dependencies that are no longer needed.

Understanding Package Removal

What Uninstalling software from the system
Why To free space or remove unneeded software
Where From installed packages
When When software is no longer needed

How it works: DNF removes the specified package and any dependencies that are no longer needed. The -y flag confirms the action automatically. Always verify you're removing the correct package.

5 Updating Packages Pending

Finally, let's update the system packages using dnf.

Type this command exactly:
sudo dnf update -y

This command will update all installed packages to their latest available versions.

Understanding System Updates

What Applying available updates
Why Security patches and bug fixes
Where From configured repositories
When Regularly for security and stability

How it works: DNF checks repositories for newer versions of installed packages. The update command upgrades all packages with available updates. For a single package: dnf update package_name.
