Welcome to the RHCSA String Search in System Files Lab. In this lab, you will search for specific strings in system configuration files and save the results.

Follow the steps in order. You must complete each step before moving to the next one.

1 Search for nologin in /etc/passwd Current Step

First, let's search for the string "nologin" in the /etc/passwd file and save the results.

Type this command exactly:
grep nologin /etc/passwd > /root/strings

This command:

  • Searches for "nologin" in /etc/passwd
  • Redirects the output to /root/strings
  • Preserves the original line order
  • Only includes lines that contain the search string

Understanding /etc/passwd

The /etc/passwd file contains user account information. Each line represents one user and has seven fields separated by colons:

  1. Username: User login name
  2. Password: 'x' indicates shadow passwords
  3. UID: User ID number
  4. GID: Group ID number
  5. GECOS: User description
  6. Home directory: User's home directory
  7. Login shell: User's default shell

The "nologin" shell prevents interactive login for system accounts.

2 Verify Output File Pending

Now let's verify that the output file was created correctly with the matching lines.

Type this command exactly:
cat /root/strings

This should display all lines from /etc/passwd that contain "nologin".

You can also check the file size:
ls -lh /root/strings

This shows the file size and permissions.

3 Count Matching Lines Pending

Let's count how many lines matched our search criteria.

Type this command exactly:
wc -l /root/strings

This will show the number of lines in the output file, which corresponds to the number of users with nologin shell.

System Accounts with nologin

System accounts typically use /sbin/nologin or /bin/false as their shell to prevent interactive login. Common examples:

  • bin, daemon - System service accounts
  • adm, lp - Administrative accounts
  • mail, news - Service-specific accounts
  • nobody - Unprivileged account

4 Compare with Original File Pending

Finally, let's compare our results with the original file to ensure accuracy.

Type these commands to compare:
grep -c nologin /etc/passwd

Counts nologin entries directly from the source file.

head -5 /root/strings

Shows the first 5 lines of our output file.

grep nologin /etc/passwd | head -5

Shows the first 5 matching lines from the original file for comparison.

Advanced grep Options

Useful grep options for different search scenarios:

  • grep -i pattern - Case-insensitive search
  • grep -v pattern - Invert match (exclude lines)
  • grep -n pattern - Show line numbers
  • grep -c pattern - Count matches
  • grep -r pattern /dir - Recursive search
