Welcome to the RHCSA User and Group Management Lab. In this lab, you will create users and groups with specific membership requirements.

Follow the steps in order. You must complete each step before moving to the next one.

1 Create Adminuser Group Current Step

First, let's create the adminuser group that will be used as a secondary group for natasha and harry.

Type this command exactly:
groupadd adminuser

This command creates a new group called "adminuser".

Understanding Linux Groups

  • Primary Group: The default group assigned to a user
  • Secondary Group: Additional groups a user can belong to
  • groupadd: Creates a new group
  • usermod -G: Adds a user to secondary groups

2 Create Natasha User Pending

Now let's create the user natasha and add her to the adminuser group as a secondary group.

Type this command exactly:
useradd -G adminuser natasha

This command creates user natasha and adds her to the adminuser secondary group.

User Creation Options

  • -G adminuser: Specifies secondary groups
  • -s /bin/bash: Sets login shell (default)
  • -m: Creates home directory
  • -c "Comment": Adds a comment/description

3 Create Harry User Pending

Now let's create the user harry and add him to the adminuser group as a secondary group.

Type this command exactly:
useradd -G adminuser harry

This command creates user harry and adds him to the adminuser secondary group.

4 Create Sarah User Pending

Now let's create the user sarah with a non-interactive shell. She should NOT be in the adminuser group.

Type this command exactly:
useradd -s /sbin/nologin sarah

This command creates user sarah with a non-interactive shell.

Non-Interactive Shells

Users with non-interactive shells cannot log into the system:

  • /sbin/nologin: Prevents login with a message
  • /bin/false: Immediately exits without message
  • Used for system accounts and service accounts

5 Set User Passwords Pending

Now let's set the passwords for all users to "redhat".

Type these commands one by one:
echo "redhat" | passwd --stdin natasha
echo "redhat" | passwd --stdin harry
echo "redhat" | passwd --stdin sarah

These commands set the passwords without interactive prompts.

Alternative method: You can also use passwd username and type the password interactively.

6 Verify Configuration Pending

Finally, let's verify that our user and group configuration is correct.

Type these commands one by one:
id natasha

Should show natasha in adminuser group

id harry

Should show harry in adminuser group

id sarah

Should NOT show sarah in adminuser group

grep -E "(natasha|harry|sarah)" /etc/passwd

Should show user entries with correct shells

Verification Commands

After creating users and groups, always verify with these commands:

  • id username - Shows user ID and group memberships
  • groups username - Shows group memberships
  • cat /etc/passwd - Shows all user accounts
  • cat /etc/group - Shows all groups
