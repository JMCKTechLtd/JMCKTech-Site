Welcome to the RHCSA User and Group Management Lab. In this lab, you will create users and groups with specific requirements and configure directory permissions.

Follow the steps in order. You must complete each step before moving to the next one.

1 Create Admin Group Current Step

First, let's create the adminuser group.

Type this command exactly:
groupadd adminuser

This creates a new group called adminuser that we'll use for our collaborative directory.

Understanding Linux Groups

Groups in Linux are used to organize users and manage permissions. Key points:

  • Each user has a primary group
  • Users can belong to multiple secondary groups
  • Group membership controls file and directory access
  • Groups are defined in /etc/group

2 Create Users with Group Membership Pending

Now let's create users natasha and harry with adminuser as their secondary group.

Type these commands one by one:
useradd -G adminuser natasha
useradd -G adminuser harry

The -G option specifies secondary groups for the user.

User Creation Options

useradd command options:

  • -G - Specify secondary groups
  • -s - Specify login shell
  • -m - Create home directory (default in most distributions)
  • -u - Specify user ID

3 Create User with Non-Interactive Shell Pending

Now let's create user sarah with a non-interactive shell and ensure she's not in the adminuser group.

Type this command exactly:
useradd -s /sbin/nologin sarah

The /sbin/nologin shell prevents interactive login while still allowing system account functionality.

Non-Interactive Shells

Using /sbin/nologin or /bin/false as a login shell:

  • Prevents interactive login via SSH, console, etc.
  • Allows system services to run under the account
  • Useful for service accounts and system users
  • Users can still access resources via other means (FTP, Samba, etc.)

4 Set User Passwords Pending

Now let's set the passwords for all users to 'redhat'.

Type these commands one by one:
echo "redhat" | passwd --stdin natasha
echo "redhat" | passwd --stdin harry
echo "redhat" | passwd --stdin sarah

Using --stdin allows setting the password non-interactively.

Password Management

Password security considerations:

  • Use strong passwords in production
  • passwd --stdin is convenient for labs but shows password in command history
  • In production, use interactive passwd command
  • Passwords are stored encrypted in /etc/shadow

5 Create Collaborative Directory Pending

Finally, let's create the collaborative directory with proper permissions.

Type these commands one by one:
mkdir /home/admins
chgrp adminuser /home/admins
chmod 2770 /home/admins

The 2770 permissions set read/write/execute for owner and group, plus the SGID bit.

Directory Permissions and SGID

Special permission bits:

  • SUID (4000) - Execute with owner's privileges
  • SGID (2000) - Files inherit group ownership of directory
  • Sticky (1000) - Only owner can delete files
  • chmod 2770 = rwxrws--- (owner: rwx, group: rws, others: ---)
