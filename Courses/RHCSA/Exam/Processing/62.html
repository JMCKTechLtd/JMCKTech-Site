Welcome to the RHCSA Root Password Recovery Lab. In this lab, you'll learn how to regain access to a system when you've forgotten the root password.

Follow the steps in order. You must complete each step before moving to the next one.

1 Reboot the System Current Step

First, we need to reboot the system to access the GRUB boot menu.

Type this command exactly:
reboot

This will restart the system and bring up the GRUB boot menu where we can modify boot parameters.

Understanding the Boot Process

When a Linux system boots, it loads the GRUB bootloader which allows you to modify kernel parameters before the system fully starts. This is key to password recovery.

2 Interrupt Boot and Edit Kernel Parameters Pending

When the GRUB menu appears, we need to interrupt the normal boot process and edit the kernel parameters.

Follow these steps in the GRUB menu:
  1. Press e to edit the current boot entry
  2. Navigate to the line starting with linux
  3. Go to the end of this line
  4. Add rd.break to the end of the line
  5. Press Ctrl+X to boot with these parameters

Note: The rd.break parameter interrupts the boot process early and drops you into an emergency shell.

3 Remount Root Filesystem with Write Permissions Pending

In the emergency shell, the root filesystem is mounted as read-only. We need to remount it with write permissions.

Type this command exactly:
mount -o remount,rw /sysroot

This command remounts the system root filesystem with read-write permissions, allowing us to make changes.

4 Change Root to System Filesystem Pending

Now we need to change our root directory to the actual system filesystem.

Type this command exactly:
chroot /sysroot

This changes our working environment to the actual system root, allowing us to run system commands like passwd.

5 Change Root Password Pending

Now we can change the root password to "redhat".

Type this command exactly:
echo "redhat" | passwd --stdin root

This sets the root password to "redhat" without requiring interactive input.

Password Security

In a real-world scenario, you should choose a strong, unique password. For this lab exercise, we're using "redhat" for simplicity.

6 Relabel Filesystem for SELinux Pending

If SELinux is enabled (which it typically is on RHEL systems), we need to trigger a filesystem relabel to ensure proper security contexts.

Type this command exactly:
touch /.autorelabel

This creates a file that tells the system to relabel all files on the next boot, ensuring SELinux contexts are correct.

Note: The next boot will take longer as SELinux relabels all files.

7 Exit and Reboot Pending

Now we need to exit the chroot environment and reboot the system normally.

Type these commands exactly:
exit
reboot

The first exit command exits the chroot environment, and the second reboot command restarts the system normally.

8 Verify Login with New Password Pending

After the system reboots, verify that you can log in as root with the new password "redhat".

After reboot completes:
  1. At the login prompt, enter root as the username
  2. Enter redhat as the password
  3. You should successfully log into the system

Password Recovery Complete

You have successfully recovered access to a system where the root password was unknown. This is a critical skill for system administrators.
