Welcome to the RHCSA NTP Client Configuration Lab. In this lab, you will configure your system to synchronize its time with an NTP server.

Follow the steps in order. You must complete each step before moving to the next one.

1 Install Chrony NTP Client Current Step

First, let's install the chrony package, which provides the NTP client functionality.

Type this command exactly:
yum install -y chrony

Chrony is a versatile implementation of the Network Time Protocol (NTP) that performs well in various conditions.

Understanding NTP

NTP (Network Time Protocol) synchronizes computer clock times in a network. Accurate timekeeping is essential for:

  • Log file timestamps
  • Authentication services (Kerberos)
  • Database transactions
  • Distributed systems

2 Configure Chrony Pending

Now let's configure chrony to use the specified NTP server.

Type this command exactly:
vim /etc/chrony.conf

Add this line to the configuration file:

server server.domain11.example.com iburst

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Chrony Configuration

The iburst option sends a burst of packets when the service starts or if the server is unreachable, which helps speed up initial synchronization.

3 Enable and Start Chrony Service Pending

Now let's enable the chronyd service to start automatically at boot and start it immediately.

Type these commands one by one:
systemctl enable chronyd
systemctl start chronyd

These commands ensure the NTP service runs now and after system reboots.

Service Management

systemctl enable configures the service to start automatically at boot, while systemctl start starts the service immediately.

4 Verify NTP Synchronization Pending

Finally, let's verify that the NTP synchronization is working correctly.

Type this command exactly:
chronyc sources -v

This command shows the current time sources and their status. Look for the server with a '*' indicating it's synchronized.

Verification Commands

Other useful chrony commands for verification:

  • chronyc tracking - Shows system time information
  • chronyc activity - Shows number of online/offline sources
  • timedatectl status - Shows current time and NTP status
