Welcome to the RHCSA Cron Job Configuration Lab. In this lab, you will configure a scheduled task using cron to run at a specific time every day.

Follow the steps in order. You must complete each step before moving to the next one.

1 Find the Echo Command Path Current Step

First, let's find the full path to the echo command. This is important for cron jobs to ensure the command can be found when executed.

Type this command exactly:
which echo

The output should show the full path to the echo command, typically /bin/echo.

Understanding the which Command

The which command locates the executable file associated with a given command by searching the directories in the PATH environment variable.

This is important for cron jobs because cron runs with a minimal environment and may not have the same PATH as your interactive shell.

2 Edit the Crontab File Pending

Now let's edit the crontab file to add our scheduled task.

Type this command exactly:
crontab -e

This will open the crontab file in the default editor (usually vim).

Add this line to the file:

23 14 * * * /bin/echo hello

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Understanding Cron Syntax

Cron uses a specific time format with five fields:

Minute
0-59
Hour
0-23
Day of Month
1-31
Month
1-12
Day of Week
0-7 (0 and 7 = Sunday)

In our example: 23 14 * * * means:

  • Minute: 23
  • Hour: 14 (2 PM)
  • Day of Month: * (every day)
  • Month: * (every month)
  • Day of Week: * (every day of the week)

3 Verify the Cron Job Pending

Finally, let's verify that our cron job has been properly configured.

Type this command exactly:
crontab -l

This command lists the current crontab entries. You should see the line we added:

23 14 * * * /bin/echo hello

Cron Job Management

Key commands for managing cron jobs:

  • crontab -e - Edit the crontab file
  • crontab -l - List current crontab entries
  • crontab -r - Remove all crontab entries
  • systemctl status crond - Check if the cron service is running

Note: Cron jobs run with the user's environment, so make sure to use full paths to commands and scripts.
