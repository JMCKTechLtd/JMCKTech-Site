In this lab you will practice using tar to create a backup archive containing multiple directories, and verify the resulting file.

Follow the steps in order. The terminal on the right simulates the commands needed to complete an RHCSA-style task.

1 Ensure /usr/local/etc exists Current Step

Make sure the target directory for the archive exists. Create it if necessary.

Type this command exactly:
mkdir -p /usr/local/etc

Why mkdir -p?

The -p option ensures the whole path is created if missing, and does nothing (no error) if it already exists.

2 Create the backup tar archive Pending

Use tar to create an archive named /usr/local/etc/backup.tar that includes both /usr/local and /etc.

Type this command exactly:
tar -cvf /usr/local/etc/backup.tar /usr/local /etc

tar options

-c = create, -v = verbose (show files), -f = specify archive file name.

When archiving absolute paths, you may see a message like ‚ÄúRemoving leading ‚Äò/‚Äô from member names‚Äù. This is normal.

3 Verify the archive file exists Pending

Confirm that /usr/local/etc/backup.tar has been created.

ls /usr/local/etc/backup.tar

4 Check the archive file type Pending

Use the file command to confirm that the backup is recognised as a tar archive.

file /usr/local/etc/backup.tar

5 List the contents of the archive Pending

Finally, list the files stored in the archive to confirm that both directories were included.

tar -tf /usr/local/etc/backup.tar

Checking your work

Being able to verify archives is important in real systems: it confirms that the right files were captured before you rely on them as backups.
