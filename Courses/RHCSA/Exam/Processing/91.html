0 Lab Introduction Completed

Welcome to the RHCSA FTP Server Configuration Lab. In this lab, you will install and configure an FTP server with anonymous access.

Follow the steps in order. You must complete each step before moving to the next one.

1 Install vsftpd Package Current Step

First, let's install the vsftpd (Very Secure FTP Daemon) package.

Type this command exactly:
yum install -y vsftpd

This will download and install the vsftpd package along with any dependencies.

Understanding FTP Server

vsftpd is a secure, fast FTP server for UNIX-like systems. It's designed with security in mind and is the default FTP server in many Linux distributions.

2 Start FTP Service Pending

Now let's start the FTP service.

Type this command exactly:
systemctl start vsftpd

This will start the vsftpd service immediately.

3 Enable FTP Service at Boot Pending

Now let's enable the FTP service to start automatically at system boot.

Type this command exactly:
systemctl enable vsftpd

This will create the necessary symlinks to start vsftpd automatically when the system boots.

4 Configure Anonymous Access Pending

Now let's configure the FTP server to allow anonymous access.

Type this command exactly:
vim /etc/vsftpd/vsftpd.conf

Find the line that says anonymous_enable=NO and change it to:

anonymous_enable=YES

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Understanding FTP Configuration

The main configuration file for vsftpd is /etc/vsftpd/vsftpd.conf. Key parameters:

  • anonymous_enable: Controls anonymous FTP access
  • local_enable: Controls whether local users can log in
  • write_enable: Controls FTP commands that can change the filesystem

5 Restart FTP Service Pending

Now let's restart the FTP service to apply the configuration changes.

Type this command exactly:
systemctl restart vsftpd

6 Verify Configuration Pending

Finally, let's verify that our FTP server is running and configured correctly.

Type these commands one by one:
systemctl status vsftpd

Should show: active (running)

systemctl is-enabled vsftpd

Should show: enabled

grep anonymous_enable /etc/vsftpd/vsftpd.conf

Should show: anonymous_enable=YES

Verification Commands

After configuring services, always verify with these commands:

  • systemctl status - Shows service status
  • systemctl is-enabled - Checks if service starts at boot
  • grep - Verifies configuration settings
