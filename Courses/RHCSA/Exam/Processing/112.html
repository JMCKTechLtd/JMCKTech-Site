Welcome to the RHCSA Network Configuration Lab. In this lab, you will create a network connection with multiple IP addresses and configure the system hostname.

Follow the steps in order. You must complete each step before moving to the next one.

1 Check Existing Network Connections Current Step

First, let's check the existing network connections to understand the current configuration.

Type this command exactly:
nmcli connection show

This will show all existing network connections managed by NetworkManager.

Understanding NetworkManager

NetworkManager is the standard network configuration tool in RHEL. It manages network connections, devices, and settings.

2 Create New Network Connection Pending

Now let's create a new network connection with the specified parameters.

Type this command exactly:
nmcli connection add con-name static ifname enp1s0 type ethernet ipv4.addresses 172.25.15.10/24 ipv4.gateway 172.25.15.2 ipv4.dns 172.25.15.2

This command creates a new connection named "static" with the specified IP, gateway, and DNS settings.

3 Configure Connection for Manual IP Assignment Pending

We need to configure the connection to use manual IP assignment instead of DHCP.

Type this command exactly:
nmcli connection modify static ipv4.method manual connection.autoconnect yes

This sets the connection to use static IP configuration and enables automatic connection at boot.

4 Add Secondary IP Address Pending

Now let's add the secondary IP address to the connection.

Type this command exactly:
nmcli connection modify static +ipv4.addresses 10.0.0.5/24

The + prefix adds the address to the existing list rather than replacing it.

Multiple IP Addresses

A network interface can have multiple IP addresses. This is useful for hosting multiple services or for network segmentation.

5 Activate the Connection Pending

Now let's activate the new network connection.

Type this command exactly:
nmcli connection up static

This activates the connection and applies all our configuration changes.

6 Set System Hostname Pending

Now let's set the system hostname as required.

Type this command exactly:
hostnamectl set-hostname node1.domain15.example.com

This sets both the static and transient hostname.

7 Verify Configuration Pending

Finally, let's verify that all our configurations are working correctly.

Type these commands one by one:
nmcli connection show

Should show the "static" connection as active

ip addr show enp1s0

Should show both IP addresses: 172.25.15.10 and 10.0.0.5

hostnamectl status

Should show hostname: node1.domain15.example.com

ping -c 2 172.25.15.2

Should successfully ping the gateway

Verification Commands

After network configuration, always verify with these commands:

  • nmcli connection show - Shows connection status
  • ip addr - Displays IP addresses
  • hostnamectl - Shows hostname configuration
  • ping - Tests network connectivity
