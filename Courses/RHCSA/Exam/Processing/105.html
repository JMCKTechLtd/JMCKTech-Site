0 Lab Introduction Completed

Welcome to the RHCSA Autofs Configuration Lab. In this lab, you will configure autofs to automatically mount LDAP user home directories when users access them.

Follow the steps in order. You must complete each step before moving to the next one.

1 Install Autofs Package Current Step

First, let's install the autofs package which provides the automount functionality.

Type this command exactly:
yum install -y autofs

Autofs automatically mounts filesystems when they are accessed and unmounts them after a period of inactivity.

Understanding Autofs

Autofs provides on-demand mounting of filesystems, which is particularly useful for:

  • NFS home directories
  • Removable media
  • Network shares that aren't constantly needed
  • Reducing network traffic and server load

2 Configure auto.master Pending

Now let's configure the auto.master file to define the mount point for LDAP home directories.

Type this command exactly:
vim /etc/auto.master

Add this line to the configuration file:

/rhome /etc/auto.ldap

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Auto.master Configuration

The auto.master file defines the mount points and their corresponding map files. The format is:

mount-point map-file [options]

In this case, /rhome is the mount point and /etc/auto.ldap is the map file that defines how to mount individual directories.

3 Create auto.ldap Map File Pending

Now let's create the auto.ldap map file that defines how to mount individual LDAP user home directories.

Type this command exactly:
vim /etc/auto.ldap

Add this line to the configuration file:

ldapuser11 -rw,sync host.domain11.example.com:/rhome/ldapuser11

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Map File Syntax

The map file defines the mapping between keys and mount specifications:

  • ldapuser11 - The key (subdirectory name under /rhome)
  • -rw,sync - Mount options (read-write, synchronous writes)
  • host.domain11.example.com:/rhome/ldapuser11 - NFS server and export

4 Enable and Start Autofs Service Pending

Now let's enable the autofs service to start automatically at boot and start it immediately.

Type these commands one by one:
systemctl enable autofs
systemctl start autofs

These commands ensure the autofs service runs now and after system reboots.

Service Management

The autofs service must be running for automount functionality to work. It reads the configuration files and manages the automatic mounting/unmounting of filesystems.

5 Verify Autofs Configuration Pending

Finally, let's verify that the autofs configuration is working correctly.

Type this command exactly:
su - ldapuser11

This command should successfully switch to the ldapuser11 account and automatically mount the home directory.

Verification

When you successfully switch to the LDAP user, autofs should automatically mount the home directory from the NFS server. You can also verify with:

  • ls /rhome/ - Should show available user directories
  • mount | grep /rhome - Should show mounted filesystems
  • systemctl status autofs - Should show service is running
