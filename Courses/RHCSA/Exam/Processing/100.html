Welcome to the RHCSA Cron Job Configuration Lab. In this lab, you will configure a scheduled task using the cron system.

Follow the steps in order. You must complete each step before moving to the next one.

1 Edit Natasha's Crontab Current Step

First, let's edit the crontab for user natasha to add our scheduled job.

Type this command exactly:
crontab -e -u natasha

This command opens the crontab editor for user natasha.

Understanding Crontab

  • crontab -e: Edit current user's crontab
  • crontab -e -u username: Edit another user's crontab (requires root)
  • crontab -l: List current crontab entries
  • crontab -r: Remove all crontab entries

2 Add Cron Job Entry Pending

In the crontab editor, add the following line:

23 14 * * * /bin/echo hiya

Save and exit the editor. In vim, use :wq and press Enter.

Cron Time Format

The cron time format has five fields:

  • Minute: 23 (23rd minute)
  • Hour: 14 (2:00 PM)
  • Day of Month: * (every day)
  • Month: * (every month)
  • Day of Week: * (every day of week)

So 23 14 * * * means "At 14:23 every day"

3 Verify Cron Job Pending

Now let's verify that the cron job was added correctly.

Type this command exactly:
crontab -l -u natasha

This command lists all cron jobs for user natasha.

Cron Job Verification

After adding a cron job, always verify:

  • crontab -l -u username - Lists user's cron jobs
  • Cron jobs are stored in /var/spool/cron/username
  • System cron jobs are in /etc/crontab and /etc/cron.d/

4 Check Cron Service Pending

Finally, let's ensure the cron service is running properly.

Type this command exactly:
systemctl status crond

This command shows the status of the cron daemon service.

Cron Service Management

The cron daemon (crond) must be running for cron jobs to execute:

  • systemctl status crond - Check service status
  • systemctl start crond - Start the service
  • systemctl enable crond - Enable auto-start on boot
  • systemctl restart crond - Restart the service
