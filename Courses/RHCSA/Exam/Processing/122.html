1 Edit natasha‚Äôs user crontab Current Step

From the root shell, open natasha‚Äôs user crontab in the editor:

Type this command exactly:
crontab -e -u natasha

User vs System crontab

Using crontab -e -u natasha edits the crontab owned by natasha, stored under /var/spool/cron/. This is typically what RHCSA tasks expect.

2 Add the daily 14:23 cron entry Pending

Inside the editor, add the following line to natasha‚Äôs crontab:

23 14 * * * /bin/echo hello >> /home/natasha/hello

Then save and exit the editor with :wq.

Reminder: Press i for INSERT mode, type the line, press ESC, then type :wq and press Enter.

Cron time fields

The fields are: minute hour day-of-month month day-of-week. So 23 14 * * * means ‚Äúevery day at 14:23‚Äù.

3 Verify natasha‚Äôs crontab Pending

List natasha‚Äôs crontab to confirm that the entry is present:

crontab -l -u natasha

You should see the 23 14 * * * /bin/echo hello >> /home/natasha/hello line in the output.

4 Check the crond service status Pending

Ensure the cron daemon is running so that scheduled jobs will actually execute.

Type this command exactly:
systemctl status crond.service

In a real exam you might need to start or enable the service with systemctl enable --now crond.service if it is not already active.
