Welcome to the RHCSA Group Management Lab. In this lab, you will create a new system group with a specific group ID.

Groups are used to organize users and manage permissions collectively. System administrators often create custom groups for specific purposes.

1 Create the Admin Group Current Step

Use the groupadd command to create a new group named "admin" with GID 600.

Type this command exactly:
groupadd -g 600 admin

Understanding the groupadd Command

The groupadd command creates a new group account using the values specified on the command line.

-g GID Specifies the numerical value of the group's ID
group_name The name of the new group to create

GID values from 0-499 are typically reserved for system groups, while 500+ are for user groups. GID 600 is in the user group range.

2 Verify Group Creation Pending

After creating the group, verify that it was created correctly with the proper GID.

Type this command exactly:
getent group admin

This command should return: admin:x:600:

Understanding Group Verification

The getent command displays entries from databases supported by the Name Service Switch libraries, including the group database.

The output format is: group_name:password:GID:user_list

  • admin - The group name
  • x - Placeholder for the group password (stored in /etc/gshadow)
  • 600 - The group ID (GID)
  • Empty - No users are currently members of this group

3 Alternative Verification Method Pending

You can also verify the group creation by checking the /etc/group file directly.

Type this command exactly:
grep admin /etc/group

This command should return: admin:x:600:

Understanding the /etc/group File

The /etc/group file contains group information and defines the groups to which users belong.

Each line in the file has the format: group_name:password:GID:user_list

When you create a new group with groupadd, an entry is automatically added to this file.
