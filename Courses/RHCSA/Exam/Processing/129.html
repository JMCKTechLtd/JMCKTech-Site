0 Lab Introduction & GRUB Edit (Conceptual) Completed

In the real environment, you would first:

  1. Reboot the server (for example with Ctrl+Alt+Del).
  2. At the GRUB menu, highlight the default boot entry and press e.
  3. Move to the line starting with linux, press End and append rd.break.
  4. Press Ctrl+x to boot with the modified parameters.

After this, you land at the emergency prompt:

switch_root:/#

This simulation starts from that point onward and focuses on the commands you must run to successfully reset the root password on node2.

1 Remount /sysroot as read-write Current Step

At the switch_root:/# prompt, remount /sysroot with read-write permissions so you can modify the real root filesystem.

Type this command exactly:
mount -o remount,rw /sysroot

Why remount /sysroot?

The real root filesystem is mounted read-only by default in this rescue environment. You must remount it read-write before changing passwords or creating files.

2 Chroot into /sysroot Pending

Change your root directory to /sysroot so that password changes apply to the real system:

chroot /sysroot

Your prompt will typically change to something like:

sh-4.4#

3 Set root password to kexdrams Pending

Now reset the root password to kexdrams using the non-interactive passwd --stdin method:

Type this piped command exactly:
echo kexdrams | passwd --stdin root

Non-interactive password reset

Using echo ... | passwd --stdin lets you script or quickly set a password without being prompted twice for it.

4 Create /.autorelabel for SELinux Pending

To ensure SELinux contexts are correct for the new password files, create /.autorelabel. On the next boot, SELinux will relabel the filesystem:

touch /.autorelabel

Skipping this on a system with SELinux enabled can cause login issues after changing the root password in rescue mode.

5 Exit the chroot shell Pending

Leave the chrooted environment and return to the switch_root:/# prompt:

exit

6 Exit again to continue boot & verify login Pending

Exit once more to continue the normal boot process. After the system comes up, log in as root using the new password kexdrams.

exit

This simulation will show a successful login banner when you complete this step.
