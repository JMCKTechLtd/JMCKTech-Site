Welcome to the RHCSA Repository Configuration Lab. In this lab, you will configure a YUM repository on a Red Hat Enterprise Linux system.

Follow the steps in order. You must complete each step before moving to the next one.

Important Note

Wrong YUM configuration will lead to some following questions not working properly. Approximately 4305 packages should be available when correctly configured.

1 Navigate to YUM Repository Directory Current Step

First, let's navigate to the YUM repository configuration directory.

Type this command exactly:
cd /etc/yum.repos.d/

This directory contains all the repository configuration files for your system.

Understanding YUM Repositories

YUM (Yellowdog Updater Modified) repositories are defined in .repo files within /etc/yum.repos.d/. Each repository file contains information about where to find packages and metadata.

2 Create Repository Configuration File Pending

Now let's create a repository configuration file using vim.

Type this command exactly:
vim /etc/yum.repos.d/local.repo

Once in the editor, add these exact contents to the file:

[local] name=local.repo baseurl=http://server.domain11.example.com/pub/x86_64/Server enabled=1 gpgcheck=0

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Repository Configuration Parameters

[local] Repository ID - unique identifier for the repository
name Human-readable name for the repository
baseurl URL where packages are located
enabled 1 = repository is active, 0 = disabled
gpgcheck 0 = skip package verification, 1 = verify with GPG

3 Verify Repository File Pending

Let's verify that our repository file was created correctly.

Type this command exactly:
cat /etc/yum.repos.d/local.repo

The output should match exactly what you entered in the previous step.

Alternative Configuration Method

You could also use the yum-config-manager tool:

yum-config-manager --add-repo=http://content.example.com/rhel7.0/x86-64/dvd

This would generate a file named content.example.com_rhel7.0_x86_64_dvd.repo and you would need to add gpgcheck=0.

4 Clean YUM Cache Pending

Now let's clean the YUM cache to ensure we're getting fresh metadata from our new repository.

Type this command exactly:
yum clean all

This command removes cached packages and metadata from all enabled repositories.

Why Clean the Cache?

Cleaning the cache ensures that YUM will download fresh metadata from your newly configured repository. This is important when:

  • Adding a new repository
  • Changing repository configuration
  • Suspecting outdated metadata

5 Verify Repository Configuration Pending

Finally, let's verify that our repository is working correctly. You should run BOTH verification commands.

Type these commands one by one:
yum repolist

Should show the "local" repository with approximately 4305 packages

yum list all | wc -l

Should show a large number of available packages (thousands)

Verification Commands

After configuring a repository, always verify with these commands:

  • yum repolist - Shows all enabled repositories and package counts
  • yum list available - Lists all available packages
  • yum search [package] - Searches for specific packages
  • yum info [package] - Shows detailed package information

Note: Correct configuration should show approximately 4305 packages. Wrong YUM configuration will cause following lab questions to fail!
