1 Create Admin Group Current Step

First, create the adminuser group using the groupadd command.

Type this command exactly:
groupadd adminuser

Understanding Group Creation

The groupadd command creates a new group. Groups are used to manage permissions for multiple users.

You can verify the group was created with:

grep adminuser /etc/group

2 Create User Natasha Pending

Create user natasha with adminuser as a secondary group.

Type this command exactly:
useradd -G adminuser natasha

Note: The -G option specifies supplementary groups. The user will still have a primary group with the same name as the username.

User Creation Options

  • useradd: Creates a new user account
  • -G adminuser: Adds user to supplementary group adminuser
  • Without -s option: Default shell is /bin/bash

3 Create User Harry Pending

Create user harry with adminuser as a secondary group.

Type this command exactly:
useradd -G adminuser harry

This command is similar to the previous step, creating another user with the same group membership.

4 Create User Sarah with Non-Interactive Shell Pending

Create user sarah with a non-interactive shell. Do NOT add her to the adminuser group.

Type this command exactly:
useradd -s /sbin/nologin sarah

Important: Do NOT use the -G adminuser option for sarah. She should not be a member of the adminuser group.

Non-Interactive Shells

The /sbin/nologin shell prevents interactive login but allows other services (like FTP) to work. Other options include:

  • /bin/false: More restrictive, denies all access
  • /sbin/nologin: Shows a message and exits

5 Set Passwords for All Users Pending

Set the password for all three users to redhat.

Type these commands one by one:
echo "redhat" | passwd --stdin natasha
echo "redhat" | passwd --stdin harry
echo "redhat" | passwd --stdin sarah

Alternatively, you can use the interactive method:

passwd natasha

Then type redhat twice when prompted.

Password Management

The --stdin option allows setting passwords non-interactively, which is useful for scripting.

Always use strong passwords in production environments.

6 Verify the Configuration Pending

Verify that all configurations are correct.

Type these verification commands:
id natasha

Should show adminuser in groups

id harry

Should show adminuser in groups

id sarah

Should NOT show adminuser in groups

grep -E "^(natasha|harry|sarah):" /etc/passwd

Check shells: natasha/harry should have /bin/bash, sarah should have /sbin/nologin

Verification Commands

  • id [username]: Shows user and group IDs
  • grep [username] /etc/passwd: Shows user configuration
  • grep [groupname] /etc/group: Shows group members
  • groups [username]: Shows user's groups
