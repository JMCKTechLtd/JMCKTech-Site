0 Lab Introduction Completed

Welcome to the RHCSA Partition and Filesystem Management Lab. In this lab, you will create a partition, format it with a filesystem, and configure automatic mounting.

Follow the steps in order. You must complete each step before moving to the next one.

1 Create a New Partition Current Step

First, we need to create a new partition on an available disk. We'll use fdisk to create a 512MB partition.

Type this command exactly:
fdisk /dev/vda

Once in fdisk, follow these steps:

  1. Press n to create a new partition
  2. Press p for primary partition
  3. Accept the default partition number
  4. Accept the default first sector
  5. Type +512M for the last sector to create a 512MB partition
  6. Press w to write the changes and exit

Understanding Disk Partitioning

fdisk is a command-line utility for disk partitioning. It allows you to create, delete, and manage partitions on storage devices.

Key fdisk commands:

  • n - Create a new partition
  • d - Delete a partition
  • p - Print the partition table
  • w - Write changes and exit
  • q - Quit without saving

2 Update Kernel Partition Table Pending

After creating the partition, we need to inform the kernel about the partition table changes.

Type this command exactly:
partprobe /dev/vda

This command updates the kernel's in-memory copy of the partition table without requiring a system reboot.

About partprobe

The partprobe command informs the operating system about partition table changes. This is necessary after modifying partitions with fdisk to ensure the system recognizes the new partition without a reboot.

3 Format the Partition as ext4 Pending

Now we need to create an ext4 filesystem on our new partition.

Type this command exactly:
mkfs -t ext4 /dev/vda5

Note: The partition number might be different depending on your system. Use fdisk -l /dev/vda to check the correct partition number.

Understanding Filesystems

A filesystem organizes how data is stored and retrieved on a storage device. ext4 is the default filesystem for many Linux distributions, offering good performance, journaling, and support for large files and filesystems.

4 Create Mount Point Directory Pending

We need to create a directory where we'll mount our new filesystem.

Type this command exactly:
mkdir -p /mnt/data

The -p flag ensures that parent directories are created if they don't exist.

5 Configure Automatic Mounting Pending

Now we need to edit the /etc/fstab file to configure automatic mounting at boot.

Type this command exactly:
vim /etc/fstab

Add this line to the end of the file:

/dev/vda5 /mnt/data ext4 defaults 0 0

To save and exit vim:

  1. Press ESC to ensure you're in command mode
  2. Type :wq and press Enter to write and quit

Understanding /etc/fstab

The /etc/fstab file contains information about filesystems and how they should be mounted at boot. Each line has six fields:

  • Device: The partition or device to mount
  • Mount point: Directory where the filesystem will be mounted
  • Filesystem type: Type of filesystem (ext4, xfs, etc.)
  • Options: Mount options (defaults, noauto, etc.)
  • Dump: Used by dump utility (usually 0)
  • Pass: Filesystem check order (0=no check, 1=root, 2=others)

6 Test the Configuration Pending

Let's test our configuration by mounting all filesystems defined in /etc/fstab.

Type this command exactly:
mount -a

This command mounts all filesystems mentioned in /etc/fstab. If there are no errors, our configuration is correct.

7 Verify the Configuration Pending

Finally, let's verify that our partition is properly mounted and configured.

Type these commands one by one:
mount | grep /dev/vda5

Should show the mounted filesystem at /mnt/data

df -h /mnt/data

Should show the filesystem with approximately 512M capacity

lsblk /dev/vda5

Should show the partition details with mountpoint /mnt/data

Verification Commands

After configuring partitions and filesystems, always verify with these commands:

  • mount - Shows currently mounted filesystems
  • df -h - Shows disk space usage in human-readable format
  • lsblk - Lists information about all available block devices
  • blkid - Shows UUIDs and filesystem types
