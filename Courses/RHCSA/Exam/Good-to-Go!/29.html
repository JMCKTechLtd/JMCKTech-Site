Welcome to the RHCSA Cron Task Configuration Lab. In this lab, you will configure a scheduled task using the cron system for a specific user.

Follow the steps in order. You must complete each step before moving to the next one.

1 Open Crontab for User Natasha Current Step

First, open the crontab editor for user natasha to create a scheduled task.

Type this command exactly:
crontab -u natasha -e

This will open the cron editor. Add the following line to schedule the task:

23 14 * * * /bin/echo hiya

Understanding Cron Syntax

Cron uses a specific format for scheduling tasks:

23 Minute (0-59)
14 Hour (0-23)
* Day of month (1-31)
* Month (1-12)
* Day of week (0-7, 0=Sunday)

23 14 * * * means "at 14:23 every day of every month"

2 Verify the Cron Job Pending

Now verify that the cron job was properly set for user natasha.

Type this command exactly:
crontab -l -u natasha

This should display the cron job you just created:

23 14 * * * /bin/echo hiya

3 Enable the Crond Service Pending

Ensure the crond service is enabled to start automatically at boot time.

Type this command exactly:
systemctl enable crond

Note: On older systems, the service might be called crond instead of cron.

4 Restart the Crond Service Pending

Restart the crond service to ensure it picks up the new cron job.

Type this command exactly:
systemctl restart crond

About the Crond Service

The crond service is responsible for executing scheduled tasks. It reads cron configuration files and executes commands at their scheduled times.

When you add or modify cron jobs, you should restart the service to ensure it recognizes the changes immediately.

5 Verify Service Status Pending

Finally, check the status of the crond service to ensure it's running properly.

Type this command exactly:
systemctl status crond

The service should show as active (running) and enabled.

Verification Commands

After configuring cron jobs, always verify with these commands:

  • crontab -l -u username - Lists cron jobs for specific user
  • systemctl status crond - Shows service status
  • systemctl is-enabled crond - Checks if service is enabled at boot
  • journalctl -u crond - Shows service logs
