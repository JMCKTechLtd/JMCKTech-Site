Welcome to the RHCSA LDAP Authentication Configuration Lab. In this lab, you will configure your system to authenticate against an LDAP server.

Follow the steps in order. You must complete each step before moving to the next one.

1 Install LDAP Client Packages Current Step

First, let's install the required packages for LDAP authentication.

Type this command exactly:
yum groupinstall directory-client

This will install the necessary packages including:

  • krb5-workstation
  • pam-krb5
  • sssd

Understanding LDAP Authentication

LDAP (Lightweight Directory Access Protocol) is a protocol for accessing and maintaining distributed directory information services. In this lab, you'll configure your system to authenticate users against an LDAP server.

2 Configure LDAP Authentication Pending

Now let's configure LDAP authentication using the graphical configuration tool.

Type this command exactly:
system-config-authentication

Configure the following settings in the authentication tool:

  1. User Account Database: LDAP
  2. LDAP Search Base DN: dc=example,dc=com
  3. LDAP Server: ldap://instructor.example.com (use domain form, not IP)
  4. Check "Use TLS to encrypt connections"
  5. Download CA Certificate from: http://ip/dir/ldap.crt
  6. Authentication Method: LDAP Password
  7. Click "Apply"

LDAP Configuration Parameters

The LDAP search base DN defines where in the directory tree the server should search for user entries. Using TLS ensures secure communication with the LDAP server.

3 Verify LDAP Configuration Pending

Now let's verify that our LDAP configuration is working correctly.

Type this command exactly:
getent passwd ldapuser40

This command should display information about the LDAP user ldapuser40 if the configuration is correct.

Understanding getent Command

The getent command displays entries from databases supported by the Name Service Switch libraries, which includes LDAP when properly configured.

4 Test LDAP Login Pending

Finally, let's test the LDAP login functionality.

Type this command exactly:
su - ldapuser40

When prompted for a password, use: password

Note: The user will not have a home directory unless you configure autofs in the following questions. This is expected behavior at this stage.

LDAP User Authentication

After successful authentication, LDAP users can log into the system. However, without a properly configured home directory (via autofs or other means), they may not have a complete user environment.
