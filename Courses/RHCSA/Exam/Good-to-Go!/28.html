Welcome to the RHCSA File Permissions with ACLs Lab. In this lab, you will configure advanced file permissions using Access Control Lists (ACLs) to implement complex access requirements.

Follow the steps in order. You must complete each step before moving to the next one.

1 Copy the fstab File Current Step

First, copy the /etc/fstab file to /var/tmp/fstab.

Type this command exactly:
cp /etc/fstab /var/tmp/fstab

Understanding File Copy

The cp command copies files from one location to another. We need a copy of fstab to work with without modifying the original system file.

2 Verify File Ownership Pending

Check the current ownership of the copied file. It should already be owned by root:root.

Type this command to verify:
ls -l /var/tmp/fstab

You should see output like:

-rw-r--r--. 1 root root 685 Nov 10 15:29 /var/tmp/fstab

If the owner or group is not root:root, use:

chown root:root /var/tmp/fstab

3 Remove Execute Permissions Pending

Ensure the file is not executable by anyone. Remove any execute permissions.

Type this command exactly:
chmod a-x /var/tmp/fstab

Verify with:

ls -l /var/tmp/fstab

Understanding chmod

The chmod a-x command removes execute (-x) permission for all users (a). The file should now show -rw-r--r-- permissions.

4 Set ACL for Natasha Pending

Configure Access Control List (ACL) to allow user natasha read and write access.

Type this command exactly:
setfacl -m u:natasha:rw- /var/tmp/fstab

Verify the ACL with:

getfacl /var/tmp/fstab

Understanding setfacl

The setfacl command modifies Access Control Lists:

  • -m: Modify ACL
  • u:natasha:rw-: User natasha gets read (r) and write (w) permissions, no execute (-)
  • The plus sign (+) in ls -l output indicates ACLs are set

5 Set ACL for Harry Pending

Configure ACL to deny user harry both read and write access.

Type this command exactly:
setfacl -m u:harry:--- /var/tmp/fstab

Verify with:

getfacl /var/tmp/fstab

Understanding Deny Permissions

The three dashes (---) mean no read, no write, and no execute permissions. Harry will be completely denied access to the file.

6 Verify Final Configuration Pending

Verify all permissions are correctly configured as required.

Type these commands to verify:
ls -l /var/tmp/fstab

Should show: -rw-r--r--+ (note the + sign indicates ACLs)

getfacl /var/tmp/fstab

Should show ACL entries for natasha and harry

su - natasha -c "cat /var/tmp/fstab"

Natasha should be able to read the file

su - harry -c "cat /var/tmp/fstab"

Harry should get "Permission denied"

Final Permission Structure

File: /var/tmp/fstab
Owner: root (rw-)
Group: root (r--)
Other: (r--)
ACL: user:natasha (rw-)
ACL: user:harry (---)
rw- for owner (root)
r-- for group (root)
r-- for other
rw- for user:natasha (ACL)
--- for user:harry (ACL)
