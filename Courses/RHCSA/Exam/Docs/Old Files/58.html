User Management Concept

You need to create a user named "eric" and restrict their interactive login capability. This is commonly done for system accounts or service accounts that shouldn't have shell access.

The login shell is specified in the last field of the user's entry in /etc/passwd.

Step 1: Create the User

To create the user "eric":

Type: useradd eric

Understanding User Creation

The useradd command creates a new user with default settings. By default, this includes:

  • A home directory in /home/username
  • Login shell set to /bin/bash
  • A user group with the same name as the user

Step 2: Set the Password

To set a password for the user:

Type: passwd eric

Then enter the password twice when prompted

Understanding Password Management

The passwd command allows you to set or change a user's password. As root, you can set any password without knowing the current one.

Step 3: Restrict Interactive Login

To prevent interactive login for the user:

Edit the password file:

Type: vim /etc/passwd

Find the line for user "eric" and change the shell to:

/sbin/nologin

OR

/bin/false

Understanding Login Restrictions

The last field in /etc/passwd specifies the user's login shell. To prevent interactive login:

  • /sbin/nologin - Shows a polite message when login attempted
  • /bin/false - Immediately exits with no message

Alternatively, you can use: usermod -s /sbin/nologin eric

Step 4: Verify Configuration

Check user exists:

Type: id eric

Check login shell:

Type: grep eric /etc/passwd

Should show shell as /sbin/nologin or /bin/false
