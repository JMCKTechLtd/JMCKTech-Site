YUM Repository Concepts

Repository Files Stored in /etc/yum.repos.d/
Each .repo file can contain multiple repositories
Standard naming: name.repo
Key Parameters baseurl: Repository URL
enabled: 1 for enabled, 0 for disabled
gpgcheck: 1 to verify packages, 0 to disable
yum-config-manager Utility to add/remove repositories
--add-repo: Creates new repo file
--enable/--disable: Controls repo state
Verification yum repolist: Shows enabled repos
yum list available: Lists packages
yum clean all: Cleans cache

Step 1: Add the Repository

Use yum-config-manager to add the repository:

Type: yum-config-manager --add-repo=http://content.example.com/rhel7.0/x86-64/dvd

This will create a file named content.example.com_rhel7.0_x86_64_dvd.repo in /etc/yum.repos.d/

Understanding Repository Addition

The yum-config-manager command simplifies repository management by:

  • Creating the proper .repo file structure
  • Setting the correct baseurl
  • Generating a standard repository name

Step 2: Configure GPG Check

Edit the repository file to disable GPG checking:

Type: vim /etc/yum.repos.d/content.example.com_rhel7.0_x86_64_dvd.repo

Add the following line under the repository section:

gpgcheck=0

Understanding GPG Checking

GPG checking verifies package authenticity but requires proper GPG keys:

  • gpgcheck=1: Enables signature verification (default)
  • gpgcheck=0: Disables verification (use with caution)
  • For production systems, always use proper GPG keys

Step 3: Clean and Verify

Clean the YUM cache:

Type: yum clean all

Verify the repository:

Type: yum repolist

You should see approximately 4305 packages available

Verification Steps

After configuring a repository, always verify:

  • The repository appears in yum repolist
  • The correct number of packages are available
  • You can install packages from the repo
