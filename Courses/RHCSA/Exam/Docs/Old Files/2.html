User Management Methods

Command Line Use useradd to create users and usermod to modify them
Graphical Use system-config-users for interactive user management
Configuration Files /etc/passwd - User accounts
/etc/group - Group definitions
/etc/shadow - Password information
Key Commands id - Show user and group information
groups - Show group memberships
passwd - Set or change passwords

Step 1: Create Users with Admin Group

Create two users (harry and natasha) with the admin group as their secondary group:

Method 1: Command Line

Type these commands:

# useradd -G admin harry # useradd -G admin natasha

Method 2: Graphical Tool

Type: system-config-users and use the GUI to create users with the admin group

Understanding User Creation

The -G option specifies supplementary groups. The admin group must exist before adding users to it. If it doesn't exist, create it first with groupadd admin.

Step 2: Create User with Non-Interactive Shell

Create user tom with a non-interactive shell:

Command Line:

Type: useradd -s /sbin/nologin tom

Graphical Method:

In system-config-users, set the shell to /sbin/nologin when creating the user

Understanding Non-Interactive Shells

The /sbin/nologin shell prevents interactive login while still allowing other system functions. This is useful for service accounts that shouldn't have shell access.

Step 3: Verify User Configurations

Check group membership:

Type: id harry and id natasha

Output should show admin in the groups list

Check shell configuration:

Type: grep tom /etc/passwd

Output should show /sbin/nologin as the shell

Verification Commands

After creating users, always verify with these commands:

  • id username - Shows user and group information
  • grep username /etc/passwd - Shows user account details including shell
  • groups username - Shows group memberships
  • cat /etc/group - Shows all groups and their members
