Kernel Upgrade Concepts

Kernel Packages Kernel RPM packages contain the Linux kernel and modules. They are installed in /boot and modules in /lib/modules.
GRUB Configuration GRUB (Grand Unified Bootloader) reads /boot/grub/grub.conf to determine boot options. The default kernel is specified by the 'default' parameter.
Kernel Installation Use rpm -ivh to install new kernels (not -Uvh) to keep old kernels available as fallback.
Verification Check /boot for kernel files, uname -r for running kernel, and /etc/grub.conf for boot configuration.

Step 1: Check Current Kernel

Check current kernel version:

Type: uname -r

This will display the current running kernel version.

View GRUB configuration:

Type: cat /etc/grub.conf

This shows the current bootloader configuration including available kernels.

Step 2: Download New Kernel

Navigate to /boot directory:

Type: cd /boot

Download kernel package:

Type: lftp it

Then in lftp: get dr/dom/kernel-2.6.36.7.1.rpm

Exit lftp with: exit

About Kernel Packages

Kernel packages typically follow naming conventions like kernel-version.rpm. The package contains:

  • Kernel image (vmlinuz-version)
  • Initial RAM disk (initrd-version.img)
  • Kernel modules in /lib/modules/version/
  • System.map file for debugging

Step 3: Install New Kernel

Install kernel package:

Type: rpm -ivh kernel-2.6.36.7.1.rpm

This installs the new kernel while keeping the old one available.

Kernel Installation Best Practices

Always use rpm -ivh (install) rather than -Uvh (upgrade) for kernels:

  • Preserves old kernel as fallback option
  • Doesn't remove previous kernel files
  • Maintains boot options for both kernels

Step 4: Configure GRUB

Edit GRUB configuration:

Type: vim /etc/grub.conf

Set default=0 to boot the new kernel by default (usually the first entry).

Understanding GRUB Configuration

/etc/grub.conf contains:

  • default=N - Which entry to boot by default (0-based index)
  • timeout=X - Seconds to wait for user input
  • Title entries for each boot option
  • Kernel and initrd paths for each option

Step 5: Verify Installation

Check installed kernels:

Type: rpm -qa | grep kernel

Should show both old and new kernel packages.

Verify GRUB configuration:

Type: cat /etc/grub.conf

Check that default=0 points to the new kernel.

Post-Installation Checks

After reboot, verify:

  • uname -r shows new kernel version
  • System boots successfully
  • All hardware and services work as expected
  • Old kernel remains available in GRUB menu
