File Search Concepts

find command Searches directory tree for files matching criteria:
find [path] [options] [actions]
User ownership Find files by owner:
-user username
Copy with attributes cp -a preserves all attributes:
timestamps, ownership, permissions
Executing commands -exec runs command on each found file:
-exec command {} \;

Step 1: Create Target Directory

Create the directory to store results:

Type: mkdir /root/findresults

Understanding Directory Creation

The mkdir command creates new directories. When working in /root, you need root privileges (which you have in this lab environment).

Step 2: Find and Copy Files

Find files owned by jack and copy them:

Type: find / -user jack -exec cp -a {} /root/findresults/ \;

Understanding the Find Command

This command:

  1. Searches from root (/)
  2. Filters for files owned by jack
  3. Executes cp -a on each found file
  4. Copies to /root/findresults/ with all attributes

The {} is replaced by each found file's path, and \; terminates the command.

Step 3: Verify Results

Check the copied files:

Type: ls -l /root/findresults

Verification

You should see:

  • All files owned by jack
  • Original timestamps preserved (-a flag)
  • Correct permissions maintained
