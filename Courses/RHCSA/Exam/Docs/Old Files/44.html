Kernel Parameters Concepts

What are Kernel Parameters? Kernel parameters are settings passed to the Linux kernel at boot time to control its behavior.
Configuration Files /boot/grub/grub.conf - GRUB bootloader configuration
/proc/cmdline - Shows current kernel parameters
Editing GRUB Always make backups before editing
Parameters are added to the kernel line
Changes require reboot to take effect
Verification cat /proc/cmdline shows active parameters
Some parameters can be checked with sysctl

Step 1: Edit GRUB Configuration

Edit the GRUB configuration file to add the kernel parameter:

Open the configuration file:

Type: vim /boot/grub/grub.conf

Find the kernel line and add kmcrl=5:

It should look like this (add to end of line):

kernel /vmlinuz-2.6.32-71.el6.x86_64 ro root=/dev/mapper/GLSvg-GLSroot rd_LVM_LV=GLSvg/GLSroot rd_LVM_LV=GLSvg/GLSswap rd_NO_LUKS rd_NO_MD rd_NO_DM LANG=en_US.UTF-8 SYSFONT=latarcyrheb-sun16 KEYBOARDTYPE=pc KEYTABLE=us crashkernel=auto rhgb quiet kmcrl=5

Understanding GRUB Configuration

The /boot/grub/grub.conf file controls how your system boots. Kernel parameters added here will be passed to the kernel at boot time. The kmcrl=5 parameter must be added to the end of the kernel line.

Step 2: Reboot System

Reboot the system to apply the kernel parameter changes:

Reboot the system:

Type: reboot

Why Reboot is Needed

Kernel parameters passed through GRUB only take effect after a reboot. Unlike sysctl parameters, these cannot be changed at runtime.

Step 3: Verify Configuration

Check current kernel parameters:

Type: cat /proc/cmdline

Should show kmcrl=5 in the output

Verification Methods

The /proc/cmdline file shows all kernel parameters that were passed at boot time. This is the definitive way to verify your changes were applied correctly.
