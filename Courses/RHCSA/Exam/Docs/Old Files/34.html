Autofs Configuration Concepts

autofs Automatically mounts filesystems when accessed and unmounts them after a period of inactivity
Configuration Files /etc/auto.master - Main configuration file
/etc/auto.* - Map files for specific mounts
Mount Options rw - Read-write access
nfs - NFS filesystem type
hard,intr - Common NFS options
Service Management systemctl enable autofs - Enable at boot
systemctl restart autofs - Apply changes

Step 1: Install autofs

Install the autofs package:

Type: yum install -y autofs

Understanding autofs

autofs provides automatic mounting of filesystems with these benefits:

  • Mounts occur only when filesystems are accessed
  • Reduces system load by unmounting inactive filesystems
  • Simplifies management of large numbers of mounts
  • Supports NFS, CIFS, and other filesystem types

Step 2: Create Mount Point

Create the mount point directory:

Type: mkdir -p /home/guests

Mount Points

The mount point is where the remote filesystem will appear in your local filesystem:

  • Should be an empty directory
  • Must have appropriate permissions (typically 755)
  • Parent directories must exist
  • The -p flag creates parent directories as needed

Step 3: Configure auto.master

Edit the master configuration file:

Type: vim /etc/auto.master

Add this line at the end of the file:

/home/guests /etc/auto.ldap

auto.master Configuration

The master file defines mount points and their associated map files:

  • Format: <mount-point> <map-file>
  • Mount point must be an absolute path
  • Map file contains mount details (typically in /etc)
  • Multiple entries can be added for different mounts

Step 4: Configure auto.ldap

Create the LDAP map file:

Type: cp /etc/auto.misc /etc/auto.ldap

Edit the LDAP map file:

Type: vim /etc/auto.ldap

Add this line at the end of the file:

ldapuser1 -fstype=nfs,rw server.domain11.example.com:/home/guests/ldapuser1

Note: If you get -bash-4.2$ prompt when testing, change to:
server.domain11.example.com:/home/guests/ldapuser1/ldapuser1

Map File Configuration

The map file defines how each subdirectory should be mounted:

  • Format: <key> <options> <location>
  • Key: Subdirectory name under mount point
  • Options: Filesystem type and mount options
  • Location: Server and export path
  • Multi-level directories may require full path

Step 5: Enable and Start Service

Enable the autofs service to start at boot:

Type: systemctl enable autofs

Start the service to apply changes:

Type: systemctl start autofs

Service Management

autofs runs as a systemd service. Key commands:

  • systemctl start autofs - Start the service
  • systemctl stop autofs - Stop the service
  • systemctl restart autofs - Restart the service
  • systemctl enable autofs - Enable at boot
  • systemctl status autofs - Check service status

Step 6: Test Configuration

Switch to ldapuser1 to test:

Type: su - ldapuser1

Password: password

Verify home directory access:

Type: pwd (should show /home/guests/ldapuser1)

Type: touch testfile (should create file without errors)

Troubleshooting

If you encounter issues:

  • Check /var/log/messages for errors
  • Verify NFS server is accessible: showmount -e server.domain11.example.com
  • Check mount options in /etc/auto.ldap
  • For -bash-4.2$ prompt, ensure correct path to home directory
  • Restart autofs after configuration changes
