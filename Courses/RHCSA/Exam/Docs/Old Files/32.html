LDAP Authentication Concepts

LDAP Lightweight Directory Access Protocol - provides centralized user authentication and account information
Base DN Distinguished Name that identifies the root of the LDAP directory tree (e.g., dc=example,dc=com)
TLS Encryption Secures communication between client and LDAP server using certificates
SSSD System Security Services Daemon - manages caching and authentication with remote directories

Step 1: Install Required Packages

Install authentication packages:

Type: yum -y install sssd authconfig-gtk krb5-workstation

Package Overview

  • sssd: Handles authentication with remote directories
  • authconfig-gtk: Graphical tool for authentication configuration
  • krb5-workstation: Kerberos authentication tools

Step 2: Configure LDAP Authentication

Launch authentication configuration tool:

Type: authconfig-gtk

Configure with these settings:

  • User Account Database: LDAP
  • LDAP Search Base DN: dc=example,dc=com
  • LDAP Server: server.domain11.example.com
  • Use TLS to encrypt connections: Checked
  • Download CA certificate from: http://server.domain11.example.com/pub/EXAMPLE-CA-CERT
  • Authentication Method: LDAP Password

Authentication Configuration

The authconfig-gtk tool modifies several configuration files:

  • /etc/openldap/ldap.conf: LDAP client settings
  • /etc/sssd/sssd.conf: SSSD configuration
  • /etc/nsswitch.conf: Name service switch configuration
  • /etc/pam.d/system-auth: PAM authentication settings

Step 3: Start and Enable SSSD

Start SSSD service:

Type: systemctl start sssd

Enable SSSD to start at boot:

Type: systemctl enable sssd

SSSD Service

SSSD provides:

  • Caching of user credentials
  • Offline authentication capability
  • Integration with various identity providers
  • Performance improvements for directory lookups

Step 4: Verify LDAP User

Check if ldapuser1 is available:

Type: id ldapuser1

Expected output: uid=1000(ldapuser1) gid=1000(ldapuser1) groups=1000(ldapuser1)

Verification

Successful LDAP binding should allow:

  • LDAP users to appear in getent passwd
  • Authentication with LDAP credentials
  • User information lookups (id, finger, etc.)
