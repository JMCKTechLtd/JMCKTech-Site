Cron Job Concepts

Cron Syntax Minute (0-59) Hour (0-23) Day-of-month (1-31) Month (1-12) Day-of-week (0-6) Command
Special Characters * = all values
, = value list separator
- = range of values
/ = step values
User Crontabs Each user has their own crontab
Edited with crontab -e
Viewed with crontab -l
System Crontabs System-wide jobs in /etc/crontab
Or in /etc/cron.d/
Requires specifying user

Step 1: Locate the echo Command

Before scheduling a command, you should know its full path:

Find the path to echo:

Type: which echo

This should return /bin/echo

Why Use Full Paths in Cron?

Cron jobs run with a minimal environment and may not have the same PATH as your interactive shell. Always use full paths to commands and files in cron jobs to ensure they can be found.

Step 2: Edit the Crontab

Edit your user's crontab file:

Open the crontab editor:

Type: crontab -e

This will open the crontab in your default editor (usually vim or nano)

Add the following line:

23 14 * * * /bin/echo hello

This schedules the command to run at 14:23 every day

Understanding Cron Timing

The five time fields represent:

  • 23 - Minute (23rd minute)
  • 14 - Hour (2 PM)
  • * - Every day of the month
  • * - Every month
  • * - Every day of the week

Step 3: Verify the Cron Job

List your cron jobs:

Type: crontab -l

This should display your scheduled job:

23 14 * * * /bin/echo hello

Cron Job Management

Additional cron commands:

  • crontab -e - Edit your crontab
  • crontab -l - List your cron jobs
  • crontab -r - Remove all your cron jobs
  • systemctl status crond - Check cron daemon status
