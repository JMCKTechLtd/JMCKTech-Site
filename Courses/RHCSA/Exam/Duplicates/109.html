Welcome to the RHCSA String Search and File Operations Lab. In this lab, you will search for specific text patterns in a file and save the results to a new file.

Follow the steps in order. You must complete each step before moving to the next one.

1 Verify Source File Current Step

First, let's verify that the source file exists and check its contents.

Type this command exactly:
ls -l /usr/share/dict/words

This will show us the file details and confirm it exists.

Understanding the words File

The /usr/share/dict/words file is a standard dictionary file on Linux systems containing English words.

It's commonly used for spell checking and text processing tasks.

2 Search for String "seismic" Pending

Now let's search for lines containing the string "seismic" in the dictionary file.

Type this command exactly:
grep seismic /usr/share/dict/words

This will display all lines from the dictionary file that contain the word "seismic".

Understanding grep Command

The grep command searches for patterns in files:

  • grep pattern file - Basic search syntax
  • By default, grep displays matching lines to stdout
  • It preserves the original order of lines in the file
  • No blank lines are added to the output

3 Save Results to File Pending

Now let's save the search results to the target file /root/wordlist.

Type this command exactly:
grep seismic /usr/share/dict/words > /root/wordlist

The > operator redirects the output to a file instead of displaying it on screen.

Understanding Output Redirection

Linux provides several ways to redirect output:

  • > file - Redirects stdout to file (overwrites)
  • >> file - Redirects stdout to file (appends)
  • 2> file - Redirects stderr to file
  • &> file - Redirects both stdout and stderr

The > operator will create the file if it doesn't exist.

4 Verify the Results Pending

Finally, let's verify that the operation was successful and the output file contains the correct content.

Type these commands one by one:
ls -l /root/wordlist

Should show the file exists with appropriate permissions

wc -l /root/wordlist

Should show the number of lines in the output file

cat /root/wordlist

Should display all lines containing "seismic" in original order

Verification Commands

After creating files with redirected output, always verify:

  • ls -l - Confirms file creation and permissions
  • wc -l - Counts lines to ensure all matches were captured
  • cat - Displays file contents for visual verification
  • head/tail - Shows beginning/end of file for large outputs
